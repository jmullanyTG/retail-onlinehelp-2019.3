Classification Task Listing

The classification task listing will display all of the tasks in their respective execution order.

  • Task Name – The assigned name for the task.
  • Task Description – The optional description for the task.
  • Default Task – This box should be checked if the task is the default task. This will handle all classification for items which are otherwise uncovered.
  • Task Enabled – Tasks may be disabled without deleting them. If unchecked, the task will not be processed upon execution.
  • Task Execution Order – The order in which tasks are executed is important, as if there are any conflicts, the task with the higher execution order will override the previous settings.
  • Task Start Date Defined – If the task is dynamic (time sensitive) and the start date specified, the task start date defined box must be checked and the task start date field supplied.
  • Task Start Date – The start date for a dynamic task.
  • Task End Date Defined – If the task is dynamic (time sensitive) and the end date specified, the task end date defined box must be checked and the task end date field supplied.
  • Task End Date – The end date for a dynamic task.
  • Task Allowed To Execute – This flag will provide a final check regarding the task’s ability to execute. The task must be enabled (task enabled flag) and fall within the specified date range if it is a dynamic policy.
