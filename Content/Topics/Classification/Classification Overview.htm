Classification Overview

Inventory classification can help a company control its inventory by reducing the amount of stock on hand and increasing the inventory turnover. This makes distribution network more efficient and lowers the overall cost.

Classifying inventory also allows the supply manager to set up a review schedule, check inventory levels, and establish inventory control.

The Pareto Principle, or the 80:20 rule, is the secret for setting priorities. It says that "A minority of input produces the majority of results."

Although the exact split may not be 80/20 (it is, sometimes, 60:40 or 90:10) this is certainly true in inventory systems, where a few items account for the bulk of investment and return on investment.

Maintaining inventory takes time and costs money. When there are limits on these resources, you can use the available resources to control and focus on the most important items in inventory. This is accomplished by giving items priorities from A, the most important, to F, the least.

ABC Analysis is the principle used for classifying inventory items based on item's consumption values. A consumption value is the total value of an item consumed over a specified period of time.

The ABC Analysis principle divides items into three categories:

  • A: items are goods where annual consumption value is the highest
  • B: items are inter-class items
  • C: items have the lowest consumption value

XYZ Analysis is the principle for classifying inventory items according to variability of their demand. Understanding XYZ Analysis can help you improve the accuracy of forecasting.

The formula for calculating variability is as follows:

Relative Deviation = (Standard Deviation / Demand History) x 100

The results from this calculation are:

Class % Relative Deviation Results
X < 50 High Stability
Y < 50 - 100 > Low Stability
Z > 100 Volatile

ABC Analysis and XYZ Analysis can help you make better classification and policy decisions.

Classification Task

The Inventory Classification solution is comprised of a list of tasks configured, in order of priority, to define the class of an item based on historical or future units, cost value, sell value, or profit in a selected grouping (e.g. Itemcode, Site, Site Group1, Inventory Group 1), and a selected time frame.

Classification tasks work in conjunction with policy tasks to set appropriate ordering policies for inventory.

Classification Task Listing

The classification task listing displays all of the tasks in their respective execution order.

Field Description
Task Name

Default Task

Select this checkbox if the task is the default task.

Task Enabled

You can disabled a task without deleting it. If this checkbox is unchecked, the task will not be processed upon execution.

Task Execution Order

The order in which tasks are executed. In case of any conflicts, the task with the higher execution order overrides the previous settings.
