Policy Overview

Inventory policies determine the rules by which the replenishment mechanism operates. Inventory policies are designed to create the default settings for all inventories in Demand Management, but may be further manipulated for specific inventories in the Inventory Manager (workbench).

Policy Tasks

Policy tasks are individual tasks that can be assigned both at start and end dates, if required. Policy tasks are sorted by their respective execution orders in the Policy Task grid. The details of each task display below the Task grid.

It is important to select the correct filters when you build a new policy to determine the application of the task to the intended items. If an appropriate filter does not exists, you can create a filter through the Filters solution.

The main screen on the Inventory Policy solution displays the current policy tasks and their respective details.

All changes to the policy tasks must be made from the Add or Edit screens.

Field Description
Task Name

The name given to the task. Any cloned tasks begin with "Cloned From" before they are changed.
Task Description A basic description of the task.
Default Task

The default task is the fallback inventory policy for any inventories not covered by other tasks. There may be only one default task.
Task Enabled

Indicates if the task is currently enabled (active).

Task Execution Order

The order in which the task rules will be applied. This is important because some task rules may supersede others, depending on their order.

Task Start Date Defined If the task is intended to begin on a certain date, this check box must be selected, and the start date specified in the next column.
Task Start Date

If the task is intended to begin on a certain date, the start date must be specified and the check box must be selected in the preceding column.
Task End Date Defined

If the task is intended to end on a certain date, this check box must be selected, and the end date specified in the next column.

Task End Date

If the task is intended to end on a certain date, the end date must be specified and the check box must be selected in the preceding column.
Task Allowed To Execute An informational check box that displays the effective status of the task. If the task is enabled and falls within the specified date range, the Task Allowed to Execute box must be selected.

To Reorder Tasks

  1. From the Policy Task screen, click Reorder. The Reorder Policy Tasks screen displays.

  1. From the Reorder Policy Tasks screen, do one of the following:
  • Select the specific task.
  • In the Task Execution Order field, change the task execution order.
  1. When finished, click OK.

If you made changes to enabled policy tasks, you may have to execute the tasks outside of the Daily Job Manager.

General Tab

The General tab displays general policy information and the filter associated with the policy task.

To Display the Items Contained in the Filter

  • Next to the Filter drop-down menu, click View Items. The Inventory Manager displays the details for the selected filter.

Policy Tab

From the Logistics section of the Policy tab, you can add a lead time day to a policy task to indicate outbound handling lead time value to account for the need of some distribution centers to prepare stock on hand for shipment. In instances where a distribution center may not be able to accommodate an order request the day it was placed, outbound handling lead time may be applied to better represent the realistic ship date from the distribution center.

The Reset to Default checkbox resets the value in the Outbound Handling Lead Time field to Null on the Inventory Policy Task screen.
