Configuring Bin Settings

A Bin item is best suited to items with intermittent, low demand. A fixed-quantity order methodology applies. For example, to ensure that two items of the same type are available at all times, as soon as one item is sold, another one is ordered to bring the stock on hand back to the set bin level of two.

To Configure Bin Settings

  1. From the Execute tab, click Policy. The Policy Task screen displays.
  2. Click Edit. The Update Policy task screen displays.
  3. From the Order Policy Type drop-down menu on the Policy section, select Bin.
  4. Make the necessary changes according to the table at the end of this topic.
  5. When finished, click OK.
Field Description
Bin Min Units

The minimum quantity of bin units. Bin type items must have a bin level specified in the number of units. When the inventory drops below this level, the item will be reordered. Like safety stock, bin levels may also be set by service levels.
Bin Max Units Reads the quantity in the Bin Min Units field and sets the Bin Max Units field to, at least, the quantity set in Bin Max Units.
Back Order Ratio

The percentage of lost sales that are expected from returned items from customers.
Minimum Safety Stock Units

The minimum number of units maintained in stock in addition to the bin level (e.g. presentation minimums).

Return Rate

The percentage of items that are expected to be returned to the store.

Days Before Return The expected number of days between the sale of an item and the return of that item to the store.
Reuse Rate

The percentage of items that were returned and may be considered as saleable stock again.

Days Before Reuse

The expected number of days between the return date of an item and it being considered as regular saleable stock again.

Order Multiple

The multiple of units in which the stock must be ordered.

Order Multiple Rounding

The rounding rules used to determine how order quantities that do not precisely align with the order multiple should be rounded up or down.

  • Up: rounds up to the nearest order multiple.
  • Down: rounds down to the nearest order multiple.
  • Nearest: rounds to the nearest order multiple (round natural).
  • Down Protected: rounds down, unless the action of rounding creates a projected lost sale, in which case it will round up.
  • Nearest Protected: rounds nearest (natural), unless the action of rounding creates a projected lost sale, in which case it will round up. This is the recommended setting.
Excess for Redistribution Days

The number of days of excess that should be considered to be protected from a redistribution action. As a day value, the calculated unit value will be a resolved function of projected demand.

Expedite Freeze Days

The forward looking days in which orders should not be considered eligible for an expedite action.

Surplus Freeze Days

The forward looking days in which orders should not be considered eligible for a surplus action.
