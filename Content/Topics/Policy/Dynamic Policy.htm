Managing Dynamic Policy

Dynamic inventory policies can be set up with start and end dates to make specific modifications for a certain period of time. These policies can be layered to systematically affect only certain measures during eligible periods. By making adjustments into the future, this feature allows you to send the correct amount of product to the store.

Policy settings can be either replaced or modified.

To Add a Dynamic Policy

  1. From the Execute tab, click Policy. The Policy Task screen displays.
  2. Click Edit. The Update Policy task screen displays.
  3. From the Dynamic Policy section, click Add Dynamic Policy.
  4. Enter information as follows:
Field Description
Policy Date The effective start date for the dynamic policy.
End Date

The end date for the dynamic policy
Notes Optional notes field.
Policy Type

Determines how dynamic policy values will affect the default settings.

Adjust

Adjustments can be additive or multiplicative. Not applicable to all policy settings (e.g. order policy type, target service level). An additive value applies a positive or negative quantity change to the default policy value. A Multiplicative value applies a multiplicative percent adjustment to the default policy value.

Replace

A value in a replace policy type overwrites the default policy value during the effective period of the dynamic policy.

Percentage Adjust A multiplicative percent adjustment to the default policy value.

Applying a multiplicative adjustment to a ratio or percentage policy value will produce a multiplicative result. E.g. if a back order ratio is set to 50%, applying a +5% change will yield a result of 52.5% effective back order ratio, not 55%.

To Delete a Dynamic Policy

  1. From the Dynamic Policy section, select a policy the policy to be deleted.
  2. Click Delete Dynamic Policy. A message prompts you to confirm the deletion.
  3. Click Yes to confirm. The dynamic policy is deleted.
