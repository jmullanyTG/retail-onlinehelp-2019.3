Translate a Language or Language and Culture

The translate section allows users to create new string values for the culture. This is where strings are translated. The prefix of StringName indicates which parts of the application will be translated. 

  • String Name – The string name as is it used by the application itself. Without a translation, this is the value which will appear in the user interface.
  • Default String Value – The friendly name or translated name as defined in the default string maintenance section.
  • String Value – The translated value or friendly name.
  • Note – Any applicable notes.
  • Date Time – The date on which the translation was created or edited.

Locating individual strings

Users may use the search bar to locate individual strings (or use a contextual keyword). New entries may be made by clicking on the Click here to add a new row section in the grid.

Sting values may be edited by clicking on the “string value” field.

Importing or exporting translated language files

The translated languages files may be imported or exported as a Demand Management Translated Language (.tlan) file type.
