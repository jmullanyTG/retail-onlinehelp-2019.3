Lock & Spread Related Toggles

  • Remove Locks On Save: user editable check box; when checked, all locks present in the current plan are removed whenever the user clicks Save in the Main Planning, Comp/Non-Comp, or Sales By Type views
  • Remove Locks on Merge: user editable check box; when checked, all locks present in the current plan are removed whenever the user merges the plan into another plan manually or via the Approve action in the Financial Plan Maintenance dialog
  • Aggregate Locked Metrics Always Hold: user editable check box; defines the calculation behavior for dependent aggregate level metrics when an aggregate level lock (i.e. product category lock with multiple subcategories beneath, or a month level lock with weeks beneath) is present
    • Dependent Metrics: when locks are engaged for any Sales or sales related metric, all dependent metrics also lock. These are:
      • Sales
      • Sales % +/- LY
      • Sales Units
      • Sales Units % +/- LY
      • AUR
      • AUR % +/- LY
      • MMU % (Product)
      • Markdowns
      • Markdowns % of Sales
      • Markdowns (Perm)
      • Markdowns % of Sales (Perm)
      • Markdowns (Promo)
      • Markdowns % of Sales (Promo)

        • Markdowns and Markdowns % to Sales will appear locked but must still recalculate at the aggregate level when changes are made beneath

        • in the Hybrid Cost Dual Markdown plan type, Markdowns (Perm) and Markdowns % of Sales (Perm) WILL hold at the aggregate level whenever Sales/Total Sales at the aggregate level also holds. This is in all cases then Aggregate Locked Metrics Always Hold is checked and whenever Sales/Total Sales is changed directly when Aggregate Locked Metrics Always Hold is unchecked

  • Checked: when checked, all aggregate level metrics that appear locked (conditionally formatted red) will hold their values when changes are made to periods/entities beneath the locked level

    • to achieve this, dependent metrics in periods/entities beneath the locked level other than the one changed may not adhere to the active calc toggle for the metric changed

    • In the example below, a lock has been placed on 2017_m07, locking all sales dependent metrics as noted above

Sales Units have been increased to 500 for 2017_wk28 and the current Sales Units calc toggle is HOLD: AUR, AUC, AUT | CALC: Sales, COGS, Markdowns, MMU %

Note that all aggregate locked metrics (except as noted above) have held their values. At the period level, Sales have recalculated for all periods, balancing to $4,800, and Sales Units have recalculated for the periods that were not changed manually to balance to 1,200. Note, also, that Markdowns have recalculated while AUR has remained constant, indicating an underlying change to AUT – which is contrary to the active calc toggle but consistent with the behavior for when Aggregate Metrics Always Hold is checked.

  • Unchecked: when unchecked, only the value of the aggregate level metric corresponding to the metric being changed for a period/entity beneath the locked level (i.e. Sales, if Sales was changed beneath the lock, etc.) will hold while other dependent aggregate level metrics may change

    • the advantage to this option is that all changes to other periods/entities below the lock, as well as those of dependent metrics at the aggregate level will adhere to the active calc toggle of the metric that was changed

    • In the next example, the view has been changed to group sales for all brands beneath each week and a lock has been placed on 2017_w28, locking all sales dependent metrics as noted above

Sales Units have been decreased to 200 for the Great Value brand and the current Sales Units calc toggle is still HOLD: AUR, AUC, AUT | CALC: Sales, COGS, Markdowns, MMU %

Note that this time, Sales Units at the aggregate level has held, but the other aggregate level metrics have been allowed to recalculate. This creates a different calculation behavior for the brand level metrics beneath the lock as well. Users should familiarize themselves with these differences and choose the option that most closely fits their planning style
