VAT Configuration

If your organization includes VAT (value added tax) in its planning process, there are a few required configuration steps that must be completed prior to beginning to plan:

  1. Create a verbose product hierarchy: in order for VAT calculation functionality to work properly, a verbose product hierarchy is needed. This basically just means that data is able to be stored at all levels of the hierarchy rather than just the lowest leaf levels.
    For example, if my hierarchy consists of the levels Company > Gender > Class, a verbose hierarchy allows data to also be stored against the Company and Gender levels, whereas a regular hierarchy would only allow data at the Class level. Because VAT rates can be set at any level of a product hierarchy, a verbose hierarchy is needed to allow the system to read the correct rates when calculating VAT related metrics.
    To do this:
    Click System Utilities > Views then locate and open the Hierarchy Filtering view:

    Next, select the Variant entity type and click Execute:

    If you need to create a new hierarchy, click New, enter a Hierarchy Name and Description, check the Allow Data Storage at All Levels check box, then Save:

    Lastly, set the Order Number and Enabled check box for the desired Item Master Groups and Save:

    If a hierarchy already exists that you would like to use for this purpose, you can simply select the hierarchy, check the Allow Data Storage at All Levels check box and Save

    check with your Demand Management technical representative before making this change to hierarchies that are used in modules other than Assortment Planning and MFP. Other modules, such as Allocation and Group Forecasting are not designed to work with verbose hierarchies.

  • Set the MFP product hierarchy VAT rates will be assigned to: the next step in setting up MFP for planning with VAT is telling the system which hierarchy to use for getting the VAT rates to be applied to your plans.
    To do this:
    While still in the Hierarchy Filtering view, click Fields in the Hierarchy Configuration section and drag the Hierarchy ID field into the grid:

    Note the Hierarchy ID for use in the next step:

    Next, click Maintenance > Policies, select the Planning VAT Rate Product Hierarchy ID from the Module: Financial Planning folder at the left, enter the Hierarchy ID you noted in the last step in the New Setting field, and click Apply Changes:

  • Enable VAT in MFP: the final step is to enable the use of VAT in MFP. To do this, while still in the Policies module, select the Enable VAT in MFP policy from the Module: Financial Planning folder at the left, select True in the New Setting field, and click Apply Changes:

    like all system policies, the Enable VAT in MFP policy is a global setting, meaning all MFP plans will have VAT enabled once this policy is set to True. That said, if you have a VAT rate of zero set for the product/site levels included in a given plan, ex VAT metric values will be identical to their regular counterparts and there will be no mathematical effect on the plan.

Assuming you have also set up VAT rates in the Planning VAT Rates view, after the above steps are complete, you will be able to plan with VAT included in MFP. For more information on setting VAT rates, refer to Currency and VAT Management > Essential Business Concepts > Planning VAT Rates.
