Navigating the Order Manager Solution

From the Navigation pane on the left side of the Order Manager screen, you can view and work with order documents. The convenient folder system also helps you organize your orders as they move through the various stages of the ordering process.

To Access Order Manager

  • From the Execute tab, click Order Manager. The Navigation pane displays.

Removing Filters from the Document List

To remove a filter that has been applied to the document list, click the the button at the top of the Navigation header. This button is only enabled when a filter is applied.

Filtering the Document List

To apply Inventory or Variant type filters to search for orders containing specific site/item combinations on the Order Document list, click the Filter button at the top of the Navigation pane.

To create, import, update, delete, or export s filter, click the icon on the Filter window.

To display the Item Selection window, click Select only some items from the current filter Filter window.

The Item Selection window displays all variants returned by the criteria in the selected filter. To select the desired variants to include in the document, click the corresponding check boxes in the Selected column.

Alternatively, you can also click the following buttons at the top of the Navigation pane to select items:

  • : selects all rows currently highlighted
  • : de-selects all rows currently highlighted
  • : selects all rows
  • : de-selects all rows

To filter the list, type in the Filter row for any column. The Start With field displays in the Product Description column.

Enter % at the start of the value you want to filter. The Contains type filter displays.

To select additional columns to include in the Item Selection view to provide additional context for determining which products are desired, click Choose Column(s). The Choose Fields window displays.

Order Status in Order Manager

Status Description
Workspace

Initial status. Contains all order documents that are not currently in one of the other document statuses.

  • Private: contains all orders with the security setting of Private that were created by the current user.
  • Public: contains all orders with the security setting of Public for which the user has sufficient permissions to view.
  • Shared: contains all orders with the security setting of Shared for which the current user was included in the approved user list.
Authorized

Contains all order documents authorized by the current user.

Depending on user access and rights based on the active workflow for an order document, you can authorize and confirm that no further changes are required to the order and that the order is ready for upload to external systems.

Forwarded

Contains all orders that have been forwarded either by or to the current user. In Order Manager, order documents can be forwarded to other users for review/approval.

  • Inbox: contains all orders forwarded from others to the current user.
  • Outbox: contains all orders forwarded by the current user to others.
Pre-Authorized

Contains all orders pre-authorized by the current user. Depending on user access and rights based on the active workflow for an order document, you can pre-authorize, locking the order document from further changes.

    Pre-Authorization is provided in Order Manager more for the reservation of stock for orders that will ship to sites; pre-authorization in the context of commitment orders does not reserve any inventory because the inventory has not been ordered. If pre-authorization is not desired within a user workflow, the rights to pre-authorization can be removed at each node of the relevant workflow for all users in Workflow Manager.
Rejected Contains all previously reject orders that were forwarded by the current user. Orders forwarded to others for approval can be rejected, indicating that changes are required.
Uploaded

Contains all order documents authorized by the current user that were successfully uploaded to external systems.
Uploaded Failed Contains all order documents authorized by the current user that experienced a failure during the upload process.
Uploaded Verified

Contains all order documents authorized by the current user for which Demand Management has received confirmation of upload from external systems.

The system can be configured to transition the order document in one of two ways after an upload is verified:
  • Transition back to Workspace.
  • Transition to the last folder the order was in prior to being Authorized.

Additionally, the system can be configured to route the order document in one of two ways after an upload is verified:

  • Route to the user who created the order.
  • Route to the user who authorized order.
Uploading Contains all order that are currently being uploaded and have been authorized by the current user. Based on the configuration and schedule of order uploads established during software implementation, authorized orders are uploaded to the external systems that process purchase order management.

Navigation Pane Positioning and Viewing Options

Auto-Hide

To hide de Navigation pane, click the (pin) icon on the top of the Navigation pane. The Navigation pane moves to a tab on the left side of the screen.

To restore the Navigation pane to its default position, hove over or click the tab, then click the pin icon to dock the pane.

Manual Docking

The Navigation pane can be docked on the left or right side of the screen. To do so, left-click, hold and drag the Navigation header area. The pane temporarily becomes a floating window.

Drag the pane to the right or left side of the screen.
