Help Overview

The Demand Management Help Module manages the location and type of help documentation which the application can access. Details of the help context are displayed and maintained within the single help context management screen of the Help Module.

  • Help Context Key – The name used to identify the location in which the help will launch. The help context key will be one of the module UIL files.
  • Help Type – The file type of the associated help document.
    • Open File – A plain text file which may be opened by an outside application on the client computer (e.g. MS Word, Notepad, etc.)
    • HTML Help Pages – A website which is accessible through use of a web browser.
    • Microsoft Help – A windows compatible Microsoft help file, such as a .chm file.
  • HTML URL – If using an HTML type help document, this field should contain the destination URL.
  • MS Help File Location – If using a Microsoft help file (.chm, etc.), this will be the file name. All .chm files should be loaded into the database (using the files module) and may be referenced only by their full file name.
  • MS Help Navigator Type – The Microsoft help navigator type assists the help module in directing the help action to the appropriate section within a .chm document. There are two types which should be used for most cases. The help file opens to:
    • a specific topic, if the topic exists – Used to link to a specific segment of a larger .chm file (such as a sub-heading). Requires an accompanying MS Help Parameter to work.
    • the table of contents – defaults to the table of contents for the .chm file. Default for module specific help.
    • the index – Opens to the index. Not intended for use with current Demand Management help files.
    • the search page – Opens to the search page. Not intended for use with current Demand Management help files.
    • the index entry for the first letter of a specified topic – Opens to the index entry for the first letter of a specified topic. Not intended for use with current Demand Management help files.
    • the topic with the specified index entry, if one exists; otherwise, the index entry closest to the specified keyword is displayed. – Gives multiple options for how the file is opened. Not intended for use with current Demand Management help files.
  • MS Help Parameter – If the help navigator type required a parameter (if using “the help file opens to a specific topic, if the topic exists”, a parameter will be required), this field will hold the parameter.
  • Other File Location – If using “open file”, this will be the file path of the file.

Importing a Demand Management Help Configuration (.jehc) file

Form most installations of Demand Management, the help files will be pre-loaded into the user interface by way of the database. During implementations, it may be necessary to import a Demand Management Help Configuration (.jehc) file, which will contain all of the appropriate file links.

Creating or Modifying Entries

If desired, users may create new entries or modify entries to point to a new location. This can be useful for custom workflow screens which would benefit from specific help documentation.

A new line may be added by clicking the “add” button. The highlighted row will be deleted by clicking the “delete” button. Any changes made to the grid will enable the “save” and “undo” buttons.

Creating Entries for MS Help Files (.chm)

When building a help configuration entry to accompany a .chm help file which has already been added to the database using the Files Module, there are a few tricks and tips which will make it an easy process.

  1. Getting the Context Key – The context key is the module file name. As some of these can be quite verbose, an easy way to collect this syntax has been included.

    1. Navigate to the Module and click the help button or press “F1”. The following window will pop up, informing the user that no help file currently exists.

    2. Click Yes. The context key syntax will be copied to the clipboard.
  2. Return to the Help Module and paste in the context key.
  3. Getting the MS Help Parameter – If using the “help file opens to a specific topic, if the topic exists” option to link to a certain sub-heading within a .chm file, the required parameter must be included.
    1. Open the .chm file and browse to the desired sub-heading.
    2. Right click and choose properties.
    3. Under the “Address (URL)” section, the end of the entry will display the file name and link for the specific sub-heading.

  4. Copy this syntax and paste it into the MS Help Parameter.

Importing and Exporting Help Configuration Files

The most common way in which the Demand Management help will be configured is via a full import of a .jehc file. This will contain the links to the appropriate help file locations, however, it does not contain the help files themselves. These must still be imported using the Files Module.

Loading .chm help files into the database

To load .chm help files into the database, the functionality included in the Files Module is the optimal tool (reference the help in the Files Module).
