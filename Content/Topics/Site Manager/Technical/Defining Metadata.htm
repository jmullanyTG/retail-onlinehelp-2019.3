Defining Metadata

The data from the created view will need to be referenced by field in order for the filtering machinery to recognize and return it. This reference exists in tbl_Common_Filter_MetaViewData and should be populated by the initialization (INIT) procedure which was created to accompany the view.

Field Description
FilterTypeID

Primary key referenced in the tbl_FilterType table.

ViewColumn

The column name as it appears in the referenced table or view in the tbl_FilterType table .

TableName

The name of the table or view that contains the data.
TableColumn

The column name as it appears in the referenced table or view n the TableName field.
FriendlyName

The friendly name as it appears in the user interface of the application, before any translations. This is the name referenced in all standard product documentation.

VisibleFlag

Determines if the field is in the user interface. Can be set to 0 (zero) and used for filtering only.

Updateable

If set to 1, the field can be edited by users.

LookupField

If the field is a lookup field, the string value must be entered in this column. Can be used to refer an ID column to a description column.

Aggregatable

The aggregation type for the field.

  • 0: None
  • 1: Sum
  • 2: Average
  • 3: Max
  • 4: Min
  • 5: Count
  • 6: Custom
  • 7: Custom Distinct Sum
  • 8: Custom Distinct Average
  • 9: Custom Distinct Count
Priority

The inherited priority for the field. If more than one priority must be applied, enter the information in separate rows. Priority 1 inherits priority 0 (zero) abilities.

  • 0: Filtering
  • 1: User Interface
UpdateTableName

If the value can be updated, the name of the write-back destination must be specified.

ExpressionColumnFlag

If the column is an expression column, set this value to 1.
ExpressionColumnString

The expression to be used.

ColumnFormatStringFlag

If active, adheres to standard format string for user interface presentation.

ColumnFormatString

Standard format string for user interface presentation.
ParentTableColumn Helps with lookup data related to trees.
ApplyUserItemAssignOnLookupDataFlag

If selected, applies user assign logic to lookups.

LookupFlag If selected, identifies the lookup.
