Site Manager Widgets

Site Modeling

The hierarchy used to populate the product dimension in the Site Modeling widget is determined by the Site Modeling: Product Hierarchy system policy.

Sales History

Site modeling creates entries in tbl_SalesHistoryByType as supersession sales history. In order to process the site modeling data, usp_Site_Model_ProcessModelData must be run. It is recommended to add this procedure as a task in the daily job, or as frequently as sales history is updated in Demand Management.

Site Opening Date

The site opening date is read from tbl_Site_Status_Temporal. It is the minimum of start dates where site status is 1 (open).

DC Source Priority

The supply chain used to populate the widget is determined by the SiteManager.DCSourcePrioritySupplyChain system policy.

Site Source Priority

The supply chain used to populate the widget is determined by the SiteManager.SiteSourcePrioritySupplyChain system policy.
