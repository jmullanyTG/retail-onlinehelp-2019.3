Site Modeling

The hierarchy used to populate the product dimension within this widget is determined by the following system policy: Site Modeling: Product Hierarchy

Sales History

Site modeling will create entries in tbl_SalesHistoryByType as supersession sales history. In order to process the site modeling data, usp_Site_Model_ProcessModelData must be run. It is recommended to add this procedure as a task within the daily job, or as frequently as sales history is updated within Demand Management.

Site Opening Date

The site opening date is read from tbl_Site_Status_Temporal. It is the minimum of start dates where site status is 1 (open).
