Adding Entries to Filter Type

After you created the views (or after the data tables have been populated), you must enter data in the tbl_FilterType table. This table determines how the information in the view is organized and accessed in the user interface and by the filtering machinery.

Field Description
FilterTypeID Unique identifier that is referenced as the primary key by the tbl_Common_Filter_MetaViewData table.
FilterTypeDescription The grouping for the filter data.
FilterViewName

Name of the view (or table) that was created and contains the reference data.

MetaDataViewName

The reference table containing the metadata. Best practice is to set this value to tbl_Common_Filter_MetaViewData.
PrimaryKeyFieldName The primary key referenced in the view.
DuplicateFlag

Not used.
DefaultFlag If set to 1, the data from this view displays in the grid when of all document types for the filter entity are launched.
FilterEntityTypeID The ID of the filter entity type.
