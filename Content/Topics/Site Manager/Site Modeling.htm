Site Modeling

When opening a new store or Distribution Center (DC), you can choose an existing store or DC to model forecasts after. This creates a store supersession, which uses the product-level sales history from the model site to forecast demand for the new site. Orders for the new site can then be generated based on the modeled forecasts.

Site Model Options

Site model options functions as the header for the supersession/model. You can select the start and end dates of the model and its active status.

Field Description
Superssesion Start Date The start date for the model. This is the first period of historical sales that is considered.
Superssesion Start Date The end date for the model. This is the final period of historical sales that is considered. If this date is beyond the date on which the new store has accumulated actualized sales, the model continues to be used to represent sales history.
Active A flag used to determine the active status of the supersession.
Site Opening Date The date on which a site is set to open. To be used for new store openings. This is not a user-editable field.
Created Date Date of creation for the supersession.
Created By User who created the supersession.
Last Updated Date of the most recent update to the supersession.
Last Updated By User who last updated the supersession record.
Model Last Process The supersession model must be reprocessed periodically. This task must be done as part of a job that coincides with sales history updates, such as nightly jobs.

Site Model Details

Site model details allows you to determine the product levels that will contribute to the creation of the model and their relative contributions.

Field Description
Product Level The level in the product hierarchy that can be modeled. The available hierarchy levels are controlled through a system policy setting.
Model Site The site used to create the model.
Model Site Opening Date

The opening date for the modeled site. This is not a user-editable field.

Factor

The ratio applied to the sales history of the modeled site. A value of 1 applies 100% of the sales to the model.

Active

A flag used to determine the active status of the product level in the model.
Created Date Date of creation for the model line.
Created By User who created the model line.
Last Updated Date of the most recent update to the model line.
Last Updated By User who last updated the model line record.

You can use the Site Model Options widget to create site model headers and then determine the effect of the models through the Site Model Details widget.

Site Model Options

The Site Model Options widget is used to create site level supersessions by determining a start date, an end date, and an active status. Any supersession without a start date goes to the beginning of the history in the system; any supersession without an end date continues in perpetuity.

Site Model Details

After the product level hierarchy has been set for the system (through use of a system policy), the applicable product levels display as rows in the Site Model Details grid.

For each line, you must select a model site to determine the set of historical sales to be copied and a factor. The factor determines the percentage of sales to be applied in the model. A factor of 1 copies 100% of the sales quantity; a factor of 0.5 copies 50% of the sales quantity.
