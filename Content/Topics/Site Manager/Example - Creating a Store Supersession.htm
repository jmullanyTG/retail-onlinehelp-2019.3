Example - Creating a Store Supersession

Because new stores do not have sales histories of their own to base demand forecasts on, the sales histories from comparable stores can be used instead. In this example, we will identify an existing store to serve as a model for a new store in Calgary. Then, we will create a store supersession, which tells Demand Management to use product level sales histories from the model store when forecasting demand for the new Calgary store.

To Create a Store Supersession

  1. From the Management tab, click Site Manager. The Items window displays.

  2. From Filter drop-down menu, select All Sites. The Site Manager Document displays.
  3. From the Columns toolbar, click Choose. The Choose Fields window displays.

  4. From the Choose Fields window, select Site Group Codes 2 to 5.

  5. In the Site document, filter site groups 2-5 to find a suitable store model for Calgary:

    • Site Group 2 Code = Canada
    • Site Group 3 Code = Retail
    • Site Group 5 Code = Medium Format

    The results show two similar stores: Edmonton and Montreal. For this example, we'll choose Edmonton as the model store for Calgary.

  6. Select Calgary in the grid.
  7. In the Site Modeling tab, click New in the Site Model Options toolbar to create a new site model.
  8. In the Site Model Options grid, enter the following details for the new site model row:
    • Supersession Start Date: Choose a date at least one year in the past. This is the date from which sales history will be copied from the model site (Edmonton) to the new site (Calgary).
    • Supersession End Date: Choose a date two years from today. This is the date on which sales history will stop being copied from the model site to the new site.
    • Click the Active check box to mark the supersession as active.
    • Click Save.
  9. Now we need to tell Demand Management which item sales histories from Edmonton to copy through this supersession. For this example, we'll select sales from all product levels.
    1. In the Site Model Details grid, select all rows.
    2. Right-click in the Model Site column and choose Multiple Changes.
    3. In the Multi-Data Change Form dialog box, choose Edmonton from the New value field list and click OK.

    4. Click Save in the Site Model details area to save Edmonton as the model site for all product levels at the Calgary store.

Results

Demand Management starts using the sales histories from the Edmonton store to generate demand forecasts for items in the new Calgary store. You can generate these forecasts manually or wait for Demand Management to generate the forecasts automatically as part of its scheduled overnight tasks. Once generated, you can review demand forecasts for the new store to see if any revisions may be required.
