Global Site Priority

Global site priority and priority group settings alter the way in which the replenishment and allocations ordering engines prioritize stock distribution. The replenish engine ostensibly operates in two separate modes based on whether or not any values are present in priority group fields for any site.

Site Prioritization Logic

In a constrained distribution center (DC) stock situation, with more than one store requesting stock:

  • If priority groups exist (meaning that the values are null, not set to 0) the stores are prioritized by their global site priority on a day-by-day basis when their stock requests are fulfilled.
  • If priority groups do not exist, stores are grouped by their respective priority group value, then ordered by the requested demand quantity on a day-by-day basis when their stock requests are fulfilled.
  • If all stores are in a single priority group, they are ordered by the requested demand quantity on a day-by-day basis when their stock requests are fulfilled. This can be used to create a demand prioritization version of the replenish engine.

Global & Group Priority

Global and group priority (G&G) is a globally applicable view that is not driven by the selected site in the top grid. I this view, you can rank sites based on priority as it relates to receiving stock in potential shortfall situations.

Priority group allows you to protect a specific set of sites before referring to global priority to resolve shortfall situations.

  • Site Code: site code for the store or DC.
  • Site Type: the site type (DC, Store, and DC/Store).
  • Site Description: the name of the site.
  • Global Priority: unique, globally applied priority. Used to resolve shortfall situations. 
  • Priority Group: associated priority group. Used to resolve shortfall situations.  Takes precedence over global priority.
