Planning VAT Rates

Some countries include tax in the actual ticket prices of products sold, rather than calculating it as a percent of the selling price during sales transactions. This is known as value added tax or VAT. In this view, users can manage VAT rates at any level of any defined product hierarchy and for Site Groups 1-20, as defined during implementation.

All ticket prices (AUT) within Assortment Planning are assumed to be VAT Inclusive unless otherwise noted by an ex VAT descriptor in the metric name.

Even if VAT does not apply to your business, you must assign a VAT Rate of zero to the highest level of all product hierarchies to be used in assortment plans. Because Site Groups are not explicitly tied to site hierarchies, you may need to enter this zero rate for more than the highest level Site Group, depending on how your groups are defined.

This screen can be accessed by navigating to System Utilities > Views > Maintenance > Planning VAT Rate.

Toolbar buttons

Button Description
Parameters Toggles between displaying and hiding the Parameters area of the view
Execute

Updates the view with all defined VAT rate entries for the intersection of the selected Hierarchy Level, Site Group, and time frame

With regard to time, any entries that overlap any portion of the time frame specified by the selected Start and End Dates will be shown.

Saving or restoring custom layouts

Custom layouts can be saved or restored by right-clicking in the Pivot Grid and selecting Save Layout or Restore Layout, respectively.

Start dates and end dates

The latest VAT rate line for a given product level will always default to the end of the system time line (currently Dec 31, 2999) to always provide an active VAT rate for system calculations. When setting VAT rate changes beyond the latest entry, the previous latest entry will change its End Date automatically to just before the new entry time frame begins.

If the latest VAT entry for Mens Short Sleeve Tops is currently from January 1, 2018 to Dec 31, 2999 (the system default) and I enter a new line with a start date of June 1, 2018 and an end date of Dec 31, 2018, the previous latest entry’s End Date will change to May 31, 2018. The new entry will begin on June 1, 2018 as entered but the End Date will change to Dec 31, 2999 to provide VAT rates through the future.

Site group columns

The columns shown depends on the levels that exist within the selected Site Group.

Column Description
VAT Rate User editable decimal (0.0001) (non-editable for saved lines); sets the VAT Rate for the specified product level and time frame
Site Count

Non-editable integer; displays the number of sites to which the VAT Rate will be applied; updates after Apply for new lines

VAT rates cannot be set for newly entered lines until they have first been saved via saving. This is a current system limitation rather than an error.
