Price Translation Hierarchy Assignment

Once Price Translation Rules have been created, they can be assigned to levels of a selected product hierarchy allowing for a great deal of flexibility in pricing strategies across different types of products.

Even if no price translations are needed, you must create at least one rule with Translation Factors of 1 for all From and To currencies to be used and with a Start Date of the earliest date to be used within any of your assortment plans and assign the rule to the Total Company level of the product hierarchies used in those plans.

Price Translation Hierarchy Assignments work in an inheritance scheme across the selected hierarchy. This means that rules set at higher levels of the hierarchy are automatically inherited by the levels beneath unless another rule is assigned at a lower level.

This screen can be accessed by navigating to System Utilities > Views > Currency > Price Translation Hierarchy Assignment.

Toolbar fields

Field Description
Hierarchy The product hierarchy to which you want to assign Price Translation Rules

Toolbar buttons

Button Description
Parameters Toggles between displaying and hiding the Parameters area of the view
Execute Updates the view with a grid of all levels of the selected Hierarchy and any Assigned Rules for each

Grid fields

Field Description
Level Description The description of the product level
Level Description Path Displays the full hierarchy path of the product level, with each level separated by “//”
Hierarchy Level Columns The specific hierarchy assignment at each level; columns displayed depend on the levels in the selected Hierarchy
Assigned Rule The Price Translation Rule to assign to the hierarchy level at the left

New Price Translation Rules must be saved, and the view must be re-Executed for new rules to appear for selection.
