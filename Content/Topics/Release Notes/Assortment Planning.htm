Assortment Planning

The following enhancements are now available in the Assortment Planning solution.

Managing Assortments from the top down EPIC 59422

Demand Management v2019.3 introduces the ability to manage Assortments top down via a new view labeled Top Down Sales Plan. The Top Down Sales Plan view has been added to the Visualize section on the top toolbar. This new view allows users to:

  • change one or many items at a time in one place
  • modify values at a total level which then spreads proportionate to sales
  • see how sales changes affect stock/receipts

Changes that are saved are then updated to the individual item plan.

Setting default time periods between a markdown date and a phase out date DMDEV-61862

A new column labeled Number of Markdown Periods has been added to the Manage Assortment Defaults screen. Setting a value in Number of Markdown Periods sets a standard default time period between markdown and phase out dates meaning pre-season plans are more consistent and more accurately account for markdown sales during planning. The default is used to determine what the phase out date should be from the markdown date, once a user enters a markdown date during Placeholder creation.

For example, if the default is 4 and the markdown date is 4/29, then the phase out date will be 5/27 which is four weeks post markdown.

Linking markdown and phase out dates DMDEV-62359

Markdown and phase out dates can be linked together by selecting a new column labeled Markdown Offset Lock. Selecting Markdown Offset Lock ensures the two dates work dependently on each other ensuring users can accurately plan the entire life of the item including markdowns.

  • If the user changes Markdown Date, then the value in Phase Out automatically changes to keep the specified number of periods between them.
  • If the user changes Phase Out, then the value in Markdown Date automatically changes to keep the specified number of periods between them.

Markdown Offset Lock can be selected on the Hindsight and Manage Assortment Defaults screens.

New and revised Sell Through metrics in to the Main Planning View DMDEV-61865

To assist in the planning of the Sell Through % prior to markdown and markdown to phase out, three new Sell Through metrics have been added to the Main Planning View for both Placeholder and Collection modes and one existing Sell Through metric has been revised. The metrics are:

  • Target S/T %
  • S/T %
  • Reg S/T %
  • S/T % Delta

Using default ratios to automatically plan post-markdown sales units amounts DMDEV-62651

Default ratios can now be set using a new column labeled Markdown Sales Plan Factor that has been added to the Manage Assortment Defaults screen. This new column lets users automatically plan post-markdown sales units amounts to more efficiently plan Assortments.

Preventing the system from ordering additional stock past the planned markdown date DMDEV-60239

When set to False, a new system policy labeled Allow Receipts After Markdown (Maintain > Policies > Engine: Commitments) prevents the system from ordering additional stock past the planned markdown date. The default is False. Set the system policy to True if additional stock should be ordered beyond the planned markdown date.

Reviewing markdown planning changes in the Hindsight Placeholder Management screen DMDEV-61706

A new column labeled Markdown has been added to the Hindsight Placeholder Management screen. The new column is located between the phase dates.

Reviewing placeholders with a markdown date DMDEV-62458

If you select a Placeholder with a markdown date, then the Cluster Sets view refreshes to display a View Style drop down at the top of the view.

The View Style drop down has the following:

Option Description
Default Displays the Totals
Markdown Allows you to view regular and markdown values separately
All Displays all values for an overall view

Setting the maximum date for a Target DMDEV-62717

A new mandatory field has been added to the Target Creation module. The drop down menu for the new field has two options: Phase Out Date and Markdown Date. The default is Phase Out Date.

  • If a user selects Phase Out Date, then the maximum date will be the phase out date.
  • If a user selects Markdown Date, then the maximum date will be the Markdown Date, and Phase Out will be calculated based on the default markdown logic.

New visibility to the difference between internal commitments and external orders DMDEV-62314

A new metric labeled On Order Exception has been added to the Assortment Receipt Plan view. This new metric displays the discrepancy when the internal and external order quantities and dates do not match exactly.

Hold Sales function revised for markdown items DMDEV-62374

Hold Sales functionality has been revised to not redistribute sales during a markdown period to ensure that the Total Sales for a plan is preserved while applying a promotion.

Supplemental Commitments DMDEV-64483

You can make supplemental commitments during a period within which an existing commitment order already exists. These additional commitment orders are supplemental and are neither an update nor a replacement. The ability to create a supplemental commitment is controlled by the Allow supplemental commitments system policy.

Placeholder description field DMDEV-62526

A new field labeled Placeholder Desc has been added to the Sales Plan view and the Receipt Plan view.
