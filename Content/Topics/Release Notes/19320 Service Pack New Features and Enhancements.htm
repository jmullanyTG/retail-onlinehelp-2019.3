New Features and Enhancements

The Demand Management 2019.3.2.0 Service Pack includes the following new features and enhancements:

Assortment Planning

With the new system policy Reference Sales Override Values for Carryover Sales in Target Creation, users now have the ability to choose whether to use the system-generated sales plan or the user revised sales plan within Target Creation. For more information, see System Policies.

To improve consistency, the Stock To Sales metrics in the Sales Plan view have been updated to calculate in the same way as the Stock To Sales metrics in the Top Down Sales Plan view.

The calculation is:

Leaf level Stock Units to Sales Units = BOP Units / Sales Units

Aggregating stock to sales over time = (First period BOP + all periods' incremental incoming stock) / Sum of Sales Units

Aggregating metrics within a period = Sum of BOP / Sum of Sales Units = Geography / Product Total STS

Replenishment Engine

The Replenishment Engine has been updated for Constraint Service Optimization. A new system policy was created: Run Supply Chain Constrained Process. This allows the engine to run in unconstrained and then run our new constraint service.

This only works for supply chains with NO ALTERNATES.

System Policies

The following system policies have been added:

Module: Assortment

System Policy Name:

Reference Sales Override Values for Carryover Sales in Target Creation

System Name:

Assortment.Target.ShowSalesOverrideValues

Description:

Reference sales override values for Carryover Sales in Target Creation for future periods

Module: Replenish Engine

System Policy Name:

Run Supply Chain Constrained Process

System Name:

Replenish.RunSupplyChainConstrainedProcess

Description:

Defined whether the execution path by which the replenish engine operates relative to supply chain constraints is used or not.
