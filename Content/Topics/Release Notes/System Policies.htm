System Policies

The following system policies have been added.

Display Name Description
Allow Receipts After Markdown When set to False, this policy prevents the system from ordering additional stock past the planned markdown date. The default is False.

Set the system policy to True if additional stock should be ordered beyond the planned markdown date.

System Name: Assortment.Engine.AllowReceiptsAfterMarkdown
Allow supplemental commitments Allows for supplemental commitment orders for a period which already contains a confirmed commitment order. The possible settings are:
  • 0 = Do Not Allow.
  • 1 = Allow Supplemental Commits and Enforce Constraints on Supplemental Commits.
  • 2 = Allow Supplemental Commits and Do Not Enforce Constraints on Supplemental Commits.

System Name: Assortment.OrderManagerAddAssortmentPlanCommit.AllowSupplementalCommits
Promotions Applied to Markdown Default Determines default property of whether promotions can be applied to a markdown period for an item. This can be overwritten at a promotion and promotion application level. If True, the default is that promotions can be applied to markdown periods.
SqlServerPivotGrid Maximum number SQL Data Rows The maximum number of rows returned for any SQL Server Pivot Grid, if exceeded, an error will be displayd in the UI. Default(60000).
Style Bank Line Vizualization Maximum number SQL Data Rows The maximum number of SQL data rows that can be queried in the Style Bank in the Line Visualization view.
Variable Numeric Precision A new system policy labeled Variable Numeric Precision addresses the need at the time of implementation to configure Pricing Variables to have different numeric precision for different geographic locations to be inline with local currencies. This system policy is hidden and is only set once at the time of implementation. This system policy is not changed after initial implementation. DMDEV-62594

Variable Numeric Precision sets the database field precision to ensure efficient updates and selects and maintenance of database growth for configured Pricing Variables and their assignment to Pricing Strategies. Further refinement of each Pricing Variable's formatting is implemented via the Price Manager Configuration screen.
