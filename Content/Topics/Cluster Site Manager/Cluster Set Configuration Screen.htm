Cluster Set Configuration Screen

In the Cluster Set Configuration screen, you can set up the actual parameters used in the creation of cluster templates and sets, including the cluster type, the sites to be included, the number and descriptions of the actual clusters, the range of data to be used, the specific criteria to be considered in the clustering algorithm, and any partitioning to be used in the generation of multiple cluster sets.

Cluster Set Properties

Property Description
Name The name of the cluster set/template.
Description An optional description of the cluster set/template.
Group The group that the cluster set/template belongs to. Click the Change button to select a different group.
Security

Indicates if this cluster set/template is available to all users (Public), some users (Shared), or just you (Private).

If you select Shared, click the Change button to select the users and/or roles to share with.
Module

The Demand Management solution (s) that this cluster set/template is available to.

Notes:

  • The cluster set/template is automatically filtered out of solutions that are not selected in this property. This prevents you from having to sort through the clutter of cluster sets that are not intended for use in a particular solution.
  • Allocation and Assortment include support for their respective scenarios in the Rate of Sale module.
Cluster Type The cluster set/template type.
Cluster Partition Site Filter By Field

When the Cluster Type property is a Site Item Partition or Site Partition, this property sets the site property used to partition cluster sets.

All site properties defined in Site Manager are available for selection, including basic store information, site attributes, hierarchy levels, site group, capacity, schedules and type.
Site Filter Type

The method used to determine the total group of stores to be clustered:

  • Site Hierarchy Level
  • Site Filter
  • Intersection Site Hierarchy Level And Site Filter
Site Hierarchy The site hierarchy to use to define the stores to be clustered.
Site Level The level of the selected Site Hierarchy to define the stores to be clustered. (Property not visible when Site Filter is selected as the Site Filter Type.)
Site Filter The site filter to use for defining the stores to be clustered.
Cluster Partition ItemMaster Hierarchy When the Cluster Type property is an Item Partition or Site Item Partition, this sets the product hierarchy to use for clustering data selection and cluster set partitioning.
Cluster Partition ItemMaster Level When the Cluster Type property is an Item Partition or Item Site Partition, this property defines the product hierarchy level of the selected Cluster Partition ItemMaster Hierarchy to be used in clustering data selection and cluster set partitioning.
Cluster Items

The clusters to assign stores to when the cluster set/template is calculated.

Click Maintain to create, delete, and edit clusters.
Cluster Settings The priority ranking of each cluster selected in the Cluster Items property. Adjust ranking up and down using the arrows on the right side of the cluster list. The cluster with the lowest number has the highest ranking and contains the stores with the highest relative value.
Attribute Settings

The priority ranking of each attribute used to calculate cluster sets. Adjust ranking up and down using the arrows on the right-hand side of the attribute list. The attribute with the lowest number has the highest ranking and will contain the stores with the highest relative value.

For more information about attributes, see Attribute Settings.
Distance Function

The distance function to use when calculating cluster sets.

  • Euclidean Distance calculates distance in a straight line (as the crow files).
  • Manhattan Distance calculates distance between two points as a sum of horizontal and vertical lines on a grid (no diagonals).
Allow Cluster Count to be Resized

Indicates if all clusters in the cluster set must have sites assigned to them. When selected, Demand Management ranks stores naturally according to the results of the selected Cluster Type algorithm. This may leave some clusters without any stores assigned to them.

If this option is not selected, Demand Management at least one store to is assigned to each cluster in the cluster set.

Cluster Types

Cluster types are based on K-Means and K-Medoids statistical clustering algorithms.

K-Means estimates and plots a number of data centers (centroids) equal to the number of desired clusters specified by the user amidst a field of store-specific data points and then tests the distance of each data point to each data center to determine which cluster each point belongs to.

K-Medoids is similar to K-Means, but identifies actual data points in the data field as data centers (medoids) and uses an alternate distance calculation which is more robust to the presence of noise and outliers in the data set than K-Means; consequently, it also takes much longer to resolve than K-Means.

Type Description
K-Means Creates a single K-Means style cluster set.
K-Means Collection Item Partition

Creates a K-Means style cluster set template and multiple cluster sets based on the lowest level of the cluster partition item master hierarchy.

the lowest level of the cluster partition item master hierarchy is Subclass and contains the subclasses Tees, Woven Tops, Knit Tops, and Sweaters. Set up a sale attribute to consider all sales from the last year to create separate cluster sets for Tees, Woven Tops, Knit Tops, and Sweaters using the last years’ worth of sales for each product area.
K-Means Collection Site Item Partition Creates a K-Means style cluster set template and multiple cluster sets based on the intersection of the lowest level of the cluster partition item master hierarchy and the values in the selected Cluster Partition Site Filter By Field.

the lowest level of the cluster partition item master hierarchy is Subclass and contains the subclasses Tees, Woven Tops, Knit Tops, and Sweaters. The Cluster Partition Site Filter By Field is set to Store Capacity, which contains the values High, Medium, and Low. Set up a sale attribute to consider all sales from the last year to create the following cluster sets using the last years’ worth of sales for the intersection of the product areas and stores that meet the store capacity criteria:

  • Tees – High, Tees – Medium, Tees: Low
  • Woven Tops – High, Woven Tops – Medium, Woven Tops: Low
  • Knit Tops – High, Knit Tops – Medium, Knit Tops: Low
  • Sweaters – High, Sweaters – Medium, Sweaters: Low
K-Means Collection Site Partition

Creates a K-Means style cluster set template and multiple cluster sets based on the values in the selected Cluster Partition Site Filter By Field.

the Cluster Partition Site Filter By Field is set to Store Capacity, which contains the values High, Medium, and Low. Set up a sale attribute to consider all sales from the last year to create separate cluster sets for High, Medium, and Low capacity stores using the last years’ worth of sales for stores that meet the store capacity criteria.
K-Medoids Creates a single K-Medoids style cluster set.
K-Medoids Collection Item Partition Creates a K-Medoids style cluster set template and multiple cluster sets based on the lowest level of the cluster partition item master hierarchy.
K-Medoids Collection Site Item Partition Creates a K-Medoids style cluster set template and multiple cluster sets based on the intersection of the lowest level of the cluster partition item master hierarchy and the values in the selected Cluster Partition Site Filter By Field.
K-Medoids* Collection Site Partition Creates a K-Medoids style cluster set template and multiple cluster sets based on the values in the selected Cluster Partition Site Filter By Field.
Manual K-Cluster Creates a single manually defined cluster set.
