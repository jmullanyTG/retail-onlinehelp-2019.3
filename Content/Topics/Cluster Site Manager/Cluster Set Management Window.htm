Cluster Set Management Window

In the Cluster Set Management window, users can view a list of all the cluster set templates, and their related cluster sets, to which they have access. Here, they can select templates or sets to view, edit, delete, and create cluster set snapshots for, and initiate the creation of new templates and sets.

To Access the Cluster Set Management Window

  1. From the Management tab, click Site Clustering. The Cluster Set Management window displays.
  2. To view a cluster set document for a given cluster set, select the cluster site in the grid and click OK. The Cluster Set screen displays.

Cluster Set Toolbar

Action Description
New Displays the Add Cluster Set window to create a new cluster set or template.
Clone Displays the Update Cluster Set window with all parameters and criteria from the selected cluster set or template pre-populated and cloned (pre-pended to the beginning of the cluster set name).
Edit Displays the Update Cluster Set window for the selected cluster set or template.
Delete Deletes the selected cluster sets.
  • Validation exists to prevent you from deleting cluster sets that are currently in use by assortment plans.
  • Cluster sets are not truly deleted until the garbage collection procedure runs on the database; the frequency and timing of this procedure are determined during software implementation.

Deleting cluster set templates also deletes all cluster sets that were created using the template.
Allow Multiple Toggles showing/hiding check boxes in the grid. When enabled, you can select multiple cluster sets and/or cluster set partitions in the grid and then click Delete, Snapshot, or Calculate to perform the action on all of the selected cluster sets.

Notes:

  • Selecting a cluster set partition automatically selects all of its cluster sets.
  • Multi-cluster set operations can be submitted as tasks to the workload queue. You receive a notification when the operations are finished.
Snapshot Creates snapshots of selected cluster sets.
Calculate Calculates the cluster assignments for all stores included in a selected cluster set, based on the parameters and criteria specified during cluster set configuration.
Report Displays the Cluster Set Site Comparison Report.
View Collection Set Configuration Displays the View Collection Set Configuration window.
Refresh Refreshes the list of cluster sets displayed in the grid below.

Grid Fields

Field Description
Name The name of the cluster set.
Description

Indicates that there is a description for this cluster set. display the description.

Indicates that there is not a description for this cluster set.
Hierarchy The site hierarchy that the cluster set is based on. Is blank if a site filter was used instead of a site hierarchy.
Level The specific level of the site hierarchy that the cluster set is based on. Is blank if a site filter was used instead of a site hierarchy level.
Filter

The site filter that the cluster set is based on. Is blank if a site hierarchy was used instead of a site filter.
Last Modified The date that the cluster set was last changed.
Last Computed The date that the cluster set was last calculated.

Allocation

Identifies if the cluster set is configured for use with the Allocation solution and allocation scenarios in the Rate of Sale solution.

Assortment

Identifies if the cluster set is configured for use with the Assortment Planning solution and assortment planning scenarios in the Rate of Sale module.

Events

Identifies if the cluster set is configured for use with the Events solution.

Promotions

Identifies if the cluster set is configured for use with the Promotions solution.

Price

Identifies if the cluster set is configured for use with the Price Manager solution.

For more information, see Cluster Set Configuration Screen

Snapshot Pane

Select a cluster set in the grid to open the Snapshot pane.

Action Description
New Creates a new snapshot of the selected cluster set.
Edit Edits the snapshot selected in the snapshot grid.
Delete Deletes the snapshot selected in the snapshot grid.
Report Launches the Cluster Set Snapshot Comparison Report.
Refresh Refreshes the data in the snapshot grid.

The snapshot grid displays the following read-only snapshot details.

Column Description
Cluster Set Name The name of the cluster set for which the snapshot was taken.
Cluster Set Description Indicates that there is a description for this snapshot. Click the cell to display the description.

Indicates that there is no description for this snapshot.
Snapshot Date Time The date that the snapshot was taken.
Snapshot Note Indicates that there is a note for this snapshot. Click the cell to display the note.

Indicates that there is not a note for this snapshot.
Classification columns For each classification column, the count of stores n each classification at the time the snapshot was taken.
Effective Date Time

The date that the cluster set snapshot became or will become effective.

Creating a new cluster set snapshot initiates recalculation of all assortment plans in which the snapshot cluster set is active. This recalculation occurs as a part of a regularly scheduled overnight processing; the frequency of which is determined during software implementation.
