Module: Financial Planning

Display Name System Name Description
Approved Snapshot Retention Days FinancialPlan.ApprovedSnapshotRetentionDays How many days approved snapshots are kept in the system. Default = 720.
DC Stock Splits future number of periods FinancialPlan.Hybrid.DCSplitCalculation.Future.Periods Number of periods forward from the current period to determine % DC stock splits by channel. Default = 8.
Deleted Plan Retention Days FinancialPlan.DeletedPlanRetentionDays

How many days plans are kept in the system after being deleted (closed). Default = 0.
Enable VAT in MFP FinancialPlan.PlanningVatRate.Enabled

Whether or not to use VAT. This should only be set once during implementation. Default = False.

  • True = Use VAT
  • False = Do not use VAT
Financial Plan Master Plan Max Future Days FinancialPlan.Masterplan.DaysForward Number of days forward to populate the master plan where periods exist. Start date being the start of the current period. Default = 720.
Include DC Stock Split into EOP / BOP FinancialPlan.Hybrid.IncludeDCEOPInPlan Whether to include DC stock, which is split into channels during actualization, into the rolling EOP / BOP calculations.
Inventory is Shared FinancialPlan.InventoryIsShared

Whether or not DC inventory is shared among all channels. (Are receipts planned for specific channels or not?) Default = False.

  • True = Share DC inventory across channels.
  • False = Do not share DC inventory across channels.
MFP Plan Size Threshold FinancialPlan.PlanSizeThreshold Set an arbitrary size threshold for comparison to a PlanID's number of Product/Channel/Period combinations. Exceeding the threshold will result in running the process in the WorkManager Queue.
Multi Plan Seeding Merging FinancialPlan.MultiPlanSeedingMerging

Whether or not to allow seeding or merging from other plans. Default = False.

  • True = Allow seeding and merging from other plans.
  • False = Do not allow seeding and merging from other plans.
Open Multiple Plans FinancialPlan.OpenMultiplePlans.Enabled This policy enables or disables the ability to open multiple plans. Default(True)
Planning VAT Rate Product Hierarchy ID FinancialPlan.PlanningVatRate.ProductHierarchyID Product Hierarchy ID to use for planning VAT rates.
Run module in Debug mode FinancialPlan.DebugMode

Whether or not to run Financial Planning module procedures in debug mode.

  • 0 = do not run in debug mode.
  • 1 = print records to application log.
  • 2 = return immediate results sets. This mode must be run through development tools; it will fail if run through the UI.
Save the LLY data for hybrid financial plans FinancialPlan.SaveLLYData

For hybrid financial plans, whether or not to save LLY data is saved on initialize and during maintenance.

  • True = LLY data is saved on initialize and during maintenance.
  • False = LLY data is not saved.
Snapshot ViewCollectionSetID FinancialPlan.Snapshot.ViewCollectionSetID Thee ViewCollectionSetID for the snapshots view collection set.
SqlServerPivotGrid Recommended Maximum number of SQL Data Rows FinancialPlan.Hybrid.Maximum.Recommended.Max.SQL.DataRows Recommended maximum number of rows to return for any SQL Server Pivot Grid. If exceeded, a warning message is displayed in the UI. Default=2250.
Use Hybrid Markdowns and COGS in Retail Accounting Method FinancialPlan.UseHybridRetailMarkdownsCOGS Enable Hybrid Markdowns In Retail Accounting. Permananent, Promo and Hybrid Markdowns and COGS fields will be displayed.
Use Supplier Receipts Only for Historic Receipts FinancialPlan.UseSupplierReceiptsOnly Use tbl_SupplierReceipts solely for historic receipts in MFP and not a combination of tbl_SupplierReceipts, tbl_SupplierOrderHeader and tbl_SupplierOrderLine.
