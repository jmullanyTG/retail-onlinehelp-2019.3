User Details

Within the user information window, the user details tab will contain most of the general information common to software user accounts, such as login credentials and password information, and assigned filters.

The concept of assigned filters will give an administrative user the ability to segment sections of the company data to restrict users, or streamline their individual workflows. Users may be assigned filters built on specific inventories, sites, item masters and many other filter types.

  • User Name – The user name as it appears in the main users window. This user name will also be the login credentials for users to gain access to Demand Management.
  • User Description – A description of the user account. If it is a cloned user, the description will read as “Cloned from _____” until it is changed (the blank is filled by the user account from which it was cloned).
  • Full Name – The actual name of the user. This is an optional field.
  • User Domain Name – If the user is a domain account and has been imported from active directory, the associated domain will appear here.
  • Cube Role Name – If there is a SQL Server Analysis Services (SSAS) OLAP cube deployed on the database, Demand Management will need to pass a cube role name to the OLAP cube to access information via the OLAP pivot grid widget. This role name will be stored by user.
  • Password and Confirm Password – The password fields will appear blank. Administrative users may change passwords for individual user accounts here.
  • Account is Disabled – A disabled account cannot log into the system, but it is still visible.
  • Inventory Assigned Filter – The default inventory level filter assigned to the specified user.
  • Site Assigned Filter - The default site filter assigned to the specified user.
  • Promotion Assigned Filter – The default promotion filter assigned to the specified user.
  • Event Assigned Filter – The default event filter assigned to the specified user.
  • Item Master Assigned Filter – The default item master level filter assigned to the specified user.
  • Supplier Assigned Filter – The default supplier filter assigned to the specified user.
  • Customer Assigned Filter – The default customer filter assigned to the specified user.
  • Customer Inventory Assigned Filter – The default filter for customer inventory (to support custom implementations of customer level forecasting only).
  • Item Master Group Assigned Filter – A default filter for item master groups, a sub-set of item master filters which can be used to further refine item master filtering within content manager.
  • Variant Assigned Filter – A default variant filter assigned to the specified user.
  • Variant Group Assigned Filter – A default filter for variant groups, a sub-set of variant filters which can be used to further refine variant filtering within content manager.
