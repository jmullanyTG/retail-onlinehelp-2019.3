Workload Queue Global Policies

Global Policies are configured by Professional Services and must not be modified without assistance from Demand Management Professional Services or Support.

There are three types of system policies: General, Expiration, and Retention.

General Policies

Policy Name Description

Default Batch Size

The default work size, if not specified or defined for the process type (Assortment Planning, Inventory Forecast, etc.)

Default Queue Marshall Delay Time

The number of seconds the system will wait before checking for new batches to prioritize and place on the queue if the last check did not find anything.

Maximum Server Work Threshold

The maximum work threshold (estimated work per item * estimate item count) the server can handle across all process types before downgrading batches marked as Synchronous to Immediate.

Ensure this setting does not conflict with defined thresholds.

Maximum Synchronous Threshold

The maximum work threshold (estimated work per item * estimate item count) the server will use when allowed for a process type to apply Synchronous priority to a batch.

When process types that are submitted in batches and are above the Maximum Synchronous threshold is prioritized as the configured Priority threshold settings. 

The Synchronous threshold works in conjunction with Maximum Server threshold across all process types.

Expiration Policies

Policy Name Description

Default Expiration Time

Daily queued tasks expire when they are moved from the active queue to archive without being processed.

  • Priority: Immediate, ASAP, Low Priority

Default Nightly Expiration Time

Nightly queued tasks expire when they are moved from the active queue to history list without being processed.

  • Priority: Nightly

Default Weekend Expiration Time

Weekly queued tasks expire when they are moved from the active queue to history list without being processed.

  • Priority: Weekend

Retention Policies

Policy Name Description

Default Queue Log Retained Days

The number of days the data in the Queue_Log table is stored after the Batch has been completed. Statuses are: Expired, Failed, or Removed.

Default Queued Archive Retained Days

The number of days the data in the Queued_Archive table is stored after the Batch has been completed. Statuses are: Expired, Failed, or Removed.

Default Queued Items Retained Days

The number of days the data in the Queued_Items table is stored after the Batch has been completed. Statuses are: Expired, Failed, or Removed.

To Configure System Policies

  1. From the Maintenance solution, click Policies. The menu of available system policies displays.
  2. Click the arrow next to the respective system policy name. The system policy details display.
  3. Change the necessary information.
  4. When finished, click Apply Changes.
