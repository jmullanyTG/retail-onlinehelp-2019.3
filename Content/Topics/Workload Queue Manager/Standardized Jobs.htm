Standardized Jobs

Standardized jobs, found in the Job Manager solution, are used to organize user and system jobs into daily, nightly, and weekend batches, according to the estimated workload.

From the Standardized Jobs solution, you can also clear historical logs that are no longer relevant (cleanup)and determine a process priority (optimization tuning).

To View Standardized Jobs

  1. From the Maintenance tab, click Job Manager. The Navigation Menu displays.
  2. From the Navigation Menu, click the job you wish to view. The job details display on the right panel.

Daily Job

Daily batches with low workload run during daytime business hours. For example, Monday through Friday, from 7a.m. to 6p.m.

Nightly Job

Batches with medium workload run overnight after business hours. For example, Monday through Friday, from 6p.m. to 11p.m.

Weekend Job

Batches with heavy workload run during the weekend. For example, Saturday, from 3a.m. to 11p.m.

Job Task Configuration

For convenience and to take advantage of parallel processing capabilities, you can configure daily, nightly, and weekly standard job tasks to include all (or specific) process types (forecasts, price actions, etc.) .

To Set Up a Job Task

  1. From the Maintenance tab, click Job Manager. The Navigation Menu displays.
  2. From the Navigation Menu, click the job you wish to view. The job details display on the right panel.
  3. Make the necessary changes according to the following table.
  4. When finished, click Save.

Method Parameter Name Default Value Description
@bDebug False Required. Whether or not the system should provide additional logging information.
@bEnforceWorkLimit True

Required. Whether or not the system adheres to the Maximum Server Work Threshold system policy.

  • If set to True and the amount of work meets or exceeds the threshold set in the policy, any un-processed job tasks will go into delay cycles until the amount of work returns below the threshold value.
@bNightly False Required. Whether or not the job task processes only nightly jobs.
@ParallelBatching False

Required. Applicable only when multiple job tasks are configured for a processing period (daily, nightly, or weekend).

  • If set to True, all job tasks work on the same job until it is completed. Then, the job tasks move on to the next job in the queue.
  • If set to False,each job task works on distinct jobs.
  • All job tasks must have the same value (True or False).
@bWeekly False Required. Whether or not the job task processes only weekend jobs.
@iProcessTypeID  

Optional. Specifies the type of jobs that this job task will run. Leave this value blank to run all job types; otherwise, enter one of the following values:

  • 1 - Inventory Forecast
  • 2 - Level Forecast
  • 3 - Group Forecast
  • 4 - Replenish Inventory
  • 5 - Assortment Receipt Plan
  • 6 - Adaptive Allocations
  • 7 - Assortment Prepack
  • 8 - Allocations
  • 9 - Replenish Set
  • 10 - Price Manager
@iUserID 1

Optional. The integer ID of the user that the job task will run as if the job process is submitted without an identified user. This user must have all necessary access rights to complete the work for the @iProcessTypeID value.

Default value is the IOAdmin ID.

@nMarshalID   Required. Unique ID value for this job task. To ensure accurate processing and logging, this value must be unique across all job tasks. Include this ID value in the job name. For example, "Workload Queue Parallel - Nightly - 1".
