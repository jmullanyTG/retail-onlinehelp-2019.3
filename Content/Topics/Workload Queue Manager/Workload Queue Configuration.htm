Workload Queue Configuration

Only Demand Management Professional Services and Support must change the workload queue configuration to ensure the integrity of corporate strategies and guiding policies that enforce standards before exported data is sent to downstream systems.

The following solutions are integrated with Workload Queue:

  • Allocations
  • Assortment (Receipt) Plan
  • Assortment (with Order Management) Prepack
  • Inventory Forecast
  • Inventory Replenishment (with Order Manager)
  • Group Forecast
  • Price Management

While Workload Queue comes with an out-of-the-box configuration, some tuning may be required depending on the available hardware resources.

Workload Queue Configuration Window

The Workload Queue Configuration window makes it easier to fine-tune workload thresholds for supported process types.

For each supported process type (Allocations, Assortment Prepack, etc.), you can view the default and customized threshold ranges for each priority level (Immediate, ASAP, Low Priority, Night, or Weekend).

To Access the Workload Queue Configuration Window

  1. From the Maintenance tab, click Workload Queue. The Workload Queue screen displays.
  2. Click the Configuration icon on the right side at the top of the screen. The Workload Queue Configuration window displays.

Workload Queue Configuration window columns:

Column Description

Priority

The supported process types, in alphabetical order, and the set of priority levels for that process type. These levels are listed from most to least urgent: Immediate, ASAP, Low Priority, Night, and Weekend.

Current Threshold

The default minimum and maximum value range for a priority level. These values do not change.

Override Min

The customizable minimum value for a priority level. This field is blank if the default thresholds are used.

Override Max

The customizable maximum value for a priority level. This field is blank if the default thresholds are used.

Reset

Click to delete the customized values in the Override Min and Override Max columns and return to the default values displayed in the Current Threshold column.
