Workload Queue History List

The Workload Queue History List displays completed Batches, whether the system was able to successfully complete, partially complete, failure, expiration or if the Batch was removed from processing. The system will provide the following information and may be resorted by you:

Field Description

Process Type

Process Types represent the supported Demand Management solutions with the ability to use the Workload Queue. Batches will be grouped by Process Types and will only be displayed if there are ‘active’ Batches of that type.

Status Icon

The system will display the appropriate icon for the final status of the Batch.

Archive Date

The date time stamp when the Batch was moved from the ‘active’ to the ‘history’ list.

Description

Name used to identify different Batches.

  • Inventory Forecast – The system will use the Item Filter Name and product filter used.
  • Assortment Planning – The system will use the Scenario Name. If a Scenario Name is not available, the Plan Name will be displayed with the product filter used.

Item Status

The number of Items completed out of the total number of Items to be processed in the Batch.

Batch Priority

The final priority assigned at the time of the Batch’s execution.

Batch Owner

The name of the user who submitted the work to the queue.

Enqueue Date

The date time stamp the Batch was submitted to the Workload Queue.

Last Changed Date

The date time stamp of the last Priority Change.

Last Changed By

The name of the user who last modified the Batch’s Priority.
