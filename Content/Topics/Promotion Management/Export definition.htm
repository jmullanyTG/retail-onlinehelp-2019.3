Export definition

To create an Export Configuration, first click New above the Export Definition grid, fill in the required columns and click Save.

Field Description

Export Type (Required)

Select an Export Type from the following drop down list:

API – For this export type, the Module ID, Export Name and Method Called columns are required.

File – For this export type, the Module ID, Export Name, Base Output File Name, Output Path/Table, and Output Method columns are required.

Module ID (Required)

Defaulted to ‘Promotions’.

Export Name (Required)

Enter the descriptive label for the export process.

Method Called (Required/Optional)

Select ‘Export Promotions FileOutput’ from the drop down list unless a custom method has been created. This column is required for File export types.

Base Output File Name (Required/Optional)

Enter the base file name for File export types. A timestamp will be added to this name.

Output Path/Table (Required/Optional)

Enter the path that the export file will be written to for File export types. Do not include closing ‘\’.

Output Method (Required/Optional)

Select either ‘Dispatch Broker’ or ‘xp_cmdShell’ from the drop down list for File export types. The former is the preferred method.

Active (Optional)

Select this column to indicate that the export is active.

Column Delimiter (Optional)

Enter the single character column delimiter to be included in the output. Ensure that the column delimiter is a unique character that is not used elsewhere e.g. in field names or values.

Max Detail Record Batch Size (Optional)

Enter the maximum number of detail records to output per batch for API export.

Output User (Required/Optional)

Select a SQL Server login that has been configured for exporting from the drop down list.
