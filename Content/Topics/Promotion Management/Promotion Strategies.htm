Promotion Strategy

The Promotion Strategy tab lets users assign Calculation Sets, Validation Sets, and Pricing Details to a Promotion Strategy. User can also setup mandatory and required fields that are seen in the Promotion Pricing grid as well as maintain whether Promotion Strategies are “active” (i.e. available for use). The Promotion Strategy Field Formats grid applies to all Promotion Strategies and lists all available price fields, including their format, when defining a Promotion Strategy.

Field Descriptions

Field Description

Strategy Name (Required)

Enter a name for Promotion Strategy that will be used to select a Promotion Strategy when creating a Promotion.

Strategy Type ID (Required)

Select the Strategy Type from the drop down list that will determine how the Promotion Strategy will be displayed.

  • Promoted Price - A promotional price.
  • Promoted Discount – An amount off or percent off Promotion.
  • Buy Get – A buy X get X or buy X get Y Promotion.
  • Spend Get – A spend X get points, dollars, or percent off.
  • Rebate – A regular retail or promotional price with a rebate.

Method (Required)

The drop down is populated by the available methods (i.e. advanced processes applied at the database level) configured for use.  Select a method the system will apply for the Promotion Strategy. This will be run before anything else on the Promotion.

Calculation Set (Required)

Select a Calculation Set from the drop down list. Populated with Calculation Sets defined in the Promotion Validation Configuration grid. Calculations will be performed after methods.

Validation Set (Optional)

Select a Validation Set from the drop down list. Populated with Validation Sets defined in the Promotion Validation Configuration grid. Validations will be performed after Calculations.

Price Output Set (Required)

Select a Promotion Pricing Detail from the drop down list. Populated with Pricing Outputs defined in the Promotion Price Output Set Configuration grid.

Default Price Method (Optional)

The drop down is populated by the available methods (i.e. advanced processes applied at the database level) configured for use.  Select a method the system will apply for the Promotion Strategy. This will be run second, after the Method is run on the Promotion.

Analysis View Collection Set (Optional)

The drop down is populated by available pivot template to display analysis information. Select a display to use in the Promotion – Analysis tab.

Price View Collection Set (Required)

The drop down is populated by available pivot template to display pricing information. Select a display to use in the Promotion – Price Details tab.

Is Group Synced (Optional)

Must be selected for all BOGO AA type Promotion Strategies.

Active (Optional)

Selected by default to indicate that the Promotion Strategy will be available when working with Promotions.
Edit User Access

By default, all Roles and Users have access to all Promotion Strategies. Clicking on this field opens the Remove User Rights dialog in which you can remove access to a Promotion Strategy from particular Roles and Users.

The list is filtered to only display Users that have User Right Allow Creation of Promotion.
Delete Deactivates a Promotion Strategy but does not delete it from the system. Click Save to save changes.
