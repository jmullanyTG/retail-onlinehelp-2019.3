Field requirements

In the Promotion Strategies Field Requirements grid, select the requirements for the fields relevant to the new Promotion Strategy and click Save.

Field Description

Field Name (Read Only)

Names of all available pricing fields.

Is Buy (Optional)

Select if you wish to indicate that the field will be used a Buy field. Use the guide below for each Promotion Strategy type:

  • Promoted Price – select Is Buy for pricing and limit fields.
  • Promoted Discount – select Is Buy for pricing and limit fields.
  • Buy Get – for BOGO AA, select Is Buy for pricing and limit fields and for BOGO AB, select Is Buy for all buy pricing and limit fields.
  • Spend Get – select Is Buy for pricing and limit fields.
  • Rebate – select Is Buy for pricing and limit fields.

Is Get (Optional)

Select if you wish to indicate that the field will be used a Get field. Use the guide below for each Promotion Strategy type:

  • Promoted Price – select Is Get for pricing and limit fields.
  • Promoted Discount – select Is Get for pricing and limit fields.
  • Buy Get – for BOGO AA, select Is Get for pricing and limit fields and for BOGO AB, select Is Get for all get pricing and limit fields.
  • Spend Get – select Is Get for pricing and limit fields.
  • Rebate – select Is Get for pricing and limit fields.

Is Visible (Optional)

Select if you wish to display this field in the display grid on the left of a New Promotion Wizard and Promotion.

 

 

Is Mandatory (Optional)

Select if you wish to indicate that the field must have a value before the Promotion can be saved.

Is Custom (Read Only)

Indicates if the field is customer specific.
