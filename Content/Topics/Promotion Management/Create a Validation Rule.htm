Create a Validation Rule

Within the Promotion Validations tab, click New above the Validation Rules grid in the bottom half of the screen to create a new Validation Rule. Enter the required fields and click Save.

Action Description

Name (Required)

Enter a descriptive label to summarize the purpose of the Validation. This will be used to identify the Validation when creating Validation Sets.

Description (Optional)

Enter a summary of the Validation.

Validation Type (Required)

Identify when the Validation will be run in relation to the Calculation. Select the Post option from the drop down menu.

  • Pre - To be developed as future functionality.
  • Post - This should be selected in all cases and will run the Validation after price and cost retrieval and Calculations have been run.
  • Both - To be developed as future functionality.

Validation Level (Required)

Select a level from the populated drop down. All levels act on the Site Cluster level.

  • Promotion Level – To be developed as future functionality. For now, selecting this option means that Calculations will be performed at the Promotion Group level.
  • Promotion Aggregate Level – Calculations will be performed at the Promotion Group level.
  • Group Level – Calculations will be performed at the Product Group level.
  • Product Level
  • Item Level – Calculations will be performed at the Item Master Level.
  • Variant Level – Calculations will be performed at the Variant Level.

The Promotion Manager system defines Product Levels as follows:

  • Inventory – A product at a particular Site so it has a single price.
  • Item Master – A product across all Sites that may have different prices at different Sites. An Item Master may have a max price for example.
  • Variant – A variant of Item Masters based on a shared attribute i.e. size, color, style, etc. An example of a Variant is Men’s Polo Shirts.

Apply to Group Type (Required)

From the drop down options, select one to identify which Group Association Type this Validation will be run on:

  • Buy Group - The Validation will only be run on the Buy Group.
  • Get Group - The Validation will only be run on the Get Group.
  • Buy Get Group - The Validation will only be run on both the Buy and Get Groups. This option should also be selected for all Promotions that are not of the Buy Get type.

Method (Optional)

The drop down is populated by the available methods (i.e. advanced processes applied at the database level) configured for use.  Select a method the system will apply for the Validation. This will supersede any Formula entered.

Formula (Required/Optional)

Enter a mathematical expression for the system to evaluate. Configured variables can be found in the right panel. This formula will be used if the Method column is blank.

Severity (Required)

Select a severity from the populated drop down:

  • Information - After displaying the Error Message, you will be allowed to continue.
  • Warning - After displaying the Error Message, you will be allowed to continue.
  • Error - After displaying the Error Message, you will not be allowed to continue. Corrections must be made.

Error Message (Required)

Enter a message that will be displayed if the Validation fails.

Deleted (Optional)

See Delete a Validation Rule.
