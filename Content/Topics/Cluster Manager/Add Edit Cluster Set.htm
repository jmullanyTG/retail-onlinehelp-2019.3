Add/Edit Cluster Set

All of the configuration for building and maintaining cluster sets.

  • Name – The name used to identify the cluster set.
  • Description – A description of the cluster set.
  • Group – A folder style file structure for organizing cluster sets. Cluster sets groups may be maintained using the “change” button.
  • Security – Will allow specific users or roles to view or maintain the cluster set.
    • Public – All users with access to the cluster manager may view and maintain this cluster set.
    • Private – Only the user responsible for creating the set may view and maintain it.
    • Shared – The cluster set may be shared with selected users. This is done by including/excluding individual users or roles by clicking the “change” button.
  • Cluster Type – This will determine the clustering method used to build the cluster sets.
    • K-Means – K-Means clustering will group the data points into logical clusters based on the nearest mean. The centroid will be a point in the center of the cluster, regardless of where the data points are.
    • K-Medoid – K-Medoid clustering will group the data points into logical clusters while restricting the centroid to a data point. K-Medoid will still use the minimum mean distance to group the logical clusters. This is a more computationally intensive process than K-Means.
    • K-Means Collection – A template used to create similar K-Means sets in which a single parameter (partition filter by value) will change and therefore create new K-Means cluster sets.
    • K-Medoid Collection – A template used to create similar K-Medoid sets in which a single parameter (partition filter by value) will change and therefore create new K-Medoid cluster sets.
  • Entity Type – The entity type will determine the granularity of data which will be clustered. Setting the entity type at a higher level will exclude lower level data.
    • Inventory – An item/site combination.
    • Item Master – An item across all sites.
    • Site – A physical or virtual location capable of holding inventory records.
    • Customer – The party at the receiving end of the sales transactions as supplied to Demand Management.
  • Filter by Type – Will show the available filter types contained in, or parented by the entity type. Sets the level of filters which will appear in the filter by filter field.
  • Filter by Filter – Will allow users to restrict the cluster set data to a specific filter (most commonly an inventory filter). This may be thought of as a cross-section of the data with an intersecting filter.
  • Group By – The group by will represent the actual data points displayed and grouped in the cluster. The available options in the group by will be based on the supplied entity type (e.g. site groups will be displayed if “site” is selected as the entity type).
  • Partition Type – The partition type will determine the mechanism by which cluster sets are segregated.
    • Filtering – Will use the familiar Demand Management filtering.
    • Filtering Lite – Will limit the filtering to only a single criterion, and only the “equals” clause (i.e. the filter must be for distribution center site).
  • Partition Filter by Type – Will determine the type/level of filter used for partitioning.
  • Partition Filter by Filter (if using traditional filtering) – Will allow users to determine which filter is used to segregate the cluster set. Choosing the “change” button will bring up the available filter list from the specified filter type.
  • Partition Filter by Field (if using filtering lite) – Will allow users to choose the specific field which must be equal to the partition filter by value supplied. Will be restricted to the available fields specified in the partition filter by type.
  • Partition Filter by Value (if using filtering lite) – The value which must equal the data to be included in the cluster set.
  • Classification Items – Classification items will determine the actual data to be classified by the clustering engine.
  • Classification Priority Order By – The way in which the classifications are ranked.
    • Ascending – Will rank each classification in ascending order using consecutive alphabetical letters to designate each cluster.
    • Descending – Will rank each classification in descending order using consecutive alphabetical letters to designate each cluster (suggested use).
  • Classification Settings – Classification settings will list the selected classification items and allow users to alter their priority. It is important to remember the priority order setting when determining the order.
  • Attribute Items – Attribute items will be included in the classification engine as inputs.
  • Attribute Priority Order By – The way in which the attributes are ranked.
    • Ascending – Will rank each classification in ascending order using consecutive alphabetical letters to designate each cluster.
    • Descending – Will rank each classification in descending order using consecutive alphabetical letters to designate each cluster (suggested use).
  • Attribute Settings – The attribute settings section will allow users to edit the priority, determine the aggregation function, and assign a weight to the attribute if desired.
    • Attribute – The attribute name.
    • Entity Type – Will show the associated entity type for the attribute (Site, Item Master, etc.)
    • Priority – The ranked priority will determine how the clusters are built. Is affected by the attribute priority order by.
    • Order By – An override for the attribute order by which may be changed for a single attribute.
    • Aggregate Function – Will determine the mathematical means by which the attribute values are aggregated.
      • Average
      • Count
      • Maximum
      • Minimum
      • Summation
      • Statistical Standard Deviation
      • Statistical Standard Deviation for the Population
      • Statistical Variance
      • Statistical Variance for the Population
    • Weight – A weight value for the attribute. Will be considered against all the other listed weight values.
  • Distance Function – Distance function will determine how the mean distances are calculated.
    • Euclidean Distance – The distance as drawn on the Euclidean plane (straight lines).
    • Manhattan Distance – The distance as drawn on a plane while restricted to 90deg right and left turns (often referred to as taxi cab distance).
  • Allow Cluster Count to be Resized – A check box which will allow for the clustering engine to resize the overall number of clusters.
  • Compute Optimal Cluster Count – A check box which will allow the clustering engine to compute and optimize the number of logical clusters.
  • Maximum Allowed Cluster Count – A practical cap on the number of clusters.
  • Number of Iterations – For use with K-Medoid clustering, this will cap the maximum number of computational iterations with which the clustering engine has to process exact fits for the centroids. Infinite if left NULL.
