Product Sets

The Product Set process defines the collection of variants to be allocated. They may be similar to the products used to model the demand or choice count models, or altogether unrelated.

The Product Set screen contains a listing of product sets and their associated details which will populate once they are created.

Product Set section

  • Product Set Name – The unique name for the product set. Will be used to look-up product sets within the scenario plan configuration screen.
  • Product Set Description – Optional description for the product set.
  • Variant Selection – A filter of variants which have been selected for potential inclusion in the product set.
  • Included Attribute Families – The attribute families which should be considered when assessing the product set.
  • Opt-In – Default behavior for variant selection on creation and update of the product set.
  • Attribute Weighting Time Frame – Time frame used to define the attribute weighting period.
  • Attribute Weighting Supplementary Filter – Provides users with an opportunity to specify the sample set of products to use when determining attribute weighting.
  • Access Level – Used to define user access to the product set. This will control its visibility in this screen as well as restrict its selection for use within a scenario plan.
    • Public – Available to all users within the application.
    • Private – Available only to the creator.
    • Shared – Available to only specific users.
  • User Select – If set to a shared access level, the list of shared users.

Variant Details section

The Variant Details section allows users to determine the sub-set of variants within the filter which should be included in the product set. Upon creation or alteration of a product set, the variant details views will refresh based on the supplied parameters.

If Opt-In is selected, then on any change to the product set, the list of variants within the variant details view will all be selected (even if they were previously un-selected).

  • Selected – Will be checked if the variant is selected to be included in the product set. The default behavior of the “selected” box is determined by the “opt-in” box in the product set configuration grid. If an update is made to the product set while the “opt-in” box is checked, upon save, all variants will become selected.
  • Variant Code – The unique code describing the variant.
  • Variant Description – The optional description for the variant.
  • Total On Hand Units – Total in-network on-hand for the variant. Includes on-order, pending, and supersession stock.
  • Total On Hand Value – The value, in base currency, of the total on-hand units.
  • Store On Hand Units – Total on-hand for stores.
  • Store On Hand Value - The value, in base currency, of the store on-hand units.
  • DC On Hand – Total on-hand for all DCs.
  • DC On Hand Values - The value, in base currency, of the DC on-hand units.
  • On Order Quantity Units – Total quantity on order.
  • On Order Quantity Value - The value, in base currency, of the on-order units.
  • Size Fulfilment – A percentage representation of ownership across the size range. While not representative of specific size quantities, it will provide a quick reference for variants with exceptionally broken size runs.

The details related to the highlighted variant:

  • Attribute – The attribute families included in the product selection set.
  • Attribute Value – The assigned value for the highlighted variant.

Displays on-hand and on-order quantities for the highlighted variant at all sites.

  • Site Code – The unique name of the site.
  • Site Description – The optional description for the site.
  • Total On Hand - Total on-hand for the variant at the site. Includes on-order, and supersession stock. It excludes pending stock.
  • On Order Quantity – Total on-order quantity at the site.

Attribute Details section

The Attribute Details section allows users to select which of the initially included attribute families should be considered as when determining allocation quantities.

Attribute families may be weighted against one another to provide more or less influence on their affinity within the draft process. All attribute families default to 100% as the default weighting factor (even weight).

  • Include in Weighting – If selected, the attribute family will be considered as part of the draft process.
  • Attribute Name – The name of the attribute family.
  • Attribute Description – The optional description for the attribute family.
  • Weighting Factor – A factor by which attributes will be weighted against one another. Defaults to 100% (even weighting for all included attributes).
