Scenario Plan Grid Reference

The Scenario Plan grid manages the configuration of scenario plans.

  • Plan Name – The unique name for the scenario plan.
  • Plan Description – The optional description for the scenario plan.
  • Demand Plan – A drop-down of the available demand plans. Must be populated to pass configuration status requirements.
  • Product Set – A drop-down of the available product sets. Must be populated to pass configuration status requirements.
  • Choice Count – A drop-down of the available choice count models. Must be populated to pass configuration status requirements.
  • Plan Length – The expected duration of the stock coverage from the resulting plan. Though not restricted to aligning with the selected demand plan time frame, it is considered best practice not to deviate from the already specified length of time.
  • Min Pick Ratio – Determines the minimum pick quantity for the draft engine based on a percentage of the ideal pick value (calculated as need target/choice count). This field is hidden by default and will default to system policy specified value (initializes to “50%”).
  • Max Pick Ratio – Determines the maximum pick quantity for the draft engine based on a percentage of the ideal pick value (calculated as need target/choice count). This field is hidden by default and will default to system policy specified value (initializes to “200%”).
  • Location Strategy – A key component to calculation of both choice count and allocation volumes, the location strategy will specify the stores which are eligible to receive stock as part of the allocation. The location strategy may be configured within Demand Management Allocations.
  • Configuration Status – Displays the status of the scenario plan. If configuration components are missing, they will be displayed in a comma separated list within this field. Missing configuration elements will prohibit scenario calculation.
  • Access Level – Used to define user access to the scenario plan.
    • Public – Available to all users within the application.
    • Private – Available only to the creator.
    • Shared – Available to only specific users.
  • User Select – If set to a shared access level, the list of shared users.
  • Plan Status – Defines the workflow options available for the scenario plan. Will updated based on changes made to the plan. Statuses include:
    • Requires Configuration – Configuration elements are missing. Refer to the configuration status field for additional information. No workflow options are available until configuration is complete.
    • Configuration Changed – Configuration elements have changed and a plan calculation is required. Any results currently displayed within the scenario details section are related to the previous configuration and are considered to be “stale”. Workflow options are identical to those of the “ready to calculate” status.
    • Ready to Calculate – Once configuration is complete, the plan status will update to “ready to calculate” and the draft engine may be run. Workflow options are identical to those of the “configuration changed” status.
    • Ready to Execute – Once calculation has been completed, the scenario plan results will be available to view within the scenario details section and the plan status will be changed to “ready to execute”. From this workflow status, users may elect to create orders if the results are satisfactory, or alter the configuration to produce different results, which will update the status to “ready to calculate”.
    • Scenario Finalized – Once orders have been created, the scenario will become finalized and exist in a read-only state to prevent order duplication.
