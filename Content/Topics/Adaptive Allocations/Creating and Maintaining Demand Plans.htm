Creating and Maintaining Demand Plans

Demand Plans are created and maintained within the Demand Plan screen within Adaptive Allocations. Within the detail configuration section, users are able to configure the demand source, and the relevant parameters associated with the source.

Fields

  • Demand Plan Name – The unique name for the demand plan. Will be used to look-up demand plans within the scenario plan configuration screen.
  • Demand Plan Description – Optional description for the demand plan.
  • Demand Source – Displayed value for the demand source selected.
  • Product Hierarchy – The system policy selected product hierarchy for use within the module. This product hierarchy should be configured to cover the product dimension from top to bottom. It should be consistent with hierarchies in use within any financial planning functions.
  • Product Hierarchy Selection – The sampled level within the product hierarchy.
  • Time Frame – Defines the desired periods of history (Rolling WOC only) to be used when creating the demand plan target values.
  • Access Level – Used to define user access to the demand plan. This will control its visibility in this screen as well as restrict its selection for use within a scenario plan.
    • Public – Available to all users within the application.
    • Private – Available only to the creator.
    • Shared – Available to only specific users.
  • User Select – If set to a shared access level, the list of shared users.

Demand Plan Results section

Once a demand plan has been associated to a scenario plan (and thus associating values within the site dimension via a location strategy) and calculating the results, users are able to view the by store need target values in the Demand Plan Results section.

Users may view different linked scenario plans by selecting them in the scenario plan drop-down menu.

  • Scenario Plan – A drop down list of all scenario plans associated with the highlighted demand plan.
  • Site Code – The unique code for the site.
  • Site Description – Description for the site.
  • Need Target – The total need target for the site based on the demand plan.
  • Site Groups 1-20 – Optional groupings used within the location dimension may be returned (e.g. channel, etc.) to further assist with need target validations. These values will appear based on names specified as part of the common table definition process during the solution design.

Demand plan results are calculated in a common “base currency” instead of unit targets to better represent coverage across a category with a diversity of AUR/AUC values.
