Product Model

Product models support the ability to create indexing of stores to support allocations. By selecting specific products for modeling, the index will be created not as in indication of overall store performance, but instead representative of how a product or set of products will perform across the stores.

Within the product modeling screen, users will create product models for use in the allocation strategy screen. These product models are built using historical data from groups or levels of similar products over a specified time period. 

There are two methods for model rule selection: fixed and set period rolling. A fixed period will supply a start and end date which creates the fixed period. The set and rolling period rule will specify a set distance in the past (365 days in the past from today) plus a defined rolling period (+90 days). This would create the period as the next 90 days from exactly one year ago.

Product model calculation

By default (system policy Allocate.ProductModel.UseAdvanced is set to False), the product model calculation uses the MEDIAN to produce the average values. If system policy Allocate.ProductModel.UseAdvanced is set to True, then the calculation will use the MEAN.

Maintaining product models

Users may create, edit, and delete product models.

Defining a product model

To define a product model:

  1. On the Product Model screen, click New.
  2. Enter a unique product model name in the Product Model field.
  3. Provide a description in the Product Model Description field.
  4. Click the Model Time Frame Selection field and choose a Time Frame.
  5. Select the appropriate field and choose the model metric and model type.
  6. Optionally, select the appropriate check box to include inner zeros and lost sales.
  7. Click on the Sales Type field to select sales types.
  8. Click on the Product Selection field to choose the Item Master filter that specifies the items to use during the calculation.
  9. Optionally, click on the Default Product Selection field to choose an Item Master Filter to be used if the first item master filter does not produce values.
  10. If you selected a location plan related model metric, then select a location plan in the Location Plan field.

  11. Click Save.

    The contributions are automatically calculated and display in the bottom detail grid.
