Location Strategy

Location strategies are groups of sites/stores which are used to ensure that the products are allocated to the correct locations.

Location strategies may be created and applied to allocation strategies to define the location dimension of both the inventory cache and the allocation orders. They may be saved and re-applied to multiple strategies by multiple users. Location strategies may be based on hierarchal relationships or attributes (when available).

Strategies may also be duplicated to reduce repetitive work by selecting the “Duplicate” box and clicking “Apply”. All changes require users to click “Apply” before taking affect.

The location strategy grid will list the location strategies.

Defining a location strategy

To define a location strategy:

  1. Click New.
  2. Provide a unique name in the Location Strategy Name field.
  3. Click on the Site Filter field and select a filter in the Items window.
  4. Select Save to populate the bottom grid.

  5. In the bottom grid, select the check box in the Site Eligible column for each site that is to be part of the location strategy.

    NOTE: Only selected sites are considered part of the strategy.

  6. Click the Save button in the bottom grid.

Location strategy grid fields

  • Location Strategy Name – User supplied name for the location strategy.
  • Location Strategy Description – An optional description for the location strategy.
  • Site Filter – List of sites/stores which will be included in the location strategy, driven by site filters. These are the sites to which inventory will be distributed; it is not intended that the distribution centers be included in this list. Location strategies are populated based on the filter contents at the time of the strategy creation and do not automatically update based on filter content changes.
  • Access Level – Access levels determine which users may use and/or edit the location strategy.
  • User Select – Displays the specific users for a “shared” access level location strategy.
  • Duplicate – If checked, upon application, the strategy will be duplicated. This allows users to easily build similar groups of stores.
