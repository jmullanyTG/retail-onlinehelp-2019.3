Allocations Overview

The Allocations module presents users with a clear and defined workflow to produce execution orders for the transfer of merchandise throughout their supply chain at a level above Item Master (typically defined as “variant”).

Components of an Allocation

There are a few building blocks for Allocations to determine the demand or need of a store. Product Model, Size Curve, Location Strategy, and Sales Curve are the main components to help calculate the demand for a store.

Product Model

The Product Model provides the contribution values to help determine the percent of contribution each Site in the Location Strategy should receive of the total demand. For more information, see Product Model.

Size Curve

The Size Curve provides the value to further break down the store demand from product model by size. There can only be one size type defined per Size Curve. For more information, see Size Curves.

Location Strategy

Location Strategy provides the sites that will always be considered when calculating demand.

An override or exception that makes a site Inactive will NOT influence the other sites’ demands. You have to remove it completely from the Location Strategy. Phase In/Out overrides can influence demand. For more information, see Location Strategy.

Demand Type and Phase In/Out

Demand Type and Phase In/Out are also components but are part of the Allocation Strategy.

Demand Type

Demand Type determines how the system calculates total demand which is always calculated from Phase In to Phase Out even if the Phase In Date has passed. For more information, see Allocation Strategy grid field reference.

Sales Curve

Once the size demand is determined for a site the Sales Curve provides the information to see how to spread the size demand by bucket.
