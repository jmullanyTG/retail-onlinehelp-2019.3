Stock Availability Restrictions tab

Stock availability restrictions allow users full control of the stock which is considered to be “available” for allocation. If stock on hand within a specific DC or on order within a specific PO should not be considered as eligible, users may unselect it; all stock in all DCs (as defined by supply chain rules) and all incoming POs (as defined by supply chain rules) are considered to be eligible for allocation by default.

Allocate Strategy Stock Availability Restrictions (DC)

Information related to the strategy stock is displayed in the allocate strategy stock availability restrictions (DC) view and should be used as a driver in the decision to include or exclude the stock from the available pool.

  • Selected – If checked, the stock on hand is considered to be “available” and will be included in the available stock calculations.
  • Site Code – The unique code identifying the site. All listed sites will be of “DC” type.
  • Site Description – The description of the site.
  • Priority – The global site priority of the DC. This is a read-only field and must be managed within Site Manager. Does not affect any calculations.
  • Priority Group – If in use, the associated priority group of the DC. This is a read-only field and must be managed within Site Manager. Does not affect any calculations.
  • Available Push Quantity – Stock on hand taking into account holdback quantity and/or percentage. Will display as “0” if the demand type is set to any value other than “total push”.
  • Stock on Hand – The stock on hand in the DC, inclusive of stock reserved. Reflective of actual stock on hand, and any orders within the available and late order windows.

Allocate Strategy Stock Availability Restrictions (PO + Breakdown)

Information related to the strategy stock is displayed in the allocate strategy stock availability restrictions (PO + Breakdown) view and should be used as a driver in the decision to include or exclude the stock from the available pool. In order to be considered as actionable stock, the arrival date of the purchase order must fall within the window defined by the late orders window (days back) and available orders window (days forward) settings. These values may be set within System Policy.

PO View

Displays the selectable purchase orders.

  • Selected - If checked, the on order stock is considered to be “available” and will be included in the available stock calculations.
  • PO # - The purchase order number. Part of the supplier order header data.
  • Destination DCs – The inbound destination site or sites of the purchase order. If more than one destination is specified, the sites will be displayed in a comma separated list.
  • PO Qty – The outstanding on order quantity (in case of partially delivered purchase orders).
  • PO Date – The date on which the purchase order was placed.
  • Receive No Earlier – The receive no earlier date specified in the supplier order information.
  • Receive No Later – The receive no later date specified in the supplier order information.
  • Contains Pre-Pack – Will display a check mark if the purchase order contains any pre-pack stock.

Breakdown View

Displays the Itemmaster/size level detail of the items contained within the purchase order.

  • Itemmaster Code – The unique item code.
  • Itemmaster Description – The description of the item.
  • Site Code – The unique code for the site at which the purchase order stock is arriving.
  • Site Description – The description for the site at which the purchase order stock is arriving.
  • Priority – The global site priority for the destination site. This is a read-only field and must be managed within Site Manager. Does not affect any calculations.
  • Priority Group – If in use, the associated priority group of the DC. This is a read-only field and must be managed within Site Manager. Does not affect any calculations.
  • Size Name – The size name associated with the itemmaster.
  • Size Type Name – The size type associated with the size.
  • Cost Per Unit – The per unit cost, reflective of the cost for the itemmaster at the destination site, displayed in the currency of the destination site.
  • Estimated Receipt Date – The calculated estimated receipt date, defined as the PO order date, plus the lead time. This is only an estimate.
  • Outstanding Qty – The by size outstanding order quantity.
  • Is Pre-Pack – Will be checked if the listed size is a pre-pack item.
