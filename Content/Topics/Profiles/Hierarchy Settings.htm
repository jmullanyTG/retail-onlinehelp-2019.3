Hierarchy Settings

Profiles are created at a location level and a product level. This makes it possible for users to apply a profile to the projected demand for dozens or hundreds of inventories in a single step. This can be incredibly useful when introducing a new product category or creating profiles for new locations.

The hierarchy has to have at least one product level and one location level; this gives the user granularity options.

Editing hierarchy settings

To edit hierarchy settings:

  1. Click Manage.
  2. Select a profile set from the list in the Profile Set Management window and click Edit.

  3. In the Update Profile Set dialog, click Edit Hierarchy.

    The Maintain Hierarchy dialog displays:

  4. In the Maintain Hierarchy dialog, optionally:
    1. add hierarchy levels using the Available Levels drop down menu.
    2. reorder the levels by highlighting one and using the reorder arrows on the right hand side.
  5. Click Ok to exit the dialogs.

Fields

  • Available Levels – Allows users to select which levels (from various entities) they would like to include.
  • Description – Once selected, the levels will be identified by their descriptions.
  • Level Order – Within the same entity type, the levels must be order to infer a hierarchal relationship (e.g. itemgroup1, itemgroup2, etc.)
  • Product Level – The explicitly defined product level (must be at least one).
  • Location Level – The explicitly defined product level (must be at least one).
