Creating a New Filter

You can create new filters from the Filters screen and from wherever filters can be selected within Demand Management. The process is the same either way, except that when creating filters in the Filters screen you will be asked to select an entity type. (When creating filter in another area of Demand Management, the entity type is pre-defined based on the current context. For example, when creating a new filter in the Supplier screen, the entity type is pre-defined as “Supplier”.)

Creating a filter from the Filters screen

  1. From the Management tab, click Filters.
  2. In the Maintain Filters dialog box, select Create a New Filter.
  3. If you want to base the new filter on an existing filter, select Clone from Existing Filter and choose the filter to clone from the Filter list below.

  4. Click Ok.

    • If you did not select Clone from Existing Filter, choose the entity type for the new filter. Otherwise, the new filter’s entity type will be the same as the filter you chose to clone.
  5. Edit the filter’s details, criteria, and conditions.
  6. Click OK to save the new filter.

Creating a filter from within another area of Demand Management

For convenience, whenever you are asked to select a filter in Demand Management, you have the option of creating a new filter without having to switch to the Filters screen.

  1. Click the Maintain () button next to the filter list to open the Maintain Filters dialog box.

  2. In the Maintain Filters dialog box, select Create a New Filter.
  3. If you want to base the new filter on an existing filter, select Clone from Existing Filter and choose the filter to clone from the Filter list.
  4. Click OK.
  5. Edit the filter details.
  6. Click OK to save the new filter.

About cloned filters

Cloned filters have the default name: Cloned From: <cloned filter name>. You can edit this name as desired. Cloned filters inherit the entity type of the filter they are cloned from.

About entity types

Filters are distinguished by the type of “entity” they support (inventory, site, supplier, etc.). As different entity types are used for different functionality and analysis throughout the system, the filters that support this functionality and analysis are also based on entity type.

When selecting filters in a Demand Management screen, only the filters created for the applicable entity type will be available for selection. For example, when choosing a site filter in the Site Manager screen, users will see only site filters listed—not site filters and inventory filters and supplier filters, etc. This keeps things simple and makes it easier to find relevant filters.

When creating filters in the Filters screen, you can manually select the entity type of the filter you are creating. In all other locations, the entity type is pre-defined based on the context of the location. For example, when creating a new filter in a site filter selection screen, the entity type of the new filter is pre-defined as Site.
