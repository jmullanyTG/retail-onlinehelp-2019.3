If constraining down:

  1. The order line is not locked
  2. The replenish order quantity (user order quantity!) is a positive number
  3. The order line was not constrained within the previous loop/iteration
  4. Ordered by replenish order quantity, ascending
  5. If replenish order quantity is identical, ordered by default priority, descending
  6. If default priority is identical, ordered by OrderLineID

This allows for the constraining up of items with a higher volume stock is contained in current period order, wherein the future period orders is relatively small.
