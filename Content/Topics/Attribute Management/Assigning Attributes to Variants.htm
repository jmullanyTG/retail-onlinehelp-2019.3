Assigning Attributes to Variants

Product attributes can be assigned to variants through Variant Manager

On launching Variant Manager, a filter selection dialog will appear, allowing the user to specify a filter of variants with which to work:

  • Filter: user editable drop down; selects a variant filter of products with which to populate the variant manager document
  • : this button launches the Maintain Filter dialog, allowing the user to create or import a new filter, or update, delete, or export an existing filter
  • Select only some items from the current filter: launches the Item Selection dialog

  • The dialog will list all variants returned by the criteria in the selected filter; select the desired variants to include in the document by clicking the corresponding check boxes in the Selected column
  • Items can also be selected using the buttons at the top left
    • : selects all rows currently highlighted
    • : de-selects all rows currently highlighted
    • : selects ALL rows
    • : de-selects ALL rows
  • Filter the list by typing in the filter row for any column

  • Typing in the filter row produces a “Starts With” filter effect by default
  • Typing % at the start of the value you want to filter for produces a “Contains” type filter instead (as shown above)
    • Choose Column(s): launches the Choose Fields Dialog, in which users can select additional columns to include in the Item Selection view to provide additional context for determining which products are desired

  • Cancel: closes the Item Selection dialog without selecting any items
  • OK: opens a variant manager document with the selected items
    • Cancel: closes the filter selection dialog and does not open a variant manager document
    • OK:  opens a variant manager document with the selected filter

One a variant manager document is open, users can select the variant(s) to be worked with in the top, or Grid, section of the document. Details for the selected variants will appear in the bottom, or Details, section.

Users can work with a variety of aspects of variants within a variant manager document by clicking the appropriate tab in the Details section. To assign or edit product attributes for the selected variant(s), click the Variant Attribute Management tab.

both the Grid and Details sections of the view can be toggled on and off by clicking the corresponding buttons () in the upper right of the view.

  • Variant Attribute Management
    • Variant Hierarchy: user editable drop down; selects the variant hierarchy for which to assign attributes

      • because it is possible to create alternate variant hierarchies which contain the same variants, it is also possible to have different attributes assigned to a variant across those hierarchies; ensure the correct variant hierarchy is selected before proceeding

    • Parameters: shows/hides the Parameters area of the Details section
    • Execute: updates the Details section of the view according to the selected Variant Hierarchy and the variant(s) selected in the Grid section
    • Family Name: non-editable alpha numeric; displays the name of the attribute family
    • Family Description: non-editable alpha numeric; displays the description for the attribute family
    • Single Values: user editable drop down; displays/sets the attribute value of the adjacent single-select attribute family for the selected variant(s)
      • when variants are selected that have different attribute values assigned, this field will show “(Multi)” to indicate this
    • Multiple Values: user editable multi-select; displays/sets the attribute values of the adjacent multi-select attribute family for the selected variant(s)
      • when multiple variants are selected that have different attribute values assigned, this field will show the combination of all values present
    • Apply: saves all changes made to attribute values for the selected variant(s) since the last Apply
    • Cancel: cancels all changes made to attribute values for the selected variant(s) since the last Apply and reverts them to previously saved values
    • Data Population: the view will populate with all attribute families that any of the selected variants is eligible for; attribute values available for selection will include all values that ANY of the selected variants is eligible for
    • Applying Changes Across Multiple Items: when multiple variants are selected in the Grid section, the system will automatically resolve any conflicts that exist between the Applied values and the attribute value eligibility of the variants the change is being applied to
      • Single-select attribute value changes will be applied to all variants that are eligible for them; ineligible variants will remain unchanged
      • Multi-select attribute value changes will be applied additively to variants that are eligible for them; if only some of the applied values are eligible, those will be applied; ineligible variants will remain unchanged
    • Dual-directional Attribute Flow: depending on where attributes are assigned – to placeholders in Assortment Planning or to variants in Variant Manager – attributes are transferred when a variant is assigned to a placeholder
      • Assigning a variant – any product attributes attached to the variant are transferred to the placeholder; any attributes that were already assigned to the placeholder but that don’t conflict with the variant attributes (i.e. single-select families that the variant already had assigned), they will remain on the placeholder and also be transferred to the variant
      • Attribute changes in Assortment Planning – any changes made to placeholder attributes in Assortment Planning will be transferred to the assigned variant for the product hierarchy assigned to the active assortment plan; they will not, however, be transferred to other placeholders that variant is assigned to
      • Attribute changes in Variant Manager – changes made to variant attributes in Variant Manager will be transferred to all placeholders that variant is currently assigned to across all existing assortment plans that have the selected Variant Hierarchy assigned
