Attribute Families Widget

The attribute families widget allows for the creation and maintenance of attribute families. Each attribute family contains attribute that pertain to that family and can be imported from host systems if available.

The attribute families widget returns a number of columns that define how attributes can be assigned and which families are available in the appropriate sets.

  • Family Name – Provides a descriptive name for the family of attributes
  • Family Description – provides a description of the attribute family and the attributes it contains
  • Entity types – the entity types selection restricts the data entities to which the attributes within the family can be applied.

    • the Variant type must be assigned for an attribute family to be definable on variants within Demand Management; the Variant Placeholder entity type must be assigned for an attribute family to be definable for placeholders within Assortment Planning

    • by only assigning the Variant Placeholder entity type to a family, that family will not be assignable to variants and will be exclusively used within Assortment Planning; this can be useful in organizations where a master data team defines the primary attributes but there are others that users within the merchandising team wish to assign while planning

  • Allow Multiple Values – This check box defines whether a user will be able to select multiple attributes from a family to assign to an entity.
  • AP Updates Variants – This check box defines whether changes will be allowed in Assortment Planning to attributes attached to assigned variants
    • Checked means the corresponding attribute families will be editable in Assortment Planning for placeholders with assigned variants, and any changes made to them will update the assigned variant’s attributes
    • Unchecked means the corresponding attribute families will be grayed out and uneditable in Assortment Planning for placeholders with assigned variants

    • this setting only appears when the system policy Update Variant Attribute Values In Assortment Planning is set to TRUE and is only valid for attribute families with the Variant Entity Type assigned; see Demand Management Assortment Planning Standard Documentation > Line Visualization > Supporting Views > Attributes View for more information

  • Attribute Set Assignment – The attribute sets selected in the filter described above will dynamically form columns which users can check to specify whether an attribute family is available for that set.

Attribute Values

On selecting the attribute family from the widget above, the attribute values widget will update.

The attribute values widget allows users to create new widgets that have not been imported from a host system.

  • Value Code – The value code is a distinct code that identifies the attribute value
  • Value Description – The value description describes the attribute value
  • Value External ID – This identifier is read only and will only be populated on attribute values populated from external systems. Users may not edit attribute values on imported data.
  • Behavior – For each attribute set to which the attribute value may be assigned, there are three behaviors for the value
    • “Blank/None” – This will mean the users cannot select this attribute value to assign to a product associated with the attribute set.
    • Allowed – This means the attribute value is allowed to be set against a product associated with the attribute set.
    • Default – This indicates the attribute value or values (depending on if multi values are allowed) that will be defaulted against any product associated with the attribute set.
