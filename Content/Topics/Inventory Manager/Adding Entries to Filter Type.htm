Adding Entries to Filter Type

Once views are created (or data tables have been populated), entries must be made to tbl_FilterType. This will determine how the data contained within the view is organized and accessed within the user interface and by the filtering machinery.

  • FilterTypeID – Unique identifier. Will be referenced as the primary key by tbl_Common_Filter_MetaViewData.
  • FilterTypeDescription – The grouping for the filter data.
  • FilterViewName – Name of the view (or table) which has been created and contains the reference data.
  • MetaDataViewName – The reference table containing the metadata. Best practice – set this value to tbl_Common_Filter_MetaViewData.
  • PrimaryKeyFieldName – The primary key referenced within the view.
  • DuplicateFlag – not used.
  • DefaultFlag – If set to “1”, data from this view will be returned within the grid at launch of all document types for the filter entity.
  • FilterEntityTypeID – The ID of the filter entity type.
    • 0 – Inventory
    • 1 – ItemMaster
    • 2 – Site
    • 3 – Supplier
    • 4 – Customer
    • 5 – Promotion
    • 6 – KeyItem
    • 7 – Event
    • 8 – KeyItemVariant
    • 9 – File
    • 10 – ItemMasterGroup
    • 11 – CustomerInventory
    • 12 – Variant
    • 13 – VariantGroup
