Defining Metadata

The data from the created view will need to be referenced by field in order for the filtering machinery to recognize and return it. This reference exists in tbl_Common_Filter_MetaViewData and should be populated by the initialization (INIT) procedure which was created to accompany the view.

  • FilterTypeID – Primary key referenced on tbl_FilterType.
  • ViewColumn – The column name as it appears in the referenced table or view within tbl_FilterType.
  • TableName – The name of the table or view which contains the data.
  • TableColumn – The column name as it appears in the referenced table or view within the TableName field.
  • FriendlyName – The friendly name as it appears within the user interface of the application, before any translations. This will be the name referenced within all standard product documentation.
  • VisibleFlag – Determines if the field will be visible within the user interface. May be set to “0” and used by filtering only.
  • Updateable – If set to “1”, the field will be write enabled for users.
  • LookupField – If the field is a lookup field, the string value must be entered in this column. Can be used to refer an ID column to a description column.
  • Aggregatable – The aggregation type for the field.
  • 0 – None
  • 1 – Sum
  • 2 – Average
  • 3 – Max
  • 4 – Min
  • 5 – Count
  • 6 – Custom
  • 7 – Custom Distinct Sum
  • 8 – Custom Distinct Average
  • 9 – Custom Distinct Count
  • Priority – The inherited priority for the field. If more than one priority should be applied, separate row entries must be made. Priority 1 will inherit priority 0 abilities.
  • 0 – Filtering
  • 1 – User Interface
  • UpdateTableName – If the value may be updated, the name of the write-back destination must be specified.
  • ExpressionColumnFlag – If the column is an expression column, set to “1”.
  • ExpressionColumnString – The expression to be used.
  • ColumnFormatStringFlag – If active, will adhere to standard format string for user interface presentation.
  • ColumnFormatString - Standard format string for user interface presentation.
  • ParentTableColumn – Used to assist with lookup data related to trees.
  • ;">AplyUserItemAssignOnLookupDataFlag – If selected, will apply user assign logic to lookups.
  • LookupFlag – If selected, will identify a lookup.
