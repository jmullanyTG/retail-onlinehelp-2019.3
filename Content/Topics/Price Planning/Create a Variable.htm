Create a Variable

Within the Price Manager Configuration Variable screen, click New to create a new Variable, enter the required fields and click Save.

Field Description

Variable Name (Required)

Enter a descriptive label to summarize the purpose of the Variable.

Variable Description (Optional)

Enter a summary describing the purpose of the Variable.

Variable Format String (Required)

A designation that determines how the system will format manual entry:

  • Currency (C2) – The system will format the data using the configured currency with two (2)

    • Database Value: 5.990000
    • Cost (C2): $5.99

  • Decimal (D6) – The system will format the data as a whole number using a minimum of six 6 digits

    • Database Value: 5.990000
    • Cost (D6): 5.990000

  • Number (N2) – The system will format the data as numeric field with two (2) decimal digits

    • Database Value: 5.990000
    • Cost (N2): 5.99

  • Percent (P2) – The system will format the data, multiplied by 100 with one (2) decimal digits

    • Database Value: 0.5990000
    • Percent Off: 59.90%

Detailed information can be found on Microsoft’s MSDN website:

https://msdn.microsoft.com/en-us/library/dwhawy9k(v=vs.110).aspx\

Active (Optional)

Determines if the Rounding Criteria will be made used when a Rounding Rule is configured with a Calculation.

Variable Output Field Name (Read Only)

The system will populate this label, for the variable, when the record is saved. This label is used when designing calculation formulas as well as the export staging tables.

Variable Input Type (Required)

Selection of an option, from the drop down, will determine the available options in the ‘Variable Input Price Type Field’.

  • Cost – An imported field, to be referenced only and not written to, for calculation formulas and validations. The list is configured during the solution implementation and is determined by actual fields configured to be imported. 
  • Price – An imported field to be referenced only and not written to, for calculation formulas and validations. The list is also configured during the solution implementation and is determined by actual fields configured to be imported. 
  • User – A manually entered or calculated to field

Variable Input Price Type Field (Required)

The drop down is populated with fields configured by the Variable Input Type. Select an available field to be associated to the Variable.

Variable Output Override (Optional)

The check box will only be editable for ‘User Entered’ variables. Marking the check box permits you to enter data to be associated to the Variable.

Variable Active (Optional)

Unmarking the check box, will remove the ability to configure the Variable in any new Calculations. When a Price Action is calculated the Price Strategy calculations and validations will be affected.
