Add a Price Strategy Rule

Once a Price Strategy has been selected, click New (i.e. in the Strategy Rule grid) to add a Price Strategy Rule, enter the required fields and click Save.

Field Description

Order (Required)

Enter a descriptive label to summarize the purpose of the Price Strategy.

Type (Required)

Select whether a ‘Calculation’ or a ‘Validation’ will be configured. Selection of ‘Type’ will populate the Rule Calculation drop down.

Description (Optional)

Enter the priority order of the Strategy Rule. The system will re-order the existing priorities accordingly, if appropriate.

Rule Calculation (Required)

Select the calculation/validation to be used in the Price Strategy from the drop down. Only ‘active’ calculation and validations will be populated to the list.

Calculation Description (Read Only)

The system will populate the Calculation Rule Description, which is not editable in this screen.

Validation Severity (Required)

Select how the system should evaluate Calculation Rule failures. Severity can only be set for ‘validation’ types.

  • Upon failure (Low Priority), the system will provide notification, in the validation log, but will continue calculation and evaluation of any following Price Strategy Rules

  • Upon failure (Medium Priority), the system will provide notification, in the validation log, but will continue calculation and evaluation of any following Price Strategy configured Calculation Rules

  • Upon failure (High Priority), the system will provide notification, in the validation log, and will not continue calculation and evaluation of any following Price Strategy configured

  • Error Override (Optional) – This check box may be marked to allow users, with permissions, to override validation failures so the system will continue calculations and validation evaluation.
Allow Validation Override

Selecting the check box allows configured Users such as Merchant\Buyer Managers to override (approve) Validation failures. This reduces the number of user reviews of Validation failures throughout the lifecycle of the Price Action. All Validation Severity Types (Information, Warning, Error) can be overridden.

The ability to override is also in conjuction with the Workflow Status State configuration.

Active (Optional)

Unmarking the check box, will cause the system to not evaluate the configured Calculation Rule when calculating/evaluating Price Strategies.
