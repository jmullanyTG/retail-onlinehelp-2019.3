Adding new Clusters to a Pricing Plan

You can create a Site Cluster and add the new Cluster to a Pricing Plan. The new Cluster must include at least one Site.

For new Site Clusters:

  • the currency is derived from the Sites selected in the filter
  • if multiple currencies exist, the system re-uses the Site Cluster name by currency (for example, EMEA(GBP) and EMEA(EUR)).
  • the system only applies Sites to the new Site Cluster not already associated to the Pricing Plan’s Clusters.

To add a new cluster:

  1. Select New Cluster in the toolbar of the Sites grid.
  2. Enter a name that is unique to the Pricing Plan in the Site Cluster Name field.

  3. Click the drop down arrow in the Site Filter field to choose a set of Sites.

  4. Optionally, to return a subset of Sites from the Site filter select the check box labeled Select only some Sites from the current filter.

    1. On the Site Selection dialog, select the check boxes for the Sites to be added to the new Site Cluster.

      NOTE: Only those Sites meeting the filter criteria and not already associated to the Pricing Plan’s pre-configured Site Clusters display.

    2. Click Ok.
  5. Click Ok on the New Pricing Site Cluster dialog.

  6. Click Ok on the following confirmation popup window.

    The Pricing Plan’s Site Cluster grid refreshes with the new Site Cluster.

  7. If the Activate the New Site Cluster(s) to Existing Action(s) window displays, then select the Price Actions, not already exported, to include the new Site Cluster.

    NOTE: The Activate the New Site Cluster(s) to Existing Action(s) window does not display if:

    • there are no future Price Actions

    • all future Price Actions are in a core status

      A core Status is a Status that is owned by the system and is not user configurable. Core statuses do not display in Workflow. The following are core statuses:

      • Preparing Data for Export
      • Ready For Export
      • Exported
      • Export Confirmed
      • Exported With Errors

  8. If required, click Save on the Activate the New Site Cluster(s) to Existing Action(s) window.

    Upon clicking Save, the Site Cluster and associated Sites are added to Price Actions as Included and the Pricing Strategy Variable default values are applied to the new Site Cluster and pushed down to Variants\Sites. Also, the system marks the following as Calculation Required:

    • Site Cluster Variant Sites

    • Site Cluster Variants

    • Site Cluster

    • Price Action

    • Price Action's Plan

    If a Price Action has gone into a calculating state, the system will add the Site Cluster and associated Sites as Excluded and display a dialog indicating that not all Price Actions were updated.

    For Price Actions that were not selected, the Site Cluster and associated Sites are Excluded. Also, the system will not apply any default values nor mark any associated Plan, Price Action, Product, or Site objects as Calculation Required.
