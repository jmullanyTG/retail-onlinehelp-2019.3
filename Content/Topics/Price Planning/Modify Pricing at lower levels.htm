Modify Pricing at lower levels

Data entered in the Price Strategy Variable grid populates to all Price Action Variants and Sites included in the Price Action (except excluded). To specify different values, from what is configured at the Price Action level (i.e. Price Strategy Variable grid), modifications may be made at three lower levels: Action Summary, Item Detail and Location.

 The following grids each have a column labeled Validation. This column uses icons to denote Validation failures and whether the Validation failures have been reviewed or not. For more information, see Price Planning Home Screen.

Modifying values in the Action Summary grid

Values are populated down from the Price Action Variable grid down to the Action Summary in which you have the ability to modify those values. Enter a new value for the applicable Variant variables within the Action Summary grid. Modifying a value for a specific Variant will apply the modified entry to all lower levels (i.e. Item and Item Location Detail). The system will also denote any differences within a level by shading the cell. Lower level differences will be noted by a marked Exception check box.

Excluding or re-including products at the Variant, Item Master, and Inventory levels

You can exclude products at the Variant, Item Master, and Inventory levels using the Advanced Product Exclusions dialog. Multiple selections at a product parent level displays the summary of children at the lower levels for modification.

To display the Advanced Product Exclusions dialog, click the Advanced Exclusions button.

The Variant grid and Item grid display by default. Click View Location to display the Item Location grid which lists all Price Action inventories associated to the selected Site Cluster. Select View Items to hide or display the Items grid.

Excluding Variants

  1. Open the Advanced Product Exclusions dialog.

  2. In the Variant grid, select Variants to be excluded.

    The Exclude button is enabled.

  3. Click the Exclude button.

    The screen automatically refreshes and the names of excluded Variants are denoted by a strikethrough.

    Also, the system saves excluded Variants, associated Item Masters, and Inventory records.

    The system will not exclude all Variants. Therefore, the following message displays if you attempt to exclude all Variants:

  4. Click Ok.

Re-including Variants

  1. Open the Advanced Product Exclusions dialog.

  2. In the Variant grid, select excluded Variants to be re-included.

    The Include button is enabled.

  3. Click the Include button.

    The screen automatically refreshes and the names of re-included Variants are no longer denoted by a strikethrough.

  4. Click Ok.

When you re-include a Variant back to the Price Action it needs to be calculated, which also means:

  • The Variant's lower levels need to be calculated (Item Master and Inventory)
  • The Site Cluster needs to display the worst-case State, of its associated Variants, which is now Calculation Required
  • The Price Action needs to display the wost-case State, of its associated Site Clusters, which is now Calculation Required
  • The Pricing Plan needs to display the worst-case State, of its associated Price Actions, which is now Calculation Required

Excluding Item Masters

  1. Open the Advanced Product Exclusions dialog.

  2. In the Items grid, select the Item Master to be excluded.

    The Exclude button is enabled.

  3. Click the Exclude button.

    The screen automatically refreshes and the names of excluded Item Masters are denoted by a strikethrough.

    Also, the system saves excluded Item Masters.

    The system will not exclude all Item Masters associated with a Variant. Therefore, the following message displays if you attempt to exclude all Item Masters associated with a Variant:

  4. Click Ok.

  2. In the Items grid, select excluded Item Masters to be re-included.

    The Include button is enabled.

  3. Click the Include button.

    The screen automatically refreshes and the names of re-included Item Masters are no longer denoted by a strikethrough.

  4. Click Ok.

When you re-include an Item Master back to the Price Action it needs to be calculated, which also means:

  • The Item Master's lower level need to be calculated (Inventory)
  • The Variant needs to display the worst-case State, of its associated Item Masters, which is now Calculation Required
  • The Site Cluster needs to display the worst-case State, of its associated Variants, which is now Calculation Required
  • The Price Action needs to display the wost-case State, of its associated Site Clusters, which is now Calculation Required
  • The Pricing Plan needs to display the worst-case State, of its associated Price Actions, which is now Calculation Required

Excluding Inventory

  1. Open the Advanced Product Exclusions dialog.

  2. In the Items grid, Click the View Locations button to display the Item Locations grid.

  3. Select the inventory to be excluded.

    The Exclude button is enabled.

  4. Click the Exclude button.

    The screen automatically refreshes and the names of excluded inventory is denoted by a strikethrough.

    Also, the system saves excluded Inventory.

    The system will not exclude all inventory per Item Master. Therefore, the following message displays if you attempt to exclude all inventory associated with an Item Master:

  5. Click Ok.

Re-including Inventory

  1. Open the Advanced Product Exclusions dialog.

  2. In the Items grid, click the View Locations button to display the Item Locations grid.

  3. Select the inventory to be excluded.

    The Include button is enabled.

  4. In the Items grid, select excluded inventory to be re-included.

    The Include button is enabled.

  5. Click the Include button.

    The screen automatically refreshes and the names of re-included Inventory are no longer denoted by a strikethrough.

  6. Click Ok.

When you re-include inventory back to the Price Action it needs to be calculated:

  • The Item Master needs to display the worst-case State, of its associated Inventory, which is now Calculation Required
  • The Variant needs to display the worst-case State, of its associated Item Masters, which is now Calculation Required
  • The Site Cluster needs to display the worst-case State, of its associated Variants, which is now Calculation Required
  • The Price Action needs to display the wost-case State, of its associated Site Clusters, which is now Calculation Required
  • The Pricing Plan needs to display the worst-case State, of its associated Price Actions, which is now Calculation Required

Revising Variant details using the Variant Details screen

For a specific Variant, click the field in the Detail column in the Action Summary grid to display the Variant Details screen. The Variant Details screen displays values at the Item and Item Location levels.

The Items grid (top) displays the individual products across all valid Sites. Modifying a value causes the system to shade the cell and to apply the modification to all sites. Site modifications can be viewed in the Item Locations grid (bottom). Modifications may affect calculated results which will be aggregated in the Action Summary grid as well as marking the Exception check box for ease of filtering.

The Item Locations grid (bottom) displays the individual products by each Site. Modifications may affect calculated results which will be aggregated in the Variants and Sites tabs of the Action Summary grid as well as marking the Exception check boxes for ease of filtering.

Note: The Validation column only displays at the Item Master or Inventory levels if the Advanced Validation Log system policy is set to True.
