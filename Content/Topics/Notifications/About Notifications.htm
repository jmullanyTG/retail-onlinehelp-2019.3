About Notifications

Notifications inform you when processes (e.g. calculations, forecasts, validations, etc.) have been initiated or completed and if any errors occurred.

Notification delivery settings are system-configured, but you can change these settings on the Notification Preferences screen.

There are three notification solution areas in Demand Management:

  • Notification Subscriptions: the Notification Subscription screen provides the ability to subscribe to system-generated notifications, by solution, and configure the default delivery method for each subscription. Additionally, administrators have the ability to create custom notifications, by solution, which can be scheduled and sent by the Demand Management Job Manager or through customized stored procedures (e.g. CUSP hooks) in the respective solution.
  • User Notification Preferences: allows you to override any system-configured delivery method.
  • User Notification Inbox: displays the notifications as they were configured. When a notification is received, the Notification icon, on the right side at the top of the screen, indicates the number of unread notifications. Click the Notification icon to display the Notification Inbox.

You can subscribe to notifications and configured them in the following solutions:

  • Allocation
  • Assortment Plan
    • Order Manager - Prepack
  • Assortment (Receipt) Plan
  • Cluster Site Manager (Custom Notifications only)
  • Content Manager (Custom Notifications only)
  • Curves
  • Events (Custom Notifications only)
  • Inventory and Level Forecast
  • Group Forecast
  • Price Manager
  • Promotions (Custom Notifications only)
  • System (Custom Notifications only)
  • Workflow (Custom Notifications only)
Notifications must be configured only by Demand Management Professional Services, Support, or a trained on-site super user.
