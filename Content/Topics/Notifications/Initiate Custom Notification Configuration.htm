Initiate Custom Notification Configuration

For more information on creating Jobs, associated Tasks and Schedules, see Job Manager. Within a Scheduled Job’s Task or in the CUSP, the Token parameters will need to be configured to initiate a Custom Notification. (See examples.)

Field Description

iMessageUUID (Required)

Enter the UUID, provided by the system, for a specific Custom Notification.

iSenderUserID (Required)

For the 3.6.0 release, this parameter should always be set to a value of one (i.e. IOAdmin).

nRecipientUserIDList (Required)

Enter the user or Role that will receive the Custom Notification. If multiple Users and/or Roles will receive the Notification, enclose the User ID list within single quotes.

nMessageToken (Optional)

The message Tokens will be passed into the Notification call as a single string for the system to parse appropriately into the Notification Message. Please refer to the examples below proper configuration.

cTokenIdentifier (Optional)

If a value is entered, the system will attempt to be pulled Tokens from the table notification.TokenStore. This table is in place to store tokens for the queue so that the XML doesn't have to be perpetuated throughout the process. If both @nMessageToken and @cTokenIdentifier is supplied @nMessageToken Tokens will override Tokens present in [notification].[TokenStore], but union tokens that do not exist in both.

bDebug (Optional)

This should always be set to a value of zero unless Demand Management Professional Services or Support are investigating an issue.
