Importing and Exporting Views

Views may be exported from an instance of Demand Management and imported into a separate instance. This may be used to upgrade standard versions, or to transport work between development, QA, and production environments.

A new view may be imported using the “Import New” function.

Importing a view as new will create the view with a new universally unique identifier (UUID) within not just the current instance of Demand Management, but across any version. Views imported as new will not replace existing views and the feature should not be used to “upgrade in place”. When customizing or extending an existing standard view, it should first be exported, then re-imported as new to give the view a fresh UUID (and thus not be overwritten when upgrading).

Import Existing

An upgraded version of an existing view may be imported using the “Import Existing” function.

Import existing will search for and replace an existing view with the same universally unique identifier (UUID) within the system. This should be used to upgrade standard views in place.
