General Tab

Displays settings for the plan selected in the top section of the Manage Assortment Plans dialog.

Editing assortment plans

To edit an assortment plan:

  • Click Edit on the toolbar and edit the plan in the Update Assortment dialog.

Update Assortment dialog buttons

Cancel: Closes the Add Assortment dialog without saving the new plan.

Save and Close: Saves the assortment plan and closes the dialog once all required parameters are set.

General tab field descriptions

Name: The name for the assortment plan.

Description: A description for the assortment plan.

Calendar: The calendar for the assortment plan.

Period: The lowest level period to be planned in the assortment plan.

Product Hierarchy: The product hierarchy for the assortment plan.

Level Contribution: The product hierarchy level at which location % contribution tables are calculated.

Site Hierarchy: The location hierarchy for the assortment plan.

Cluster sets to be used within an assortment plan must be contained within a single leaf level of the assigned Site Hierarchy.

For example, if a hierarchy is assigned that goes down to Country level and contains the leaves North America, Canada, and UK, then cluster sets within the assortment plan cannot have sites that exist in more than one country.

Linked Financial Plan: An optional merchandise financial plan to which the assortment plan is linked.

Default Currency: The base currency to be used for all revenue metrics in the assortment plan.

Home Site: A site to use for determining the home currency VAT % to apply when calculating AUT/AUR within the assortment plan. This field populates from sites within the selected site hierarchy that have an assigned currency matching the selected default currency:

Notes
As VAT rates can fluctuate throughout the planning season, this value is linked to a site rather than using an explicit rate so that the Home VAT % can change dynamically as needed.
To accommodate cases where the site chosen as the Home Site closes or becomes otherwise inactive, this parameter remains editable even after the user begins making changes within the assortment plan.
The system policy Allow DCs as Home Site (accessed via Maintenance > Policies > Module: Assortment) determines whether sites of type DC appear for selection as the Home Site. This is to accommodate scenarios where planning occurs in a currency and/or country where there are no non-DC sites present.
Changing the home site in an existing plan initiates a recalculation of all VAT dependent metrics within such as AUT/AUR, etc.
The site selected for the Home Site should mirror the currency and VAT % you will use as a basis for your planning activities.

Sales Types: The sales types used in the location % contribution calculations for the assortment plan.

Period Horizon: The rolling number of weeks into the future that the assortment plan will extend.

Replen AUT Price Type: The price type from which AUT will be derived for placeholders of Replen Status.

Replen AUR Price Type: The price type from which AUR will be derived for placeholders of Replen Status.

Replen AUC Cost Type: The cost type from which AUC will be derived for placeholders of Replen Status.

Created By: The name of the user who created the assortment plan.

Created Date: The date the assortment plan was created on.

Last Initialized: The date the Seed process was last completed for the assortment plan.
