Hindsight Placeholder Management

The Hindsight Placeholder Management screen can be used to create new placeholders and manage Like Style mappings for new and existing placeholders within your plan.

To launch this screen, select Placeholder > Hindsight in the Main Planning View (this is not available in Collection view).

Drag and drop capability

Regardless of the view configuration (with Current on the left and Historical on the right or vice versa) users can drag and drop items from both sections to the other to set Map From Variants/Map To PHs (see below for details on these fields).

Toolbar functionality

  • # Placeholders: The number of new placeholder lines to add when New is clicked.
  • New: Adds the number of new placeholder lines specified in # Placeholders to the bottom of the Current grid.
  • Clear: Removes mapped Like Styles (Map From Variant) for selected placeholders
  • Delete: Deletes selected placeholders. Only valid for new placeholders. Existing placeholders are deleted in the Main Planning View.

  • Mirror Options: When highlighted, copies the selections made for the topmost new placeholder line to any beneath it. It is not necessary to un-highlight this option if you want to deviate from the selections made in the topmost placeholder line. Simply edit as you would normally and the button un-highlights automatically.

  • Current Collection: When highlighted, automatically sets the Coll Code and Coll Desc columns for newly created placeholders to the values for the collection of the placeholder that was selected when the Hindsight Placeholder Management screen was launched.

  • Historical: Flips the Current and Historical sections in the view.

    The dynamic of assigning historical variants to placeholders is essentially the same regardless of the layout, but be aware that while a single historical variant can be assigned to many current placeholders, when assigning in reverse, only one variant can be mapped to a current placeholder (if a blend of multiple variants is desired to seed rate of sale for a placeholder, the Rate of Sale Calculator can be used.

    Placeholders that are Inactive or in Committed status will not be shown in the Current section of the view.

    For more information, see Rate of Sale Calculator View.

  • Parameters: shows/hides the data selection parameters for the view.

Parameter options are the same as in the Main Planning View defaulting to the same product, site, and cluster set selections as in the main view. Window of Time defaults to the LY equivalent of the time selected in the main view.

Data selection parameters only determine the variants/placeholders that will populate in the Historical section of the view. Current placeholders are populated based on those shown in the Main Planning View.

Current section

  • Coll Code: Only editable drop down for new placeholders. Double-click in the drop down list to select a collection code to which the placeholder is assigned.

    Type in the space below Code or Desc of the drop down to filter the results returned; this acts as a “starts with” filter by default, but can accept “%” as a wildcard to do “contains” type filtering instead.

    To create a new collection, enter a description for the collection in the space provided and click New.

  • Coll Desc: The corresponding description for Coll Code for the placeholder
  • PH Code: Unique system generated code for the placeholder
  • PH Desc: The description for the placeholder
  • Image: The primary image associated with the placeholder. If no primary image exists, then No image data displays

  • Product Level: The product hierarchy level to which the placeholder is assigned.

    This defaults to the lowest common product hierarchy level present in the Product filter in the data selection parameters for Placeholder Mode.

  • Cluster Sets: The cluster sets to which the placeholder ise assigned. The cluster sets listed include all default and additionally available cluster sets for the product level selected in the Product parameter.

  • Phase In: The phase in date for the placeholder. If set, selections limit to periods prior to Phase Out.

    Defaults to the earliest non-actualized period within the time frame selected.

  • Markdown: The markdown date for the placeholder.

  • Phase Out: The phase out date for the placeholder. If set, selections limit to periods after Phase In.

    Defaults to the last period within the Window of Time in the data selection parameters.

  • Mkdn Offset Locked: Links the markdown and phase out dates to each other so they work dependently on each other. This helps users accurately plan the entire life of the item including markdowns. The standard default mode keeps the markdown and phase out dates independent of each other. Locked mode cannot be enabled on an item with infinite phase out.

  • Mkdn No Periods: The time frame for the markdown date to phase out date. Determines what the phase out date should be from the markdown date, once a user enters a markdown date during Placeholder creation (i.e. default is 4, markdown date is 4/29, phase out date will be 5/27, four weeks post markdown.) The default is ignored if the markdown falls outside of the phase in/out window or in the past.

    • If Phase Out is set first and a markdown default is set, then Markdown date will auto-set to N default periods before the user's chosen Phase Out.
    • If Phase Out is set first but there is no markdown default for the selected hierarchy level, then markdown date is left empty. If Markdown Date is set first and a markdown default is set, the Phase Out will auto-populate with the N default periods after the user's chosen Markdown Date.
    • If Markdown Date is set first and there is no markdown default, Phase Out will need to be manually entered by the user before saving, or standard phase out logic applies.
    • Any change to either Markdown Date or Phase Out Date after initial value is set does not preserve the default.

  • AUT: Currency ($10.01) Sets the home currency average unit ticket (AUT) for the placeholder. AUT is set to that of the selected variant if a Map From Variant is specified.

    AUT is assumed to be VAT inclusive for the purpose of all calculations within Assortment Planning. For companies in which VAT is not in play for the home currency assigned to the active assortment plan, no special actions are required as long as the associated VAT % is set to zero.

    For more information, see Currency and VAT Management Overview. For more information, refer to the Currency and VAT Management User Guide.

  • AUR: Currency ($10.01) Displays the average unit retail (AUR) price for the placeholder.
  • Margin: Percentage (10.01%) Gross margin percentage for the placeholder.
    • Calculation: ((AUR ex VAT / (1 + VAT %)) – AUC) / (AUR ex VAT / 1 + VAT %)
  • ROS $: Currency ($10.01) Displays the current ROS $ for the placeholder. Blank for new placeholders.
  • ROS U: Decimal (1.01) Displays the current ROS U for the placeholder. Blank for new placeholders.
  • Ttl Sls $: Currency ($10.01) Displays the current total sales revenue planned for the placeholder. Blank for new placeholders.
  • Ttl Sls U: Displays the current total sales units planned for the placeholder. Blank for new placeholders.
  • S/T %: Percentage (10.1%) Displays the target sell-through percentage for the placeholder.
    • Calculation: (Ttl Sls U / Ttl Buy U)
  • Promo?: Whether the placeholder has any active promotions applied to it.

  • Map From Variant: (Optional) Selects a placeholder item to use as a Like Style from which the system can seed a sales plan and other information.

    Like Styles can also be dragged from the Historical section onto the target placeholders in the Current section. Dragging a Like Style onto a placeholder that already has a Like Style assigned will replace the previous assigned style with the new one.

    Populates with placeholders and/or variants shown in the Historical section that exist within the selected Product Hierarchy for the line.

    If set, this copies the selected item’s Size Option, Curve Mode, Seasonality Curve, AUT (unless specified in Ticket Price), Min Pres, Location Exceptions, and receipt plan constraints (Receipt Variant Min, Receipt Size Min, Variant Order Multiple, Periods of Supply, Periods Before Phase In, Flow Period Frequency, Min Stock to Sales, Shrink % of Sales, Recalc Buy, and Hold Receipt Units) and calculates a sales plan based on sales present for the selected Like PH between the Comp Start and End dates specified.

    Typing in the field below Code, Desc, Phase In, or Phase Out of the drop down to filters the results returned. This acts as a “starts with” filter by default but can accept “%” as a wildcard to do “contains” type filtering instead.

  • Attrib Copy: Only valid if Map From Variant is set. Defines whether any product attributes assigned to the Map From Variant are copied to the new placeholder line or if any defined attribute defaults for the Product Hierarchy level will be assigned instead.

    For more information, see Attribute Management solution. For more information, refer to the Attribute Management User Guide.

  • Cluster Copy: Only valid if Map From Variant is set. Defines whether the placeholder copies the cluster assignments of the selected Map From Variant or use those set by the user on the placeholder line.

    Existing cluster assignments are overwritten when Map From Variants are assigned and their cluster assignments are copied for placeholders that existed prior to launching the Hindsight Placeholder Management screen.

    When Map From Variants are assigned but their cluster assignments are NOT copied for placeholders that existed prior to the launching of the Hindsight dialog, it is possible that some of the pre-assigned clusters will not have rates of sale if there were no sales in those clusters between the Comp Start and Comp End dates for the Map From Variant selected.

  • Cluster Set Message: Displays system messages regarding the copying of cluster assignments from a Map From Variant for the specified placeholder.

    An example of a scenario that would generate a message:

    Scenario: Placeholders X and Y are both assigned to Variant A.  Placeholder X is assorted to Cluster Set 1 and Placeholder Y is assorted to Cluster Set 2. Placeholder X in the Hindsight view is to be changed.

    If a Map From Variant that is assorted to Cluster Set 2 is selected, then cluster sets cannot be copied because it would cause both Placeholders X and Y to be assorted to Cluster Set 2 for the same variant.

    Cluster Set mesages

    • The Like Style is assorted to more cluster sets than are visible: (<list of cluster sets if the length of list is less than 512 characters OR the text "x cluster sets">), If applied with Cluster Assortment Copy checked, these assortments will also be created for this placeholder
    • The Placeholder to be altered is assorted to more cluster sets than are visible: (<list of cluster sets if the length of list is less than 512 characters OR the text "x cluster sets">). If applied with Cluster Assortment Copy checked, these assortments will also be removed from this placeholder
    • The Placeholder to be altered is assorted to cluster sets to which you do not have access: Assortments to this placeholder cannot be altered using this method
    • Changing assortments for this variant would cause overlapping cluster sets to exist over the same time ranges: This is when the placeholder to be altered is assorted to a variant, that variant is also assigned to other placeholders in the plan, and those other placeholders' combined cluster set / cluster / date assignments would be in conflict if the change was made.
  • Mod Factor: Defaults to 1.0. Only valid if Map From Variant is set. Defines a factor to multiply the sales history of the Map From Variant.
  • Comp Start: Defaults to the Phase In of the Map From Variant. Only valid if Map From Variant is set. Defines the start date for the desired window of time from which to capture sales for the Map From Variant.
  • Comp End: Defaults to the Phase Out of the Map From Variant. Only valid if Map From Variant is set; defines the end date for the desired window of time from which to capture sales for the Map From Variant. If set, selections are limited to periods prior to Comp Start.

  • Adj ROS $: Currency ($10.01). Blank unless Map From Variant is set. Displays the placeholder level ROS $ calculated from the Map From Variant based on the Comp Start and Comp End Dates and any Mod Factor specified that will be assigned to the placeholder.

    As rates of sale are calculated and assigned at the cluster level, actual rates of sale assigned to individual clusters may vary from the value shown here, just as differences can exist between the ROS $ shown in Placeholder view versus those in Cluster Set view.

  • Adj ROS U: Decimal (10.01) Blank unless Map From Variant is set. Displays the placeholder level ROS U calculated from the Map From Variant based on the Comp Start and Comp End Dates and any Mod Factor specified that will be assigned to the placeholder.

    As rates of sale are calculated and assigned at the cluster level, actual rates of sale assigned to individual clusters may vary from the value shown here, just as differences can exist between the ROS U shown in Placeholder view versus those in Cluster Set view.

Historical section

This section displays the placeholders and/or variants contained within the Product, Site Hierarchy, Cluster Set, and Window of Time parameters of the Hindsight Placeholder Management screen. This section functions similarly to the Style Bank in that items listed here can be dragged onto placeholders in the Current section.

The label Historical is a generic term; based on the parameter selections the user makes. Placeholders/variants from any time frame can be displayed here.

All metrics in this section are essentially the same as their Current section counterparts except that none are editable when the standard view configuration is engaged (meaning the Current section on the left and the Historical section is on the right).

When the Historical Section is on the left

This is achieved by clicking the Historical. When this is done, Map to Variant is changed to Map to PH, and the Attrib Copy, Mod Factor, Comp Start, Comp End, Adj ROS $, and Adju ROS U fields become part of the Historical section.

  • Map to PH: Selects a placeholder from those in the Current section with compatible product levels to map the item to as a Like Style.

    This field functions very similarly to the Map From Variant field except that each placeholder in the Current section can only be selected as a Map to PH once in the Historical section, as this view is not capable of mapping multiple items to a single placeholder. To blend multiple items together for the purpose of seeding rates of sale, users must use the Rate of Sale Calculator supporting view. For more information, see Rate of Sale Calculator View.

    Users can also drag and drop placeholders from the Current section to set the Map to PH. If the placeholder being dropped is already selected as the Map To PH for another line in the Historical section, it is removed from that line and set as the Map To PH for the new line.

    • Buttons

    • Cancel: closes the Hindsight Placeholder Management screen without committing any of the changes made.
    • Save: Adds all new placeholders created and commits all Map From Variant/Map To PH changes made. Also, this initiates recalculation of placeholder sales plans based on changes made.
