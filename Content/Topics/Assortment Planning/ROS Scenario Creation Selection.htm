Creating or selecting an ROS Scenario

Using the parameters and controls at the top and at the far left of the view, users can either select a saved rate of sale scenario to apply to the placeholders/collections selected in the Main Planning view, and edit that scenario as desired, or define the inputs for an entirely new scenario.

Creating a new scenario

  1. Select Variants and Sites as required:
    1. Select an existing placeholder/variant filter or create a new filter to select the items for the scenario (this determines which scenarios will appear after scenarios have been created and saved)
    2. Select an existing site filter or create a new filter to select the sites for the scenario (for application in assortment planning, select (All Sites)

  2. Click the Execute button.

  3. Click the Add button to create a new scenario.

    This resets the parameters at the left to their defaults to allow for a new scenario but does not alter any saved scenario that was selected previously.

    4. Field Description
    Name A name for the scenario (Populates with a default name that can be overridden)
    Description A description for the scenario
    Variant Filter The product filter for the scenario, this will drive which items are included in the calculation
    Cluster Set Names The cluster sets to be included for the scenario, this will drive which clusters are included in the calculation
    Time Frame

    An existing time frame or create a new time frame for the scenario

    Creating a new time frame for a scenario

    1. Complete the following fields as required:
      1. Name: A name for the Time Frame
      2. Description: A description for the Time Frame
      3. Dynamic Rule: Makes the time frame dynamic.  Save scenarios recalculate based on the most up to date historical data.
      4. Access Level: Whether a time frame that has been created is public, private or shared
      5. Offset days: Sets the number of days from today’s date to include in the data set; use negative numbers for days back and positive numbers for days forward
      6. Number of Rolling Days: Number of days forward from the day defined by the offset days (ex. If offset days is -365 and rolling days is 365 that gives the user the last year of historical data in the data set)
      7. Fixed Start Date: Utilize if the scenario is static to define the begin date of the data set to be utilized in the calculation
      8. Fixed End Date: Utilize if the scenario is static to define the end date of the data set to be utilized in the calculation
    Scenario Type Select Revenue or Sales Units to model against
    Sales Types The types of sales to include in the scenario (for example, Regular or Markdown)
    Price Type

    Select the price type to include in the scenario. Price type is only valid with Max Price, Min Price, and/or Max Price Discount are specified; price types are defined by your organization during software implementation and can be things like MSRP, Clearance, etc.
    Max Price Discount The max % discount to ticket price of variants to include in the scenario; set to zero to ignore this parameter
    Access Level Select access level to the scenario. If shared, select the shared users by clicking on the ellipsis

  4. Click Save.
  5. Click Calculate to calculate the scenario lines.

The difference between Sales Type and Price Type

Sales Types define the larger bucket of sales from which to draw. Price Type is an optional data point that can change by period and can be used to further limit the data selection for the scenario.

For example, it is possible to include the Sales Type Markdown but then further limit that data by setting a Price Type of Clearance and a Max Price Discount of 75% to only capture variants that had the Clearance Price Type and selling prices that were no more than 75% the original ticket price for each period in the scenario time frame.
