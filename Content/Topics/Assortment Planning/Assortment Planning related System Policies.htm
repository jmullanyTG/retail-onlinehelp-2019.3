Assortment Planning Related System Policies

The following table describes the available Assortment Planning related system policies.

System Policy System Name Description
Active Variant Threshold Factor Assortment.Target.ActiveVariantThresholdFactor Target Creation - Calculation and Rationalization Processes. Percentage of a variant's deseasonalized average sales that must be met for that period to be considered "Active". Formatted as a decimal (e.g. 0.3 = 30%).
Active Variant Time On Offer Minimum Assortment.Target.ActiveVariantTimeOnOfferMinimum Target Creation - Rationalization Process. Percentage of the scenario time frame that an item must have been on offer for to considered "Active". Formatted as a decimal (e.g. 0.31 = 31%).
Allow DCs as Home Site Assortment.Config.AllowDCAsHomeSite Determines whether sites of type DC are allowed to be selected as the Home Site (used in determining Home VAT %) in Assortment Planning.
Allow Strict Site Exclusion Assortment.Placeholder.AllowStrictSiteExclusion Site Exclusion is controlled at either Placeholder [True] or PlaceholderInstance [False]
Allow supplemental commitments Assortment.OrderManagerAddAssortmentPlanCommit.AllowSupplementalCommits Allow for supplemental commitment orders for a period which already contains a confirmed commitment order. 0 = Do Not Allow. 1 = Allow Supplemental Commits and Enforce Constraints on Supplemental Commits. 2 = Allow Supplemental Commits and Do Not Enforce Constraints on Supplemental Commits.
Assortment days history to actualize Assortment.Maintenance.DaysHistoryToActualize The number of days back to go when actualizing sales, receipts and stock on hand for assortment planning.
Assortment days history to maintain Assortment.Maintenance.DaysHistoryToKeep The number of days back maintain in a live assortment plan. Detail data older than this date will be summarized and then deleted. Entering the value "0" will keep all history.
Assortment Order Type Assortment.Report.APOrderType Variant Orders created from assortment planning are reconciled to the cluster when coming back as confirmed orders from the host system. Inventory Orders uses a calculated split factor to reconcile inbound order quantities to clusters.
Assortment sales types to actualize Assortment.Maintenance.SalesTypesToActualize The sales types, as defined in tbl_SalesType, to use when computed actualized sales numbers.
Average Inclusion Threshold Assortment.Target.LowSellingWeeksRemove Target Creation - Calculation and Rationalization Processes. Percentage of a variant's maximum sales that must be met for a period to be considered in the deseasonalization process. Periods of sales for the variant below the threshold will be removed before deseasonalization. Formatted as a decimal (e.g. 0.03 = 3%).
Batch size used in Assortment Actualization. Assortment.Maintenance.ActualizationBatchSize The number of PlaceholderIDs processed in a single batch during Actualization.
Days To Keep Deleted Data Assortment.Maintenance.DaysToKeepDeletedData The maximum number of days that deleted data will be held (hidden) in the database. Set a lower number for more aggressive deleting and more space conservation.
DC Stock Splits future number of periods Assortment.Maintenance.DCSplitCalcFuturePeriods Number of periods forward from the current period to determine % DC stock splits by cluster set / cluster. Default is 8.
Default Commitment Merge Assortment.OrderManagerAddAssortmentPlanCommit.DefaultMergeType Default Merge OrderLines when commitments are created - 0 = None, 1 = Merge by Variant
Editable Elements on Line Visualization when Replen Assortment.LineVisualization.EditableObjectsWhenReplenStatus Editable Fields / Objects when Placeholder Status has been set to Replen.
Image View Maximum number SQL Data Rows Assortment.ImageGrid.Maximum.SQL.DataRows The maximum number of SQL data rows that can be queried in a view that returns images.
Line Visualization Allow Price Exceptions Assortment.LineVisualization.AllowViewPriceExceptions Allow View / Edit of Price Exceptions in Line Visualization
Line Visualization Maximum number SQL Data Rows Assortment.LineViz.Maximum.SQL.DataRows The maximum number of SQL data rows that can be queried in a Line Visualization view.
Marginal Return Curve Extrapolation Assortment.Target.MarginalReturnCurveExtrap Target Creation - Rationalization Process. Integer representing the minimum number of data points required in a given period for the system to use the original calculated intra-style marginal return curve in determining the number of recommended choices to assign to each recommended collection. If this minimum is not met, the system will calculate a weighted curve based on the original intra-style marginal return curve and a curve calculated at a higher level of the product hierarchy to provide a more accurate recommendation.
Median Period Total Threshold Assortment.Target.PercentOfMedianPeriodTotal Target Creation - Calculation and Rationalization Processes. Percentage of a periods's median sales that must be met for a period total to be considered in the deseasonalization process. Periods with total sales below the threshold will be overwritten with the percentage of the median. Formatted as a decimal (e.g. 0.5 = 50%).
Multi Value Attribute Inheritance Assortment.Attributes.MultiValueOverwrite Defines the inheritance of multi-value attribute families attached to variants by the placeholders those variants are assigned to within Assortment Planning. Value of 1 indicates placeholder values will be entirely overwritten by the variant attributes on assignment of the variant. Value of 0 indicates unique values attached to the variant will be added to the existing placeholder values on assignment of the variant.
Prepack Max Overage Percent Resolution Assortment.DefaultResolution.MaxOveragePercent Aggregation Function to resolve default collisions
Prepack Max Overage Resolution Assortment.DefaultResolution.MaxOverage Aggregation Function to resolve default collisions
Prepack Max Pack Configs Resolution Assortment.DefaultResolution.MaxPackConfigs Aggregation Function to resolve default collisions
Prepack Max Pack Per Location Resolution Assortment.DefaultResolution.MaxPacksPerLocation Aggregation Function to resolve default collisions
Prepack Max Pack Percent Total Resolution Assortment.DefaultResolution.MaxPackPercentTotal Aggregation Function to resolve default collisions
Prepack Max Units Per Pack Resolution Assortment.DefaultResolution.MaxUnitsPerPack Aggregation Function to resolve default collisions
Prepack Min Pack Configs Resolution Assortment.DefaultResolution.MinPackConfigs Aggregation Function to resolve default collisions
Prepack Min Units Per Pack Resolution Assortment.DefaultResolution.MinUnitsPerPack Aggregation Function to resolve default collisions
Prepack Overage Weight Resolution Assortment.DefaultResolution.OverageWeight Aggregation Function to resolve default collisions
Prepack Pack Tolerance Resolution Assortment.DefaultResolution.PackTolerance Aggregation Function to resolve default collisions
Prepack Pack Weighting Resolution Assortment.DefaultResolution.PackWeighting Aggregation Function to resolve default collisions
Prepack Underage Weight Resolution Assortment.DefaultResolution.UnderageWeight Aggregation Function to resolve default collisions
Promotion Planning Time Periods Assortment.Promo.PromotionPlanningTimePeriods Promotions - Approximation. Integer representing the number of time periods before and after a promotion's application period that will be shown in Promotion Planning.
Promotions Applied to Markdown Default Assortment.Promo.ApplyPromosOnMarkdownsDefault Determines default property of whether promotions can be applied to a markdown period for an item. This can be overwritten at a promotion and promotion application level. If True, the default is that promotions can be applied to markdown periods.
Receipt Cluster Merge All Assortment.Receipt.MergeClustersTogetherByDefault When commitments are generated, whether to group receipts for variants together across clusters by default, or whether to keep them separate by default.
Require display Primary Image Assortment.ImageManagement.RequirePrimaryImageClassification Require usage of Primary Image Classification in Assortment Planning
Style Bank Line Visualization Maximum number SQL Data Rows Assortment.LineViz.StyleBank.Maximum.SQL.DataRows The maximum number of SQL data rows that can be queried in the Style Bank in the Line Visualization view.
Update Variant Attributes in Assortment Planning Assortment.Attributes.ChangeAttributesOnAssigned Defines the ability to make attribute changes on assigned placeholders. Value of 1 indicates that placeholder attribute changes are allowed on assigned placeholders. Value of 0 indicates attribute changes cannot be made to assigned placeholders.
Variant Primary Image Classification Update Assortment.ImageManagement.AllowUpdateVariantPrimaryImage Allow Update of Primary Image Classification if Variant Present - Assortment Planning / Variant Manager
VAT Rounding Digits Assortment.Maintenance.RoundVATDigits The number of digits to round to when calculating historical VAT. A number 4 here means that the VAT value will be rounded to 0.xxxx, or xx.xx%.
