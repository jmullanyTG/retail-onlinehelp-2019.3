Order Document Details – Break By Size

The Break By Size tab allows users to split order line quantities across assigned size curves, or specific sizes selected from curves, based on planned sales and anticipated size inventory across clusters. Sizing analysis is done for all order lines on a document simultaneously.

  • Header Area Controls

    • Save: saves all changes made to the Break By Size information for the order document since the last time it was saved
    • Undo: undoes all the changes made to the Break By Size information for the order document since the last time it was saved

    • Calculate: calculates the ideal units per size to order by variant and cluster, based on the Size Curve and Selected Sizes for each order line on the order document

  • Approve: approves the Break By Size information for the order document and transitions the Workflow Status of the document to Break By Size
  • Approve & Route: approved the Break By Size information for the order and launches the Commitment Status dialog, allowing the user to select the Workflow Status the document will be transitioned to from available states, based on the active workflow

    • Enforce Size Minimum: user editable check box; related to Receipt Size Min set in Receipt Plan view of Assortment Planning; when checked, enforces the total minimum quantity per size defined by that constraint and increases the order quantity to meet it, if necessary

  • Variant Detail Grid (left side)

    • Variant: non-editable alpha-numeric; displays the variant code for the corresponding order line
    • Cluster Set: non-editable alpha-numeric; if applicable, displays the cluster set for the corresponding order line
    • Cluster: non-editable alpha-numeric; if applicable, displays the cluster for the corresponding order line
    • Receipt Date: non-editable alpha-numeric; displays the effective delivery date for the corresponding order line
    • Size Curve: user editable drop down; displays/ sets the size curve to be applied to the corresponding order line; defaults to the size curve set for the variant in the assortment plan from which the order line was created
    • Selected Sizes: user editable multi-select; displays/sets the sizes to include in the corresponding order line; defaults to all sizes included in the selected Size Curve
    • Site Count: non-editable integer; displays the number of sites the corresponding order line is for

  • Sizing Pivot Grid (right side)

    • Avg U/Site: user editable integer; sets the desired average units per site for the corresponding size as a constraint for calculation of sizing needs

      • entering Avg U/Site can cause an increase in the original order quantity for the order line

  • Act Avg U/Site: non-editable integer; displays the calculated average units per site for the corresponding size by simply dividing the corresponding Order Units by the Site Count for the cluster

  • Quantity: non-editable integer; displays the raw system suggested units to be ordered for the corresponding variant/cluster/size

    • this quantity does not include changes for the corresponding user specified Avg U/Site constraint, if any

  • Constrained Qty: non editable integer; displays the constrained system suggested units to be ordered for the corresponding variant/cluster/size

    • this quantity includes changes for the corresponding user specified Avg U/Site constraint, if any

  • Final Qty: user editable integer; displays/sets the units to be ordered for the corresponding variant/cluster/size

    • Defaults to Constrained Qty value, but can be overridden by the user
    • User overrides are conditionally formatted in orange

    • Final Qty can also be entered in the subtotal/total rows of the Pivot Grid; doing so will proportionally split the number entered down to the lower levels based on their current values
