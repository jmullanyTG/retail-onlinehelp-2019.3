Smart Start

Once a scenario has been rationalized, a user has the option of Smart Starting. On Smart Start, the system generates placeholder items for each segment/cluster based on the following:

  • TY Act New Choices & TY Act New AUR
  • The results of the Inter and Intra-style marginal return curves
  • The Collection Option of each segment
  • The configuration of choices to clusters within each segment
  • The default minimum presentation, seasonality curve, curve mode, periods of supply, rec var min, rec size min, size curve, periods before phase in, flow cadence, min stk/sales, and shrink % of sales

Given these factors, the system generates a fully defined sales plan for each placeholder item, including rates of sale, total sales, sales by period, receipt units and flow strategy. Users can proceed straight to the assignment of actual variants and placing buys or use Line Visualization and its supporting modules to perform additional analysis and fine tune plans before proceeding.

If multiple users attempt to Smart Start scenarios at the same time, the system holds the second process until the first one finishes. This is transparent to the user aside from an apparent lag in processing time compared to normal for the user initiating the second Smart Start.

Overlapping scenarios

If the intersection of product and site hierarchies and time of a scenario being Smart Started overlaps any part of another scenario that has already been Smart Started, then the following warning displays:

Proceeding deletes any placeholders in Open status. This is because the sales volume those areas account for within the business have already been accounted for in the overall assortment plan and the system is attempting to prevent over-buying.

A Smart Start exists for Mens Tops, US Retail, 4th Quarter 2017 and all generated placeholders are in Open status.

If a new scenario is created, Mens Short Sleeved Tees, for US Retail – Southeast District, for December 2017 only, then all placeholders for the first scenario are deleted and replaced by those generated for the new scenario. This is because the new scenario contains portions of the sales volume that was accounted for in the first scenario.

Initial Receipt Plans

The system calculates an optimized receipt flow strategy by cluster using:

  • the rates of sale assigned to each placeholder from the Marginal Return Curve Matrix, and
  • the default seasonality curve, minimum presentation, and other receipt planning parameters set for the relevant level of the product hierarchy.

Saving time by setting up default parameters

Setting up default parameters before Smart Starting a scenario saves time and the effort of manually setting parameters later for each placeholder/cluster combination.

Reconciling Smart Start Output Back to Scenarios

The sum of the Ttl Sls $ and Ttl Sls U in Line Visualization for the placeholders generated by Smart Starting a scenario should be equal to the sum of TY Act New Sales $ or U for the scenario. These sums may also tie back to the Scenario Target minus the TY C/O Sales $ or U, but because TY Act New Sales $ and U can be edited by the user and do not have to tie back to 100% of the Scenario Target, if overrides are made to these metrics, they will not likely tie to the output of Smart Start.

Markdown dates

When a user Smart Starts a scenario and phase dates are automatically created from the scenario's plan dates, the Markdown Date is automatically created by backing the Number of Markdown Periods out from the phase out date.

For example, if the scenario is dated from Wk1 - Wk10, and the default is 5 weeks, then Markdown is automatically be set to Wk6.
