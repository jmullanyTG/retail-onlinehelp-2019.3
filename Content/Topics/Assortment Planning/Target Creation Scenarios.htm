Scenario Manager Target Creation section field descriptions

Field Description
Scenario ID System ID number for the scenario
Scenario Name Name for the scenario
Plan Basis Name of the MFP plan used as a basis for sales volume in the scenario; not required
Start Date Start date of the scenario’s Window of Time
End Date End date of the scenario’s Window of Time
Product Hierarchy Product Hierarchy levels planned within the scenario
Site Hierarchy Site Hierarchy levels planned within the scenario
Scenario Target Currency or integer (based on the Scenario Target Type); the planned sales volume in revenue or units for the scenario
Scenario Target Type The type of target sales volume for the scenario; can be either Revenue or Units
New Variants Sum of the maximum number of new choices for each segment within the scenario
Total Variants Sum of the maximum number of new choices + carryover choices for each segment within the scenario
Status The current state of the scenario within the established workflow; valid values are Created, Calculated, Rationalized, and Smart Started
Has Been Smart Started? Whether the scenario has even been Smart Started; default is unchecked
Privacy The privacy setting for the scenario; valid values are Public, Private, and Shared
Sharing Settings List of roles and/or users who can view/edit the scenario; only valid if Privacy is Shared
Create Date The date the scenario was created on
Create User The name of the person who created the scenario
Sales Types List indicating the sales types included in the scenario
Scenario Description Description for the scenario
End Date Definition Type The drop down menu has two options: Phase Out Date and Markdown Date. The default is Phase Out Date.
  • If a user selects Phase Out Date, then the maximum date will be the phase out date.
  • If a user selects Markdown Date, then the maximum date will be the Markdown Date, and Phase Out will be calculated based on the default markdown logic.
