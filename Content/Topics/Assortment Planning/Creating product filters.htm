Creating filters for items to which a promotion or promotion application has been applied

Information about promotions created within Assortment Planning can be used as a quick way to create product filters for the items to which a particular promotion or promotion application has been applied.

To create a product filter

  1. Navigate to the Promotion Application section on the Promotion Definition tab.
  2. Click on the Product Filter field to display the Items dialog.
  3. Click the icon to display the Maintain Filters dialog.
  4. Select Create A New Filter and click Ok.
  5. Complete the fields in the Maintain Filter dialog as required.
  6. In the Criteria section, click New and configure the criteria in the Filter Condition dialog as required.
  7. Click Ok.

Filter onditions

Various filter conditions are available under the Assortment Promotions heading.

Promotion Application

Defines the filter based on the variants included in specific promotion applications:

Condition Description
Equals allows the user to select one from the  promotion applications in the current assortment plan
Contains the Phrase allows the user to specify text contained anywhere within the desired promotion application description
Contains the Phrase at the Beginning allows the user to specify text contained at the beginning of the desired promotion application description
Contains the Phrase at the End allows the user to specify text contained at the end of the desired promotion application description
Is One of the Following allows the user to select many from the  promotion applications in the current assortment plan

promotion application descriptions are system generated in the following format: (Promo Name): (Product Filter Name) / (Cluster Set Name(s)) / (Promo Application Start Date) – (Promo Application End Date). For example, My Test Promo: Mens Graphic Tees / North America Clusters / 2017/03/01 – 2017/06/01.

Promotion Name:

Defines the filter based on all variants included in any promotion application of the selected promotion. The conditions are the same as Promotion Application but with Promotions and not Promotion Applications.

Product Filter

Defines the filter based on all variants contained in the selected product filter for which there are promotion applications. The conditions are the same as Promotion Application but with Product Filters and not Promotion Applications.

Selectable Product Filters are only those that have promotion applications against them.

Cluster Sets

Defines the filter based on all variants assorted to the selected Cluster Set for which there are promotion applications.

Start Date

Defines the filter based on all variants having promotion applications that begin relative to a selected start date.

Condition Description
Equals allows the user to specify a single Start Date for promotion applications to include
Is After allows the user to specify a single date as a starting point; all promotion applications beginning after the date selected will be included
Is After or On allows the user to specify a single date as a starting point; all promotion applications beginning on or after the date selected will be included
Is Before allows the user to specify a single date as an ending point; all promotion applications beginning before the date selected will be included
Is Before or On allows the user to specify a single date as an ending point; all promotion applications beginning on or before the date selected will be included
Is Between allows the user the specify a start and end date as a range; all promotion applications beginning between the start and end dates specified will be included
Is One of the Following allows the user to specify multiple Start Dates for  promotion applications to include

End Date

Defines the filter based on all variants having promotion applications that end relative to the selected end date. The conditions are same as for Start Date but with end dates instead of start dates.

Placeholder is on Active Promotion

Another related Promotions parameter can be used in conjunction with the above to return only items that are on Active promotion applications. This parameter can be found in the (Basic Variant Placeholder Information) section:

To use this parameter, set the Condition to Is then check the single check box that appears below the Condition drop down in the Filter Condition dialog.

If this parameter is NOT used when setting other Promotions related filter criteria, variants included in ALL promotion applications specified by your criteria are returned, regardless of whether those promotion applications are active.
