Promotion Application Section

This section displays information for promotion applications already created for the promotion selected in the Defined Promotions section. Users can create new applications and delete or change existing applications.

  • New: Creates a new Promotion Application line for the promotion selected in the Defined Promotions section.

  • Delete: Removes the selected item from the list of promotion applications.

    Users are not asked to confirm deletion of promotion applications so be certain you want to remove them before clicking Delete.

  • Fields: Opens the Customization dialog. Users can drag fields to or from the tab’s header area and this dialog to customize the view. Custom layouts can be saved or restored by right-clicking in the grid and selecting Save Layout or Restore Layout, respectively.
  • Save: Saves all newly created promotion applications and changes or deletions of existing promotion applications made since the last save. For promotions marked as Active, this applies the promotion as per the specifics in the Promotion Application Detail section.
  • Cancel: Removes all newly created promotion applications and undoes deletions or change of existing promotion applications made since the last save.

Grid Fields

Product Filter: Clicking in this field displays the Items dialog allowing users to select a product filter of placeholders to include in the promotion application.

Selecting the drop down arrow in the Filter field displays a list of filters.

The icon launches the Maintain Filters dialog where users can create new product filters, edit, delete, or export existing product filters, and import product filter files.

Choosing a product filter with products that are not included in Product selection of the Parameters area may cause promotion applications to disappear from the view when the screen is refreshed. To view them again, simply adjust the Product selection to include the Product Filter you chose for the application.

Cluster Sets: The cluster sets for the promotion application.

Any cluster sets related to the Site Hierarchy parameter are available for selection here. However, if the cluster sets chosen are not included in the Cluster Sets parameter, then the promotion application may disappear from the view when the screen is refreshed.

Start Date: The start date (as a period) for promotion application Defaults to the Eligibility Start Date for the promotion or the nearest non-actualized period. This date cannot be set prior to the Eligibility Start Date for the promotion. Start Dates at the Promotion Application Details level cannot be set prior to this date 

End Date: The end date (as a period) for promotion application. Defaults to the Eligibility End Date for the promotion. This date cannot be set after the Eligibility End Date for the promotion. End Dates at the Promotion Application Details level cannot be set after this date

Final Index: The Promotional Index for this specific application of the promotion. Defaults to the Final Index of the selected promotion. Changing the Final Index at this level pushes down and overwrites all placeholder/cluster Final Indices on Apply for the promotion application in the Promotion Application Details section.

Apply to Markdown Periods?: When checked, prevents promotions from being applied to any markdown periods.

Hold Sales: Whether the promotion application has the ability to increase total sales of the placeholder/clusters it affects. If:

  • Checked: (Enabled) the total sales of affected placeholder/clusters remains the same as before the promotion was applied and only magnitude of sales in each period is adjusted.

  • Unchecked: the total sales of the affected placeholder/clusters can increase above the magnitude prior to promotion application. This is effected through recalculation of the placeholder/cluster rates of sale.

: Indicates that only some of the placeholder/clusters included in the promotion application have the Hold Sales option checked.

Checking or unchecking this option at the Promotion Application level pushes the change down to all placeholder/clusters included in the application.

# Placeholders: The number of placeholders to which the application is actually being applied per the Promotion Application Details section.

Location Exceptions: Whether site level exclusions have been made to the promotion application

Location Exceptions Clicking in this field launches the Assortment Promotion Locations dialog. For more information, see Assortment Promotions Locations dialog.

Approximation: Launches the Assortment Promotion Approximation dialog. For more information, see Assortment Promotion Approximation dialog.

Status: The status of the promotion application. Options are:

  • Active: The application is applied to the placeholder/clusters as per the Promotion Application Details section with all related sales plans being effected accordingly.
  • Inactive: Default setting. The application is still saved and ready for application but has not yet be actually applied. It is visible in the Promotions supporting view of Line Visualization but it will not have an effect on the sales plans of the placeholder/clusters shown in the Promotion Application Details section.
