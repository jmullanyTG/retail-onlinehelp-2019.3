Seasonality Curve View

The Seasonality Curve view within Assortment Planning is a scaled down version of the full Curves module found in Execute > Curves. Using the Seasonality Curve view, users can select and apply seasonality curves to the entire selected placeholder/collection or only to selected cluster sets/clusters it is assorted to.

Creation of new curves is still done within the full Curves module.

Seasonality Curve

  • Filter Tags: Selects tags to be used in filtering the curves available for selection in Saved Curves. Tags can be attached to curves within the Curves module. The list of tags can be filtered by typing in the blank field above above the list. Use % at the beginning to do a “Contains” type filter instead of a “Begins With” type.

  • Saved Curves: A curve generated within the Curves module to apply to the selected placeholder/collection. Only curves of the Manual or Specified Conditions Using Enhanced Filtering types are available for selection within Assortment Planning.
  • Curve Mode: The mode of the selected curve:
    • Static: The selected curve is applied exactly as it exists at the time of application and the placeholder/collection’s curve will not change if changes are made to the curve, whether manually in the Curves module or through dynamic recalculation. Think of this as making a copy of the applied curve.
    • Dynamic: Changes made to the curve are pushed down to the placeholder/collection via automated system processes and the dependent sales plans recalculate when those changes are pushed.
  • Apply To Cluster: The cluster sets/clusters to apply the selected curve to from those assorted to the selected placeholder/collection
  • Apply Lowest Level Curve: The level of curve to be applied:
    • Unchecked: Applies the curve exactly as shown to all selected cluster sets/clusters.
    • Checked: Looks for related lower level curves to apply at the cluster set/cluster level and applies them instead of applying the higher level curve as shown.
      • During implementation, a system process is configured to auto-generate curves at levels of the product/site hierarchies on a specified schedule. If desired, this process can be configured to generate these curves at multiple levels.
      • “Related lower level curves” refers to these auto-generated curves – created using the same criteria

You choose an auto-generated channel level curve from the Saved Curves drop down and check the Apply Lowest Level Curve option; the selected placeholder has 2 cluster sets, each with 3 clusters, assorted to it, and curves were also auto-generated using the same criteria for each of those 6 clusters; the system will apply the cluster-level curves to each cluster in a single action, saving you the effort of applying the lower level curves individually.

Curve Details section

  • Generation Type: The generation type for the selected curve. Only curves of the Manual or Specified Conditions Using Enhanced Filtering types are available for selection within Assortment Planning.
  • Bucket Type: The bucket type for the selected curve. For the purposes of Assortment Planning, bucket type is the same as period type.
  • Time Range: The time range used to generate the selected curve.
  • Forecasted Demand: If configured, generates curves based on projected future demand from the Forecasts module.
  • Curve Source: The curve source for the selected curve. Historical Demand By Sales Type Using Sales History generates curves based on historical sales by type
  • Generate By: The generation type for the selected curve. This is essentially the metric used to calculate the seasonal indices.
    • Volume Contribution (Units) – sales units
    • Cost of Sales Contribution – COGS (cost of goods sold)
    • Revenue Contribution – sales $
    • Profit Contribution – gross margin $ (sales $ - COGS)
  • Sales Types: The sales types used in generation of the selected curve. Not applicable for Forecasted Demand Curve Source.
  • Includes Returns: Whether returns (negative sales) were included in the calculation of the selected curve.
  • Inactive Items: Whether inactive items were considered in the calculation of the selected curve.

Product Filtering section

The fields shown depends on the options used to generate the selected curve.

  • Hierarchy: The product hierarchy used in generation of the selected curve.
  • Hierarchy Items: The product hierarchy levels used in generation of the selected curve.
  • Filter: The product filter used in generation of the selected curve.

Site Filtering section

The fields shown depends on the options used to generate the selected curve.

  • Hierarchy: The site hierarchy used in generation of the selected curve.
  • Hierarchy Site: The product hierarchy levels used in generation of the selected curve.
  • Filter: The site filter used in generation of the selected curve.
  • Cluster Sets:The cluster sets used in generation of the selected curve.
  • Cluster: The clusters used in generation of the selected curve.
