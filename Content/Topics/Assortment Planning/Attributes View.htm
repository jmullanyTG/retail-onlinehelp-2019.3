Attributes View

The Attributes view is displayed or hidden in Placeholder mode using the View Selection drop down menu. The Attributes view allows users to view and manage product attributes for placeholders/collections.

For more information on Attribute Families, Values, and how their eligibility is set for items within the product hierarchy, see Attribute Management and Assignment Overview.

Data Population

The view populates with all attribute families that any of the selected placeholders/collections is eligible for. Attribute values available for selection include all values that any of the selected items is eligible for. Attribute family and value eligibility can be set via Maintenance > Attribute Management.

Applying Changes Across Multiple Items

Whether multiple lines are selected in the Main Planning view, changes are being applied in Collection mode, or both the system automatically resolves any conflicts that exist between the Applied values and the attribute value eligibility of the items the change is being applied to.

Single-select attribute value changes are applied to all placeholders/collections that are eligible for them. Ineligible placeholders/collections remain unchanged.

Multi-select attribute value changes are applied additively to placeholders/collections that are eligible for them. If only some of the applied values are eligible, then those will be applied. Ineligible placeholders/collections will remain unchanged.

Dual-directional Attribute Flow

Depending on where attributes are assigned – to placeholders in Assortment Planning or to variants in Variant Manager – attributes are transferred when a variant is assigned to a placeholder

A system policy exists that governs the inheritance of multi-value attribute families when variants are assigned. The system policy is Maintenance > Policies > Module: Assortment > Multi Value Attribute Inheritance. When set to FALSE, multi-value families are applied additively to existing values when variants are assigned.

A placeholder has the multi-value family Embellishments and the values Flocked and Sequinned currently set. If a user assigns a variant with the Embellishments values Flocked and Embroidered to that placeholder, then the resulting values for the Embellishments family are: Flocked, Sequinned, and Embroidered.

Assigning a variant

Any product attributes attached to the variant are transferred to the placeholder; any attributes that were already assigned to the placeholder but that don’t conflict with the variant attributes (i.e. single-select families that the variant already had assigned), they will remain on the placeholder and also be transferred to the variant.

When set to TRUE, multi value families will overwrite the existing values when variants are assigned.

Using the same case as the example above, the resulting values for Embellishments would be: Flocked and  Embroidered only.

Attribute changes in Assortment Planning

System policy Update Variant Attributes In Assortment Planning governs whether changes are allowed to attribute families attached to variants. This policy can be found in Maintenance > Policies > Module: Assortment. When set to FALSE, all attribute families attached to the assigned variant will appear grayed out and will be locked for editing.

Notice the Heel Height family in the screenshot is still editable. Creating attribute families that only have the entity type Variant Placeholder assigned allows additional attribute families that cannot be attached to variants and will only be used inside of Assortment Planning for placeholders.

When set to TRUE, attribute families can be locked/unlocked individually within the Attribute Management solution. Families assigned to the variant entity type and having the AP Updates Variants setting checked are editable in Assortment Planning even when variants are assigned. Changes made to these families in Assortment Planning are transferred to the assigned variant for the product hierarchy assigned to the active assortment plan but are not transferred to other placeholders that variant is assigned to.

For more information on AP Updates Variants setting, see Attribute Management and Assignment Overview.

Attribute changes in Variant Manager

Changes made to variant attributes in Variant Manager are transferred to all placeholders that variant is currently assigned to across all existing assortment plans that have the selected Variant Hierarchy assigned.

Field descriptions

Field Description
Family Name The name of the attribute family.
Family Description The description for the attribute family.
Single Values

The attribute value of the adjacent single-select attribute family for the selected placeholder/collection.

This field displays (Multi) to indicate when multiple placeholders/collections are selected that have different attribute values assigned.
Multiple Values

The attribute values of the adjacent multi-select attribute family for the selected placeholder/collection.

This field displays the combination of all values present when multiple placeholders/collections are selected that have different attribute values assigned.

Buttons

Save: Saves all changes made to attribute values for the selected placeholder/collection.

Cancel: Cancels all changes made to attribute values for the selected placeholder/collection and reverts them to previously saved values.
