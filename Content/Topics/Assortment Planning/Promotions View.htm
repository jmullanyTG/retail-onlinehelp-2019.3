Promotions View (not available in Collection mode)

The Promotions view gives users visibility to the various promotions, whether Active or Inactive, that have been applied to the selected placeholder or collection within the plan. Users can adjust start and end dates for applied promotions.

The creation and management of promotions is done in the Promotions area of Assortment Planning. For more detailed information on promotional applications and the effects they have on the sales plan, see Promotions.

Field descriptions

Field Description
Promotion Name The name of the promotion.
Behavior

The stacking behavior of the promotion:

  • Stacking: all stacking promotions that are active in a given period combine their effects. Sstacking is always applied in order depending on the type of AUR effect assigned to each: New AUR, % Off, $ Off.
  • Non-Stacking: non-stacking promotions operate on best-deal logic. The system compares the resulting AURs of applied promotions and applies the one with the lowest AUR.
  • Supersede: the Supersede behavior is only valid for Global promotions. When set, supersede promotions are applied in place of any others applied during a given period regardless of whether their resulting AURs are lower.

AUR Effect Displays the defined effect to AUR for the promotion:New AUR, % Off, or $ Off and the value or magnitude associated with each.
Global? Whether the promotion is a global promotion. Global promotions are typically used for storewide promotions like “30% everything in stock except ______.”.
Application Status The status of the promotion application: Active or Inactive.
Start The start date of the promotion application for the selected placeholder/collection. Selections are limited to periods before End and within the promo application time frame defined in Promotions.
End The end date of the promotion application for the selected placeholder/collection. Selections are limited to periods after Start and within the promo application time frame defined in Promotions.
Hold Sales

Whether the promotion has the ability to increase total sales of the placeholder/cluster:

  • Checked: the total sales of the placeholder/cluster remain the same as before the promotion was applied, and only magnitude of sales in each period is adjusted.
  • Unchecked: the total sales of the placeholder/cluster can increase above the magnitude prior to promotion application. This is effected through recalculation of the placeholder/cluster rate of sale.
Final Index Decimal (1.10) The promotional index to apply to the placeholder/cluster.
Included? Whether the promotion is applied to each the placeholder/cluster or not.

Grid buttons

Button Description
Fields Opens the fields Customization dialog; users can drag fields to and from the promotions column header area and this dialog to customize the view. Custom layouts can be saved or restored by right-clicking in the Snapshot Templates area and selecting Save Layout or Restore Layout.
Save Saves changes made to start and end dates since the last Apply and recalculates the placeholder sales plan
Cancel Cancels changes made to start and end dates since the last Apply and reverts them to their previously saved values
