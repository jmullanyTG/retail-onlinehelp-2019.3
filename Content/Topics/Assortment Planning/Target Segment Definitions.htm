Target Segment Definitions field descriptions

Target segments can be defined to subdivide the product and site hierarchy levels selected for your scenario in a variety of ways, from the most common scenario of creating segments by lower hierarchy levels to the more complex scenarios where segments are created based on criteria, such as product attributes, that may cross traditional hierarchy levels.

When creating segments as needed with Target Filters, or when using a Plan Basis plan whose hierarchies don’t directly align with those in the active assortment plan, it is advised to use the View Exclusions function found on the top toolbar after defining your segments to identify any products that are part of the scenario hierarchy selections but that were inadvertently excluded.

The flexibility of target segment definitions allows for scenarios in which the resulting segment cluster details may not add up to 100% of the hierarchies selected for the scenario – instead equaling more or less than 100% depending on how your segments are defined.

  • Target Segment: User or system defined name for the target segment. Segment By Hierarchy defaults segment names to the corresponding product hierarchy level description.
  • Target Filter: Product filter selection to define the items included in the segment. The filters used here are Placeholder Filters. If you define a new Placeholder Filter while already working in a scenario, you must click Execute on the Target Creation dialog again to update the values in the drop down.
  • Supplemental Filter: Product filter selection to define the items used in Rationalization only. Setting a Supplemental Filter does not supersede the list of items used to return historical data for the segment in the Segment Cluster Details section. It is used only in the calculation of the marginal return curve used for choice count recommendations. See Rationalization section below for more information.

  • Target Filter Generated?: Using Segment By Hierarchy to define segments checks this box, which indicates that the system will use a temporary filter based on the product hierarchy level corresponding to the segment in place of a standard Target Filter.

    Subsequently selecting a Target Filter after using Segment By Hierarchy will uncheck this box and the system will revert to using the Target Filter selected for calculation of segment data. The only way to re-check it is to click Segment By Hierarchy again, thus recreating the segment.

  • Cluster Sets: Selects the cluster sets to define the site groupings for the segment in the Segment Cluster Details section. Cluster sets selected will also be those initially eligible to be assorted placeholders generated for the segment through the Smart Start process.

    Using Segment By Hierarchy or Applying Target Segment Definitions with no cluster sets selected will default this to the cluster set(s) defined in Assortment Plan Defaults (see Assortment Plan Defaults above) for the assigned segment Product Hierarchy, provided they have been defined.

    Because cluster set configurations can change over time in Assortment Planning, the system uses the current Cluster Set Snapshot to determine site cluster assignments when calculating historical data used in the Segment Cluster Details section.

    If the cluster sets selected don’t encompass all sites within the scenario Site Hierarchy selection, the data returned will only represent sales volume which occurred in the selected cluster sets.

    Because it is possible to have multiple cluster sets with valid configurations that include some or all of the same sites, when assigning cluster sets to segments, the system checks to make sure overlapping cluster sets have not been assigned.   • Comparison Time Frame: Sets the comparison time frame from which to gather historical data for the segment.

    Using Segment By Hierarchy for segment creation or Applying Target Segment Definitions without setting a Comparison Time Frame defaults this to LY, an un-selectable system identifier indicating the last year periods directly correlating to the scenario Window of Time will be used.

  • Product Hierarchy: Sets the product hierarchy level to which placeholders generated from the segment through Smart Start will be assigned. Segment By Hierarchy defaults this to the same product level as indicated by the segment Name.

    For manually defined segments, when a user clicks Save this setting defaults to the lowest common level of items returned in the Target Filter. For example, if a filter defined to return all apparel products that are part of the color group Red, assuming there are items belonging to this color group in each of the lowest product levels below Apparel, Product Hierarchy will be set to Apparel, the lowest common level.

    This setting does not have to be set to a leaf of the lowest level of the assortment plan Product Hierarchy; if you aren’t sure at the time of scenario creation where all the placeholders will be ultimately assigned, you can choose a higher level and change by placeholder later in the assortment planning process.

  • Collection Option: Defines the configuration of placeholders generated for the segment through Smart Start within collections.
    • Multiple Variants per Collection (default): allows the system to optimally assign placeholders to collections based on the results of Rationalization. This yields the most common “Style-to-Style-Color” configuration (ie. A Crew Neck Tee in 5 colors).
    • All Variants in One Collection: All placeholders generated for the segment through Smart Start will be assigned to a single collection, using the results of only the Intra-Style Marginal Return Curve (see Rationalization below) to determine rates of sale for each placeholder.
    • One Collection per Variant: Placeholders generated for the segment through Smart Start will each be assigned to their own collection, using the results of only the Inter-Style Marginal Return Curve (see Rationalization below) to determine rates of sale for each placeholder.
  • Refresh: Refreshes the items associated with the Target and Supplemental Filters based on their criteria.
  • Last Filter Process Date: Displays the date of the last filter Refresh for the segment. Using Segment By Hierarchy or on Apply of newly created segments, defaults to the current date.

  • New Variants: Sum of the Max Act TY New Choices for all segments within the scenario. Defaults to blank. Updates after scenario has been Rationalized or TY Act New Choices has been updated, Applied, and the scenario has been refreshed.

    All new choices within a single segment are assumed to be shared among all assigned clusters.

    For example, for a scenario with 2 segments, both with a single cluster set with 3 clusters assigned, the first segment has TY Act New Choice counts of 5, 3, and 2, and the second segment has TY Act New Choice counts of 4, 2, and 1, New Variants will equal 9 (max of 5 from segment one and 4 from segment two). This means that the clusters receiving fewer choices will still share from the max new choices by segment, they just won’t receive all of them.

  • Total Variants: Sum of the Max Act TY New Choices and the Max TY C/O Choices for all segments within the scenario. Defaults to blank; updates after scenario has been Rationalized or TY Act New Choices has been updated, Applied, and the scenario has been refreshed.
  • Active Variant Threshold: Percentage (10.00%). Defaults to the value of the Active Variant Threshold Factor system policy. Sets the percent of deseasonalized average to be used in step 3 of the data cleansing process for the segment. Changes to this value are valid within the active scenario only and will not alter the system policy value.
  • Product Hierarchy Path: Displays the full product hierarchy path of the Product Hierarchy selection for the segment. Used for specific identification of product levels in hierarchies where multiple levels may share the same description.
  • Clear Supplemental Filter?: If checked, then when the user clicks Save this clears the Supplemental Filter set for the segment.

Toolbar buttons

    New: Creates a new target segment line in the Target Segment Definitions section
  • Supplemental Filter defaults to (none) and Collection Option defaults to Multiple Variants Per Collection; all other fields are blank by default
  • Delete: Deletes the selected target segment(s)
  • Fields: Launches the customization dialog.

  • Save: saves changes made to Target Segment Definitions parameters
  • Cancel: reverts changes made to the Target Segment Definitions parameters since the last Apply to their previous values
