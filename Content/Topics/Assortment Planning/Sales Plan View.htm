Sales Plan View

The Sales Plan view allows users to make period level edits to the sales plan of the selected placeholder/collection, either in total or by average store and compare them to the high level plan created in the Main Planning view of Line Visualization. Users can also track inventory and plan shrink and adjustments for more detailed control both pre and in-season.

Toolbar buttons

Button Description
Clear Sales Plan Overrides

Reverts all user metrics back to the values of their System counterparts; this also reinstates all periods as valid Selling Periods (see Selling Period description below).

Metrics without system counterparts (the Shrink and Adjustments metrics, for example) cannot be reverted using this function. To revert a single cell to the value of its system counterpart,select the desired cell, press the Delete key, then click Apply.
Parameters Shows or hides the parameters pane at the top of the view.
Execute

Refreshes the view.

This is not typically necessary, as the view refreshes on its own when a new placeholder/collection is selected in the main planning view, when changes are applied, and when the Comparison Snapshot or View Mode are changed
Save Saves all changes made since the last save.
Cancel Cancels all changes and reverts them to their previously saved values.
Export Exports and saves the Pivot Grid in many common formats including Microsoft Excel and Adobe PDF.
Open Opens the Pivot Grid in many common formats including Microsoft Excel and Adobe PDF. Does not require save prior to viewing in the desired format.
Graph Graphs the selected data in the view.
Fields Allows users to customize the field layout for the view. Custom layouts can be saved and restored by right-clicking in the Pivot Grid and selecting Save Layout or Restore Layout, respectively.

Fields

Field Description
Comparison Snapshot A snapshot all saved snapshots within the current plan that are relevant to the current Product and Site Hierarchy selections in the Main Planning view parameters
View Mode:

The view mode for sales plan view: Total Company or Average Store.

  • Total Company: allows the user to view/plan revenue and unit metrics as total company values – meaning the total volume for all sites contained in whatever level the Pivot Grid is aggregated to.
  • Average Store: allows the user to view/plan revenue and unit metrics as average store values – meaning the per site volume for the number of sites contained in whatever level the Pivot Grid is aggregated to.

Metrics Notes

  • System: Denoted by the word System at the beginning (for example: System Sales) and green row coloration. System metrics are those based on the values shown in the Main Planning and Cluster Set views of Line Visualization.
  • User: user metrics default to the System metrics of like name, but these are based on edits made in Sales Plan view to the editable metrics, thus, the user plan can diverge from the system plan in Sales Plan view

  • Ex VAT Metrics: Denoted by ex VAT appended to the end of the metric name (for example, System Sales ex VAT). Where VAT (value added tax) is in play, ex VAT metrics provide accurate profit calculations (margin) after taxes have been removed.

    Where VAT is not in play, no customization of calculations is necessary. Simply go to System Utilties > Views > Maintenance > Planning VAT Rate and set the VAT rates for all levels of the product hierarchy assigned to your assortment plan to 0. This sets the ex VAT metrics equal to the non ex VAT metrics.

  • Comparison Metrics: Denoted by purple row coloration and (Comparison) appended to the end of the metric (for example, Sales (Comparison). These metrics are direct data reads from the selected Comparison Snapshot and default to zero if no snapshot is selected or no data is present in the snapshot.
  • Editable Metrics: Denoted by light blue row coloration. These are the only metrics that are editable by the user in Sales Plan view. All others are calculations or reads from other sources:
    • * System, ex VAT, and Comparison metrics are never editable in Sales Plan view

    • with the exception of Selling Period, edits made in Sales Plan view DO NOT change the plan values in the Main Planning and Cluster Set views of Line Visualization – this is by design to allow for comparison of original plans with period level changes. You can, however, bring the SPV Sls $ and Sls U metrics into your Main Planning view layout to view these edits at placeholder/collection or cluster set/cluster level

  • Calculations: Calculations in Sales Plan view all work based on one-directional locking calculations, depending on the metrics changed.

    For more information on the calculation model used in this view, see the supplemental Microsoft Excel file Assortment Planning – Sales Plan View.

  • Plan Horizon vs. Receipt Planning Engine Horizon: If the Plan Horizon, set at the time of plan creation, extends beyond the receipt planning engine horizon for a given item, data that extends past the receipt engine horizon appears in blue italics with zero values ($0.00, for example). See example in Plan Horizon vs. Replen Forecast Horizon below).

  • Replen Placeholders/Collections: When a placeholder/collection is of the Status Replen, the system pulls all relevant data to form its sales plan from the Forecasts module.

    Users should be aware of the following differences in this view when dealing with Replen items: Read Only: all normally editable metrics in Sales Plan view become read only, being populated directly from aggregated replenishment forecast data, with the exception of all Shrink and Inventory Adjustments related metrics, which remain editable and can still be planned normally.

  • Plan Horizon vs. Replen Forecast Horizon: If the Plan Horizon, set at the time of plan creation, extends beyond the replenishment forecast horizon for a given item, the system will conditionally format the periods that go beyond the replenishment forecast as shown below to draw the user’s attention to this. Selling Period check box

The Selling Period check box allows the user to specify periods where sales are not planned for the selected placeholder/collection but is still in phase. If unchecked, then zeros out System Sales, System Sales Units, Sales, and Sales Units for the period. Thus, metrics in the Main Planning and Cluster Set views are updated by this change. Users cannot enter sales for unchecked periods. When unchecked:

  • inventory (BOP, EOP) still calculates normally for the period
  • receipts can still be realized for the period
  • all Inventory Adjustments metrics (see below) can still be planned normally

Common uses for this feature might be items that repeat seasonally, such as holiday themed goods, where inventory in the items is retained between selling periods.

Metric definitions

Seasonality

  • Seasonal Index: Decimal (1.01); seasonal index for the period based on the assigned seasonality curve
  • Promotion Index: Decimal (1.01); combined promotional index for all applied promotions in the period
  • System Effective Index:Decimal (1.01); index applied to period rate of sale to calculate period sales
  • Sellthrough: Sales / BOP as a percentage.

Sales

  • Sales: Currency ($100); planned/actual sales revenue for the period.

    Calculation: (Sales Units * AUR)

  • Sales Build Ratio: Decimal (1.01); Ratio indicating the relationship of Sales of the current period to the previous period.
  • Sales Units: Planned/actual sales units for the period.

    Calculation: (Sales / AUR)

  • Sales Units Build Ratio: Decimal (1.01); Ratio indicating the relationship of Sales Units of the current period to the previous period

    Calculation: (Sales Units / Previous Period Sales Units)

  • AUT: Currency ($1.01); average unit ticket price for the period.

    Calculation: N/A

  • AUR: Currency ($1.01); average unit retail price for the period.

    Calculation: (Sales / Sales Units)

  • Sellthrough: Percentage (1.01%); percentage of inventory sold for the period.

    Calculation: (Sales Units / BOP Units)

Cost of Sales

  • COGS: Currency ($100); cost of goods sold for the period
    • Calculation: (Sales Units * AUC)
  • AUC: Currency ($1.01); average unit cost price for the period
    • Calculation: (Avg BOP Unit Cost)

Margin

  • Margin: Currency ($100); planned/actual margin revenue (profit) for the period
    • Calculation: (Sales ex VAT - COGS)
  • Margin % Percentage (1.01%); planned/actual margin percentage for the period
    • Calculation: (Margin / Sales ex VAT)

BOP Inventory

  • BOP Units: Integer; planned/actual beginning of period inventory units
    • Calculation: (Previous Period EOP)
  • BOP Cost: Currency ($100); planned/actual beginning of period inventory cost
    • Calculation: (Previous Period EOP Cost)
  • Avg BOP Unit Cost: Currency ($1.01); average beginning of period cost per inventory unit
    • Calculation: (BOP Cost / BOP Units)

Receipts

: Receipt metrics are not editable in Sales Plan view because they are managed in greater detail in Receipt Plan view.

  • Receipt Units: Integer; planned/actual receipt units for the period
    • Calculation: From Receipt Plan
  • Receipt AUC: Currency ($1.01); average cost per unit for receipts in the period
    • Calculation: From Receipt Plan
  • Receipt Cost: Currency ($100); planned/actual cost of receipts for the period
    • Calculation: (Receipt Units * Receipt AUC)
  • Minimum Presentation: Integer; Minimum units required to maintain floor presentation for each site selling the item
    • Calculation: From Receipt Plan

On Order

  • On Order Units: Integer; confirmed units on order for the period
    • Calculation: (purchase order units not yet received but scheduled for the period)
  • On Order Units var to Receipt Units: Integer; Difference of On Order Units to the planned/actual Receipt Units
    • Calculation: (On Order Units – Receipt Units)
  • On Order AUC: Currency ($1.01); average cost per unit of inbound orders for the period
    • Calculation: (On Order Cost / On Order Units)
  • On Order Cost: Currency ($100); cost of inbound orders for the period
    • Calculation: N/A
  • On Order Cost % +/- Receipt Cost: Percentage (1.01%); percentage of On Order Cost to planned/actual Receipt Cost
    • Calculation: (On Order Cost / Receipt Cost)

Stock To Sales

  • Stock Cost to COGS: non-editable decimal (1.01); ratio expressing the relationship of BOP Cost to COGS
    • Calculation: (BOP Cost / COGS)
  • Stock Units to Sales Units: non-editable decimal (1.01) ; ratio expressing the relationship of BOP Units to Sales Units
    • Calculation: (BOP Units / Sales Units)

Inventory Adjustments

  • Shrink Units: Integer; planned/actual shrink units for the period
    • Calculation: (Shrink Units % of Sales Units * Sales Units)
  • Shrink Units % of Sales Units: Percentage (1.01%); percentage of sales units represented by shrink units for the period
    • Calculation: (Shrink Units / Sales Units)
  • Shrink Cost: Currency ($100); cost value of planned/actual shrink units for the period
    • Calculation: (Shrink Cost % of COGS * COGS)
  • Shrink Cost % of COGS: Percentage (1.01%); percentage of COGS represented by Shrink Cost for the period
    • Calculation: (Shrink Cost / COGS)
  • Inventory Adjustment Units: Integer; planned/actual inventory adjustment units for the period
    • Calculation: N/A
  • Inventory Adjustment Cost: Currency; planned/actual cost of inventory adjustments for the period
    • Calculation: N/A

EOP Inventory

  • EOP Units: Integer; planned/actual end of period inventory units
    • Calculation: (BOP Units – Sales Units + Receipt Units – Shrink Units + Inventory Adjustment Units)
  • EOP Cost: Currency ($100); planned/actual end of period inventory cost
    • Calculation: (BOP Cost – Sales Cost + Receipt Cost – Shrink Cost + Inventory Adjustment Cost)
  • Avg EOP Unit Cost: Currency ($1.01); average end of period cost per inventory unit
    • Calculation: (EOP Cost / EOP Units)

VAT

  • VAT: Currency ($100) value added tax value of planned/actual Sales for the period
    • Calculation: (Sales - (Sales / (1 + VAT %)))
  • VAT % Percentage (1.01%) value added tax rate of the planned/actual Sales for the period
    • Calculation: (VAT / Sales ex VAT)
