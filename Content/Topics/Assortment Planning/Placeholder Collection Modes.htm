Placeholder & Collection Modes

Line Visualization is where you will do the majority of your work in Demand Management Assortment Planning. The Assortment Planning module provides access to core functionalities through expandable side and bottom pane views that can be quickly configured on one screen or multiple and then saved as workspaces for quick recall later. This allows users to always maintain visibility to the core of their plan, including product images and see the effect of changes at different levels of the plan as they are made.

The Placeholder mode and the Collection mode can be accessed via the Assortment Planning toolbar.

Toolbar options

The following options are available on the toolbar:

New: launches the Add Placeholders dialog (not available in Collection mode). See, Add Placeholders dialog.

Hindsight: (not available in Collection mode) For more information on the Hindsight functionality, see Hindsight Placeholder Management..

Split: (not available in Collection mode) launches the Split Placeholders dialog. For more information, see Split Placeholders dialog.

Delete: deletes all selected placeholder/collection rows. To undo, use Undo > Deletions rather than Cancel. Deletions are permanent when you click Save. Committed placeholders and collections cannot be deleted.

Undo:

  • Selected Row Changes: undoes changes made prior to Apply for the selected rows only. Use this when you have made changes to multiple rows but only want to undo certain rows – using Cancel will undo all changes made since the last Apply
  • Deletions: undoes all deletions of placeholder/collection rows made since the last Apply. Use this method instead of clicking Cancel to undo row deletions

  • Unassign Selected: unassigns variants attached to the selected placeholder rows

    • Not available at header level in Collection mode – use in Placeholder mode or Variant view of Collection mode
    • If any product attributes were attached to the variants being unassigned, system will revert the overwritten attribute values back to their pre-variant assignment states
    • You cannot unassign variants from placeholders in Committed status

Locations: launches the Manage Locations window (not available in Collection mode). use this dialog to exclude specific sites from clusters to which a placeholder is assorted and adjust their sales plans accordingly. For more information, see Manage Locations dialog.

View Options

Option Description
Column Chooser Opens the Customization dialog. Users can drag fields to or from the main planning area and this dialog to customize the view. Custom layouts can be saved and restored by right-clicking in the Snapshot Templates area and selecting Save Layout or Restore Layout, respectively.
Side Pane

Choose which side pane you want to view. (You cannot have both open at the same time.)

  • Cluster Details: shows the Cluster Details pane at the right of the screen
  • Lifespan: shows the Lifespan pane at the right of the screen
  • None: Hides the side pane
Image Tooltip This option toggles whether image tooltips appear when the user hovers the mouse over a thumbnail image in the main view (not available in Collection mode).

Receipts

  • Recalc Req. %: displays the % of placeholders/collections in the current view that require receipt plan recalculation based on changes that have been made. This value is directly tied to the Receipt Status field in the main view; placeholders/collections with a Receipt Status of “Dirty” require recalculation.

  • Recalculate: recalculates receipt plans for all placeholders/collections with a “Dirty” status in the Receipt Status field. Receipt plan default constraints can be set via Manage Assortment Plans > Defaults. System policies affecting the Assortment Engine output can be set via Maintenance > Policies > Engine: Commitments. For more information see, Line Visualization Supporting Views.

Receipt recalculations require complex calculations and, thus, are sent to the Workload Queue module for scheduling and processing. Though, often, receipts are recalculated immediately, they may be deferred for later calculation based on the number of placeholders requiring recalculation and other jobs currently in the queue. Placeholders awaiting recalculation will become locked and conditionally formatted to indicate this.

Receipts for placeholders/collections in Replen status are calculated in the Forecasts module and read in, thus they will never appear “Dirty” and will not be recalculated when this button is clicked.

All changes to your plan, including changes made to constraints in the Receipt Plan supporting view, must be saved prior to recalculating receipts or they will not be taken into account by the Receipt Engine.

Replen

Recalculate: recalculates sales plans for placeholders/collections of Status “Replen” with a “Dirty” status in the Replen Status field. The date Replen placeholders/collections were last recalculated is shown in the Last Replen field.

Replenishment recalculations require complex calculations and, thus, are sent to the Workload Queue module for scheduling and processing. Though replenishment placeholders are frequently recalculated immediately, they may be deferred for later calculation based on the number of placeholders requiring recalculation and other jobs currently in the queue. Placeholders awaiting recalculation are locked and conditionally formatted to indicate this.

Health %: Indicates what % of an assortment has issues that need to be addressed. More details are provided in the Health Error and Health Error Details columns in the Main Planning View.

Managing Workspaces

Use the Workspaces field to manage your Workspaces.

multi-screen workspace configurations can be saved as well as single screen ones

Reset:reverts the workspace configuration to the system default for the view

Save:  launches the Manage Workspaces dialog, allowing users to save the current workspace configuration for later recall

Requires a workspace name to be entered in the Workspaces drop down

Field or option Description
Name sets the name for the workspace
Description sets a description for the workspace; not required
Family selects a workspace family in which to save the workspace
Protection Type sets the permissions for the workspace – Public (all users can access) or Private (only the user who created the workspace can access)
Cancel closes the Manage Workspaces dialog without saving the workspace
Apply saves the current workspace with the specified parameters

Edit:  launches the Manage Workspaces dialog as with Save but locks the workspace name

Delete: deletes the current workspace

The system does not ask the user to confirm deletion of workspaces, so be sure you want to do so before clicking this

Execute: refreshes everything in the current view, including all supporting views, based on the selected Product, Site Hierarchy, Cluster Sets, and Window of Time
