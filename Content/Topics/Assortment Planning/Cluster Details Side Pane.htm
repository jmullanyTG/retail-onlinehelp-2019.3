Cluster Details Side Pane

The Cluster Details side pane allows users to view/edit the assortment of placeholders/collections to clusters.

  • While in Placeholder or Collection mode, you can open the Cluster Details side pane by:
    • Selecting View Options > Side Pane > Cluster Details, or
    • Right-clicking the grouping section of the main planning view and selecting Show Cluster Details.

    Editing cluster assignments

    The side pane contains a column group for each cluster set selected the Cluster Sets parameter. Under each group is a column for every cluster in that cluster set.

    Because not every cluster shown in the Cluster Details side pane is available to each placeholder/collection, the fields in each placeholder/collection row are color-coded to indicate whether or not a cluster is available to that placeholder or collection.

    Color Cluster Set Description
      Available
      Not available to this product level
      Not available because sites in this cluster overlap with one or more already-selected clusters

    To edit a placeholder/collection's cluster assignments

    1. Select the placeholder or collection row.

    2. Click in a cluster column to toggle assigning or unassigning that cluster.

      Use the All column under each cluster set group to toggle assgning and unnassigning all clusters in the cluster set.

    3. Click Save when you are finished editing.

    Notes

    • Placeholders generated by Smart Start are assorted to clusters per the logic described in Smart Start.
    • Placeholders/collections generated using the Add Placeholders dialog are assorted to all cluster sets they were assigned to when added.
    • Assorting a placeholder/collection to a cluster it was not previously assorted to will not automatically create a sales plan for that cluster; you can create one through manual entry of metric values and/or use of the Rate of Sale Calculator. See Supporting Views - Rate of Sale Calculator or Rate of Sale Calculator help for more information

    Clusters that are unassorted from placeholders/collections still retain their sales plans in the database – they are just zeroed out in the user interface for calculation purposes – and they will be reinstated if the cluster is reasserted to the placeholder/collection. This enables users to quickly try different assortment strategies without losing work done previously.
