Assortment Upcoming Commitments Report

The Assortment Upcoming Commitments report allows users to easily track key milestones in the commitment process for orders created from assortment plans and quickly navigate to those orders to complete the next required steps. 

Planned receipts that have not yet had commitments created for them will still appear in this report. However, certain fields may be blank and the Due Date will calculate based on the intended receipt date and the Default Lead Time Days assigned to the corresponding product.

Importance of the Assortment Upcoming Commitments Report in keeping track of created commitment orders

The Assortment Upcoming Commitments report is essential to keeping track of created commitment orders and ensuring that the activities required to provide sufficient information to suppliers are completed on-time, thus ensuring on-time delivery of products.

Efficient use of this report involves setting the Window of Time parameter to include a period from the current week to a number of weeks out based on how long ordering activities such as Break By Size and Prepack analysis normally take.

If managerial approvals are required in your organization prior to submitting orders, then the Window of Time parameter selected should be increased to include the amount of time anticipated for review and any required changes.

Parameters

The Assortment Upcoming Commitments report is controlled by four parameters:

  • Product Hierarchy: Selects the levels of current plan’s Product Hierarchy levels to include in the current view.
  • Site Hierarchy: Selects the levels of the current plan’s Site Hierarchy to include in the current view.

  • Window of Time: Selects the window of time to display in the current view.

Order documents are returned based on associated milestone due dates that fall within the selected window. Multiple future due dates may exist for an order but if any of them fall within the selection then the due date displayed is the one nearest to the current date.

  • Display Currency: Sets the currency to display for all revenue based fields in the view; defaults to the Home Currency for the active Assortment Plan.

Fields

Clicking Execute returns the following read-only columns once the parameters have been set:

  • Order Header Name: Thename of the order document associated with the listed due date. Blank for receipts that have not yet had commitment orders created for them.
  • Product Code: The variant or placeholder codes associated with the corresponding order document.
  • Product Description: The description for the variant or placeholder associated with the corresponding order document.
  • Receipt Units: The total receipt units for the corresponding products.
  • Receipt AUC: Currency such as $5.00. The average unit cost of the receipt units for the corresponding products.
  • Receipt Cost: Currency such as $5,000.00. The total cost of receipt units for the corresponding products Calculated as Receipt AUC * Receipt Units.
  • Order Created: Whether a commitment order has been created for the corresponding receipt.
  • Order Date: The date the corresponding order document was created. Set to the Due Date if no order has been created.
  • Delivery Date: The date the corresponding order is due for delivery.
  • Required Action: The nearest milestone based on the assigned workflow that is due within the selected Window of Time for the corresponding order.

  • Due Date: The date the corresponding Required Action is due based on the workflow assigned to the order document. For receipts without created commitments, based on the Default Lead Time Days of the corresponding product.

    Due dates that are in the past are conditionally formatted for easy identification. This does not mean the listed Required Action can no longer be completed. However, failure to meet milestones may adversely effect on-time delivery of orders.

  • View Order: clicking this button automatically launches Order Manager and displays the corresponding order document for review/action

    Clicking this button for receipts that have not yet had orders created (ie. Order Created check box is blank) will do nothing.
