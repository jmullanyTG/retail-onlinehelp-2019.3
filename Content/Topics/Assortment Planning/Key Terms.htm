Assortment Planning Glossary

Term Definition
Cluster A group of stores defined by user established criteria within Cluster Site Manager
Cluster Set A collection of clusters
Cluster Set Snapshot The configuration of stores within the clusters of a cluster set at a specific point in time
Cluster Site Manager The Demand Management module in which users can create cluster sets and clusters
Collection A group of placeholders that can be planned as a single product; these are often referred to as styles or key items
Curves Module A supporting module within Demand Management that is used to model seasonality curves given a set of user defined inputs
Merchandise Financial Plan A planned set of financial, inventory, and other targets by period, typically created at a level of the Product Hierarchy above Style and at a level of the Site Hierarchy above Site/Store; can be from Demand Management MFP or external planning systems
Period The smallest unit of time being used within the assortment plan; typically, period represents a single week. However, it is also possible to plan in months or other periods as defined during your product implementation periods forward in time an assortment plan will calculate by default. This is used where the end of a time frame is undefined, such as when Phase Out is not set for a placeholder.

This does not mean that you cannot have plans further out than this many periods from the current date
Phase In The first period a product is planned to be available for sale in stores
Phase Out The last period a product is planned to be available for sale in stores
Placeholder A non-specific product to which a variant will be assigned later. Users work with placeholders throughout Assortment Planning as if they were variants.
Plan Basis A merchandise financial plan used as the basis for a Target Creation scenario
Product Filter A collection of products defined by user established criteria
Product Hierarchy The structure of product levels within an organization. For example, a 3 level Hierarchy with the levels Category, Gender, and Class, where the Categories are Apparel, Accessories, and Footwear, the Genders and Male and Female, and the Classes include things like knit tops, short sleeve tees, etc.
Rate of Sale (ROS) The average sales per store per period for a given product
Rate of Sale Calculator A supporting module within Demand Management used to create rate of sale scenarios
Rationalization A process which recommends the number of placeholders and collections required to achieve a specified financial target based on user specified data
Scenario A specific set of user defined variables, including product, location, and time dimensions, used to arrive at a probable outcome; relevant for both Target Creation and Rate of Sale Calculator
Seasonality Curve A collection of indices across a window of time used to define the magnitude of sales in each period as it relates to the rate of sale
Site Hierarchy The structure of location levels within an organization. For example, a 3 level Hierarchy with the levels Company, Channel, and District, where Company represents all locations, the Channels are Retail, Wholesale, and Ecommerce, and Districts include things like Northwest, Southeast, etc.
Smart Start The process of generating placeholder items from a Target Creation scenario, assigning them to collections, and seeding them with initial sales plans
Snapshot A save of the assortment plan metric values at a specific point in time. This is used as a means of comparing prior plan states with your working plan.
Target Creation A process through which users can establish plan targets for specific areas of their business and determine the number of items required to meet those targets
Target Segment A user defined subdivision of a product hierarchy level
Variant An actual product to be planned and bought through the Assortment Planning process and often referred to as style-colors or options
Window of Time A contiguous range of periods used to define the data returned in a view or an analysis
