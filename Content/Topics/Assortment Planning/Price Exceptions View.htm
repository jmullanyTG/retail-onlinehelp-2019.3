Price Exceptions View

The Price Exceptions view allows users to manage price exceptions by placeholder for all of the currencies used in the clusters to which that placeholder is assorted.

The system policy Line Visualization Allow Price Exceptions, accessible via Maintenance > Policies > Module: Assortment, sets whether this view is viewable/editable. If set to False, the view still appears but is disabled. To affect a change in this policy on an already open plan, the plan must be reopened through the Manage Assortment Plans dialog.

Conditional Formatting

Orange shaded cells in the Price Exceptions view indicates a change in Local AUT from the prior period for the currency indicated by the column section header.

Reset Default Factors

Restores default translation factors for all non-actualized periods for the product level of the selected placeholder (as defined in the Price Translation Management, and Price Translation Hierarchy Assignment views).

Restoring default factors causes the Local AUT and Home AUT Equivalent fields to recalculate accordingly.

Information displayed in this view

This view reads information from the Currency Exchange Rates, Price Translation Management, and Price Translation Hierarchy Assignment views located in System Utilities > Views > Currency to populate the fields shown but provides users with the flexibility to override any of them to set local ticket prices in whatever way is most meaningful to them.

Field descriptions

Field Description
Period The period being changed. This aligns with the period type for the current assortment plan.
Home AUT

The selected placeholder AUT in default currency for the active assortment plan (as defined in the plan settings – Manage > Default Currency).

This field shows a weighted average AUT for all clusters for the period in questio. If differences exist by cluster and/or from period to period, values shown may not align with those shown in the main planning view or the AUT view.
Home AUT (ex VAT)

(Hidden by default) The selected placeholder AUT excluding VAT in the default currency for the active assortment plan (as defined in the plan settings – Manage > Default Currency)

Calculation: Home AUT / (1 + VAT %)
Local AUT

The ticket price in local currency (as indicated in the column section header – EUR in the shot above) for the selected placeholder for the period.

Calculation: Local AUT ex VAT + (Local AUT ex VAT * VAT %)
Local AUT (ex VAT)

The ticket price excluding VAT in local currency (as indicated in the column section header – EUR in the shot above) for the selected placeholder for the period.

Calculation: Home AUT ex VAT Equivalent * Exchange Rate
Home AUT (ex VAT) Equivalent

The ticket price as a local equivalent price excluding VAT in the home currency after a translation factor has been applied for the selected placeholder for the period.

Calculation: Home AUT ex VAT * Translation Factor
Translation Factor

The translation factor to apply to determine the final ticket price in home and local currencies (as indicated in the column section header – EUR in the shot above) for the selected placeholder for the period.

Calculation: Home AUT ex VAT Equivalent / Home AUT ex VAT

Buttons

Save: Saves all changes and initiates recalculation of blended AUR values for the selected placeholder.

Cancel: Cancels all changes and reverts them to previously saved values.
