Assortment Buy Sheet Report

The Assortment Buy Sheet Report is intended to provide users the ability to view and evaluate planned items within an assortment for a defined time frame.  The report contains key measures for placeholders/variants/collections related to what those entities are, where they are planned to sell, and when they are planned to sell.  Additionally, users have visibility to sales, pricing, margin, and receipts and on order for all items contained within the report. 

Once the values have populated upon hitting execute, users can use this report to slice and dice their assortment for a defined time frame.  This view can be utilized to pass key information to users that do not have access to the system.  For example, merchandise planners can export this report as a PDF or spreadsheet to share with merchants or buyers. 

Additionally, this report is intended to provide insight to how an assortment is planned and how items fit into the assortment in terms of their selling behavior.  For example, by using a product attribute filter, a denim planner could look at how skinny jeans that have a certain wash are planned and how much presence in the assortment these attributes will have as they are merchandised across the selling channels of the organization.

Parameters

The Assortment Buy Sheet Report is controlled by five parameters:

  • Product: Selects a product filter for the current view

    •   Launches the Filter Conditions dialog, allowing the user to dynamically edit the criteria for the selected filter prior to executing it. The option also exists to create a new filter from the adjusted criteria dynamically.
    •   Launches the Maintain Filter dialog, allowing the user to create or import a new filter, or update, delete, or export an existing filter.
  • Site Hierarchy: Selects the levels of the current plan’s Site Hierarchy to include in the current view.

  • Cluster Sets:   Selects the cluster sets to include in the current view. The cluster sets listed include all default and "additionally available" cluster sets for the product level selected in the Product parameter.

While cluster sets are not explicitly part of the Site Hierarchy, they are directly related, in that cluster sets in a given assortment plan must be contained within a single lowest level of that hierarchy; because of this, the Cluster Set drop down will populate with the cluster sets contained beneath the selected Site Hierarchy levels

  • Window of Time: Selects the window of time to display in the current view.

    Any placeholder whose life cycle intersects with any part of the selected Window of Time displays in the view. This allows users to view placeholders holistically as displaying subdivisions of their life cycle might alter the decisions made by users if they have forgotten or are unaware of the full length of time a placeholder is for sale.

    The selected time frame must be a contiguous range of periods otherwise the system still uses the window of time defined by the first and last selected periods.

  • Display Currency: Sets the currency to display for all revenue based fields in the view. This defaults to the Home Currency for the active Assortment Plan.

Report fields

Clicking Execute returns the following columns in the Assortment Buy Sheet Report once the parameters have been set:

Fields Description
Image Displays a thumbnail of the primary image for the placeholder or attached variant.
PH/PLM Code Unique system generated code for the placeholder or if a variant is assigned in Line Visualization, displays the PLM code associated with the assigned variant.
PH/PLM Description Description for the placeholder or if a variant is assigned in Line visualization, displays the PLM description associated with the assigned variant.
Placeholder Status Displays the placeholder status: Open, Created, Assigned, Committed, or Replen.
Avg. Unit Ticket Average unit ticket (AUT) price for the placeholder.
Avg. Unit Retail Average unit retail (AUR) price for the placeholder.
Avg. Unit Cost Average unit cost (AUC) price for the placeholder.
Margin % Margin percentage for the placeholder.

Calculation: (AUR – AUC) / AUR
Phase In The phase in date for the placeholder, first period of planned sales.
Phase Out The phase out date for the placeholder, last period of planned sales.
# Weeks Total number of weeks the placeholder is on offer to the customer.
Rate of Sale $ Average sales per site per period in revenue for the placeholder.
Rate of Sale U Average sales per site per period in units for the placeholder.
Total Sales $ Total sales revenue for the item from Phase in through Phase out.
Total Sales U Total sales units for the item from Phase in through Phase out.
Promo If checked a promotion exists for the placeholder, if unchecked, then no promotion exists.
Plan Receipt $ Total cost of planned receipts for an placeholder from Phase in through Phase out.
Plan Receipt U Total units of planned receipts for an placeholder from Phase in through Phase out.
# of Flows Total number of periods that receipts are planned to deliver for an placeholder.
Site Count Total number of sites that an placeholder is assorted to.
Avg. Units Average amount of planned receipts per site.
On Order U Total number of committed units planned to deliver for an placeholder.
% Commit % of total planned receipts that have been committed to for an placeholder.

Calculation: (On Order U + prior actual receipts) / Planned Receipt U
Cluster Sets Displays the cluster sets assigned to the placeholder.
