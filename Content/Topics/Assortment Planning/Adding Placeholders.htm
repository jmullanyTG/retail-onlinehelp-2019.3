Adding Placeholders

To add new collections, use the New button in Placeholder mode or in Variant view of Collection mode to display the Add Placeholders dialog:

If the check box Current Collection is checked, then when Generate is clicked Coll Code and Coll Desc are set to the values for the collection of the placeholder that was selected when the Add Placeholders dialog was launched.

Manually selecting a collection

To manually select a collection:

  1. Ensure Current Collection is unchecked.
  2. Click in the Coll Code column to display a collection selector.

  3. Double-click a collection from the drop down list.

    You can filter the results by typing in the Collection Desc field.

Creating a new collection

New collections can be created dynamically if you don’t wish to assign the placeholder to an existing one.

  1. Click in the Coll Code column to display a collection selector.
  2. Enter a description for the collection in the Collection Desc field.
  3. Click New.

Field Descriptions

  • New: Adds a single placeholder line.
  • Delete: Deletes the selected placeholder line.
  • # of Placeholders: The number of new placeholder lines to add when Generate is clicked.

  • Mirror Options: When checked, copies the selections made for the topmost placeholder line to any beneath it. It is not necessary to uncheck this option if you want to deviate from the selections made in the topmost placeholder line Simply edit as you would normally and the box unchecks automatically.

  • Current Collection: When checked, automatically sets Coll Code and Coll Desc to the values for the collection of the placeholder that was selected when the Add Placeholders dialog was launched.

  • Generate: Creates the number of new placeholder lines specified in # of Placeholders. If the number of Placeholders is omitted or zero, then the following displays:

Grid columns

  • Coll Code: The collection code to which the placeholder line is assigned.
  • Coll Desc: The corresponding description for Coll Code for the placeholder line.
  • PH Code: A unique system generated code for the placeholder line.
  • PH Desc: The description for the placeholder line.
  • Product Hierarchy: The product hierarchy level to which the placeholder line will be assigned. This defaults to the lowest common product hierarchy level present in the Product filter in the data selection parameters for Placeholder Mode.
  • Cluster Set: The cluster sets to which the placeholder line will be assigned. Defaults to any cluster sets assigned to the Product Hierarchy level in Assortment Plan Defaults.

    New placeholders are assorted to all clusters within the assigned cluster sets by default.

  • Phase In: The phase in date for the placeholder. Selections are limited to periods prior to Phase Out, if set. Defaults to the earliest non-actualized period within the Window of Time in the data selection parameters.
  • Markdown: The markdown date for the placeholder/collection.
  • Phase Out: The phase out date for the placeholder.

    Defaults to the last period within the Window of Time in the data selection parameters.

  • Mkdn No Periods: The time frame for the markdown date to phase out date. Determines what the phase out date should be from the markdown date, once a user enters a markdown date during Placeholder creation (i.e. default is 4, markdown date is 4/29, phase out date will be 5/27, four weeks post markdown.) The default is ignored if the markdown falls outside of the phase in/out window or in the past.
    • If Phase Out is set first and a markdown default is set, then Markdown date will auto-set to N default periods before the user's chosen Phase Out.
    • If Phase Out is set first but there is no markdown default for the selected hierarchy level, then markdown date is left empty. If Markdown Date is set first and a markdown default is set, the Phase Out will auto-populate with the N default periods after the user's chosen Markdown Date.
    • If Markdown Date is set first and there is no markdown default, Phase Out will need to be manually entered by the user before saving, or standard phase out logic applies.
    • Any change to either Markdown Date or Phase Out Date after initial value is set does not preserve the default.
  • Markdown Offset Locked: Links the markdown and phase out dates to each other, so they work dependently on each other, which helps users accurately plan the entire life of the item including markdowns. The standard default mode keeps the markdown and phase out dates independent of each other. Locked mode cannot be enabled on an item with infinite phase out.

    The new mode is set at the Placeholder Instance level.

    • If Markdown date changes, Phase Out also changes to keep the specified number of periods between them.
    • If Phase Out changes, then Markdown date also changes to keep the specified number of periods between them.
    • If markdown falls outside of phase in/out window, or in the past, then the default is ignored and the lock-step mode is disabled. Date selectors will take into account the lock step / default range so that the user cannot select themselves into a contradictory state.
  • AUT: The home currency average unit ticket (AUT) for the placeholder line.
  • Map From Variant: (Optional) Another placeholder item to use as a Like Style from which the system can seed a sales plan. Populates with placeholder items that exist within the selected Product Hierarchy for the line. Type in the relevant field to filter the results.

    If set, this copies the Like PH’s Size Option, Curve Mode, Seasonality Curve, and AUT (unless specified in Ticket Price) can calculate a sales plan based on sales present for the selected Like PH between the Comp Start and End dates specified.

  • Attrib Copy: Only valid if Map From Variant is set. Defines whether any product attributes assigned to the Like PH are copied to the new placeholder line or if any defined attribute defaults for the Product Hierarchy level are assigned instead. For more information, see Attribute Management solution. For more information, refer to the Attribute Management User Guide.
  • Mod Factor: defaults to 1.0. Only valid if Map From Variant is set. Defines a factor to multiply the sales history of the Map From Variant by.

  • Comp Start: Only valid if Map From Variant is set. Defines the start date for the desired window of time from which to capture sales for the Like PH.

    • Selections are limited to periods prior to Comp End, if set
    • Defaults to the Phase In of the Map From Variant

  • Comp End: Only valid if Map From Variant is set. Defines the end date for the desired window of time from which to capture sales for the Like PH.

    • Selections are limited to periods after Comp Start, if set
    • Defaults to the Phase Out of the Map From Variant.
