Splitting Placeholders

The Split Placeholders dialog is available in Placeholder mode. The Split Placeholders dialog is not available in Collection mode.

Placeholder Code: The code of the placeholder to be split.

Placeholder Description: A description of the placeholder to be split.

Number of Sets: The number of placeholders the selected placeholder splits into.

A placeholder with a phase window of 11/1-11/30/2016 has a 2 sets applied. The original line now has a phase window of 11/1-11/15/2016 and the additional placeholder created has a phase window of time of 11/15-11/30/2016.

When applying a number of sets that doesn’t evenly divide into the number of periods in a placeholder’s life cycle, the system allocates 1 additional period onto each created placeholder in set order until the remainder is used.

For example, 15 periods, 4 sets = 4 total placeholders, 3 with 5 periods, 1 with 4 periods.

When applying a number of sets greater than the number of periods in a placeholder’s life cycle, system defaults to a number of placeholders equal to the number of periods with each being allotted one period.

For example, 9 periods, 10 sets = 9 total placeholders, each with a 1 period life cycle.

Placeholders resulting from splits calculate their new sales plans and related metrics based on the sales by period of the original placeholder and the specific periods allotted to each new placeholder:

Buttons

Cancel: Closes the dialog without making any changes to the selected placeholder.

Save: Splits the selected placeholder into the specified number of sets.

  • The original placeholder remains intact after a split only changing its phase dates and related metrics.
  • Additional placeholders resulting from the Split function copy the original PH code and description but have
    • “(Split #)” appended to the code. # is the number of the split in sequence.
    • “(Split from PH ID, number #)” appended to the description. PH ID is the unique system ID for the placeholder.
  • Additional placeholders retain all non-sales related properties of the original placeholder (product level, min pres, seasonality curve, size option, attributes, etc)
