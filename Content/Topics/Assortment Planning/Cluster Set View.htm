Cluster Set View

The Cluster Sets view is displayed or hidden in Placeholder mode using the View Selection drop down menu. The Cluster Sets view contains all the essential metrics for building a sales plan and allows users to make Cluster Set or Cluster level changes to the single placeholder or collection selected in the Main Planning view.

Placeholders with a markdown date

If you select a Placeholder with a markdown date, then the Cluster Sets view refreshes to display a View Style drop down at the top of the view.

The View Style drop down has the following:

Option Description
Default Displays the Totals
Markdown Allows you to view regular and markdown values separately
All Displays all values for an overall view

Data field descriptions

  • Site Count (Active): non-editable decimal (10.1) The average total site count and the average active site count (in parenthesis) of the cluster set/cluster between Phase In and Phase Out. Typically, this metric is a static whole number that equates to the sum of the sites in the cluster set or cluster. However, when multiple cluster set snapshots are present during a placeholder or collection’s life, it is possible for site count by cluster to vary over time which necessitates the need for an average count.

  • AUC (Average Unit Cost): user editable currency ($10.01) The average unit cost price for the cluster set/cluster. Displays the weighted average of AUC values for all periods, both actualized and future, beneath.

    AUC is not be directly editable once a placeholder has begun to actualize (current date is greater than Phase In date).

  • AUT (Average Unit Ticket): user editable currency ($10.01). The average unit ticket price for the cluster set/cluster. Displays the weighted average of AUT values for all periods, both actualized and future, beneath. AUT is assumed to be VAT inclusive and is treated as such for all dependent calculations.

    Because of the potential for multiple currencies, VAT (value added tax) rates, and promotional participation to vary by site within a single cluster, changes to AUT require an Apply to properly calculate the effects on the rest of the plan for a given placeholder/collection. Details for this calculation are available in a supplemental Excel model.

    Period level changes to AUT can be made in the AUT right pane view. See AUT View for more information.

    • AUT is no longer directly editable once a placeholder has begun to actualize (current date is greater than Phase In date).

    • AUR (average unit retail) being an outcome of the calculation mentioned above is not directly editable.

  • Gross Margin %: non-editable percentage (10.01%); gross margin percentage for the cluster set/cluster

    • Calculation: (AUR ex VAT – AUC) / AUR ex VAT

  • Phase In Date: The phase in date for the cluster set/cluster. If set, selections limit to periods prior to Phase Out.

    defaults to the earliest non-actualized period within the Window of Time in the data selection parameters.

  • Markdown Date: The markdown date for the cluster set/cluster. Selections are limited to periods from Phase In through Phase Out. A markdown date tells the system which period to stop respecting the various constraints utilized in the Receipt Plan supporting view including Min Pres.

    Min Pres drops to zero in the period specified by Markdown Date when viewing the by-period plan for the cluster set/cluster in Sales Plan or Receipt Plan view.

    The markdown date can be pushed by changing Phase In or Phase Out.

  • Phase Out Date: The phase out date for the cluster set/cluster. If set, the selections limit to periods after Phase In. For placeholders/collections of Replen Status, Phase Out dates are derived from the maximum Phase Out date set in the Forecasting module for all relevant inventories.

    Defaults to the last period within the Window of Time in the data selection parameters.

  • Curve Mode: The mode of the selected Seasonality Curve.

    • Static: The selected curve is applied exactly as it exists at the time of application and the cluster set/cluster’s curve will not change if changes are made to the curve, whether manually in the Curves module or through dynamic recalculation. Think of this as making a copy of the applied curve.
    • Dynamic: Changes made to the curve will be pushed down to the cluster set/cluster via automated system processes and the dependent sales plans will recalculate when those changes are pushed.

  • Seasonality Curve: The seasonality curve applied to the cluster set/cluster. Used to break down Ttl Sls $ and Ttl Sls U by period in Sales Plan View. Curves must be generated with the Enhanced Filter type to be eligible for use within Assortment Planning and selectable here. For more information, see Sales Plan View.

    Click the ellipsis () within the cell to launch the Curve Selection screen. For more information, see Curve Selection Screen.

  • ROS $ (Rate of Sale Revenue): user editable currency ($10.01). The rate of sale (average sales per site per period) in revenue for the cluster set/cluster.

    • Calculation: (Ttl Sls $ / No. Periods / Site Count)

  • ROS U (Rate of Sale Units): The rate of sale (average sales per site per period) in units for the cluster set/cluster.

    • Calculation: (Ttl Sls U / No. Periods / Site Count)

  • Ttl Sls $ (Total Sales Revenue): user editable currency ($10.01). The total sales revenue for the cluster set/cluster from Phase In through Phase Out.

    • Calculation: (ROS $ * No. Periods * Site Count)

  • Ttl Sls U (Total Sales Units): The total sales units for the cluster set/cluster from Phase In through Phase Out.

    • Calculation: (ROS U * No. Periods * Site Count)

  • AUR (Average Unit Retail): non-editable currency ($10.01). The average unit retail price for the cluster set/cluster.

    Because of the potential for multiple currencies, VAT (value added tax) rates, and promotional participation to vary by site within a single cluster, AUR is not directly editable and is only changed as a result of changes to AUT, Phase In, Phase Out, and promotion applications. Details for this calculation are available in a supplemental MS Excel model.

    • Calculation: (Ttl Sls $ / Ttl Sls U)

  • AUR ex VAT: Non-editable currency ($1.01). Average retail (selling) price excluding VAT per unit for the placeholder item.

    • Calculation: AUR / (1 + VAT %)

  • Min Pres (Minimum Presentation): The desired minimum presentation stock per site for the cluster set/cluster. This represents additional non-sales units to be purchased for the cluster set/cluster to maintain adequate stock for presentation within each store.

    • Calculation: SUM (cluster site count * cluster min pres – for all clusters within a set) / cluster set Site Count (yields a weighted average) or (cluster site count * cluster min pres) for single clusters

  • Ttl Buy $ (Total Buy Cost Value): user editable currency ($10.01). The total buy in revenue for the cluster set/cluster. Changes to this metric directly affect the cluster set/cluster sales plan.

    • Calculation: (Ttl Sls U + (Min Pres * Site Count) + Additional Buy) * AUC

  • Ttl Buy U (Total Buy Units): The total buy in revenue for the cluster set/cluster. Changes to this metric directly affect the cluster set/cluster sales plan.

    • Calculation: (Ttl Sls U + (Min Pres * Site Count) + Additional Buy)

  • Additional Buy (Additional Buy Units): The quantity of additional non-sales and non-min pres units to buy for the cluster set/cluster. The primary function of this field is to show discrepancies between the sum of Ttl Sls U and (Min Pres * Site Count) and Ttl Buy U that can occur because of the ability to directly edit S/T %. To remove unwanted excess units and rebalance a cluster set/cluster’s sales plan, simply zero this field out and click Save.

    • Calculation: (Ttl Buy U – Ttl Sls U – (Min Pres * Site Count))

  • S/T % (Sell-Through Percentage): user editable percentage (10.1%); sets the target sell-through percentage for the cluster set/cluster. Changes to this metric alter Ttl Sls U in relation to Ttl Buy U to achieve the entered S/T %.

    • Calculation: (Ttl Sls U / Ttl Buy U)
  • Receipt ($): non-editable currency ($10.01). The Sum of Receipt Cost from Receipt Plan view for the placeholder/collection at cluster level. This is included as a comparison of the actual receipt plan to the estimated Ttl Buy $ in Cluster Set View.
  • Receipt (U): The sum of Receipt Units from Receipt Plan view for the placeholder/collection at cluster level. This is included as a comparison of the actual receipt plan to the estimated Ttl Buy U in Cluster Set View.

  • Units Per Location: The average units per location for a cluster set/cluster based on Ttl Buy U and Site Count.

    • Calculation: (Ttl Buy U / Site Count)
  • Target S/T %: Defaults to S/T % and has no calculation effect or relationship.
  • S/T %: Percentage (10.1%). The target sell-through percentage for the placeholder/collection. Changes to this metric alter Ttl Buy U to achieve the entered S/T %.
  • Calculation: (Ttl Sls U / Ttl Orders U).
  • Reg S/T %: The calculation is Reg Sales U / Total Orders U.
  • S/T % Delta: The calculation is ( Target S/T % - S/T % ) / ( S/T % ).
