Top Down Sales Planning

The Top Down Sales Plan view within the Visualize section of the Assortment Planning module gives users the ability to manage Assortments top down. This view allows users to:

  • change one or many items at a time in one place
  • modify values at a total level which then spreads proportionate to sales
  • see how sales changes affect stock/receipts

Changes that are saved are then updated to the individual item plan.

The comparison options are selected as a parameter.

Changes made in the Top Down Sales plan view are seen as overrides in the Main Planning View and Sales Plan View.

Cell locking

Locking is done at individual cell level. Only editable cells can be locked.

When a lock is in place, the entire row is colored to indicate lock status overriding any other formatting in place. When the lock is removed, the original formatting returns. Locks in the Top Down Sales Plan view do not carryover into other views.

Parameters

The following parameters are available:

Parameter Description
Product Selects a product filter for the current view.
Site Hierarchy Selects the levels of the current Plan’s Site Hierarchy to include in the current view.
Cluster Sets Selects the cluster sets to include in the current view. The cluster sets listed include all default and "additionally available" cluster sets for the product level selected in the Product parameter.
Window of Time

Returns the entire range between the minimum and maximum date selection and not just specified values. This is due to the BOP and STS rolling logic.

For example, if we selected [week 3 of August, weeks 2 and 3 of September, and all of November] - we should see August to November in its entirety.

Otherwise the values for August and September on the totals level would be different than the values for the selected weeks, and may cause confusion.

Comparison Type

The following options are available:

  • None
  • System Sales
  • User Sales
  • Snapshot options
Comparison Time

Comparison Types None, System Sales, and User Sales have the full set of dates mirroring the Window of Time parameter.

Snapshots will have the dates specified within the snapshot only. This option behaves like the Window of Time parameter where selecting a few weeks may bring back extra rows because of the BOP and STS rolling logic.

Display Currency Sets the currency to display for all revenue based fields in the view; defaults to the Home Currency for the active Assortment Plan.

For additional information, refer to Placeholder and Collection mode parameters.

Metrics notes

Field Description

AUR

Stock To Sales

BOP Units

Always locked and shaded gray to indicate it's not editable

Stock To Sales updates in relation to sales changes, and is flagged when above or below default thresholds.
Sales, Sales Units

When Sales changes Sales Units updates and vice versa

When change to Sales or Sales Units are made to subtotals, totals, and grand totals, then data is spread proportionately and applicable PH sales Sales and Sales Units adjust.
   
   
   

Grid color scheme

The cells in the grid follow the color scheme indicated below:

Color Description
Red text The STS value is lower than the lower threshold or greater than the upper threshold
Red Locked cell
Orange Partially locked cell, shown only on the aggregate level
Yellow Markdown sales
Green Promotion sales
Blue Regular sales
