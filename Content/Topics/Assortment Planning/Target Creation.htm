Creating Scenarios with Target Creation

Selecting the Target Creation button in the Assortment Planning module opens the Scenario Manager. Within Scenario Manager, users can access scenarios relevant to an intersection of the plan product and site hierarchies and a window of time. Users can create as many scenarios as they wish, but only one can be in active use within a plan for a given intersection of product, site, and time. This screen helps users quickly identify scenarios currently in use through scenario status and conditional formatting, and also provides details about selected scenarios without having to edit them.

Target details screen

Selecting New, Copy, or Edit displays a Target Creation screen which includes the following sections: Target Data Selection, Target Segment Definitions, and Target Worksheet.

For more information on this screen, see:

  • Target Data Selection pane field descriptions
  • Target Segment Definitions field descriptions

Creating new targets

To create a new target:

  1. Click New to launch the Target Creation (new) window.

  2. Complete the fields in the Target Data Selection pane as required.

    For information on the fields, refer to Target Data Selection pane.

  3. Click Save.

    The Target Creation screen displays.

  4. If desired, Click New in the Target Segment Definitions section to create a new Target Segment.

    For more information on the fields, refer to Target Segment Definitions .

Creating Target Segments

To create a Target Segment:

  1. Display the Target Creation screen by creating a new Target or selecting the Copy or Edit buttons on the toolbar.

  2. Complete the fields as required.

    For more information on the fields, refer to Target Segment Definitions .

  3. Click Save.

Parameters

The following parameters are available:

Parameter Description
Product Hierarchy Sets the product hierarchy levels a Target Creation scenario must contain to be shown in the Target Creation Scenarios section. Selectable levels are based on the intersection of the product hierarchy assigned to the current assortment plan and the product hierarchy permissions of the active user.
Site Hierarchy Sets the site hierarchy levels a Target Creation scenario must contain to be shown in the Target Creation Scenarios section. Selectable levels are based on the intersection of the site hierarchy assigned to the current assortment plan and the site hierarchy permissions of the active user.
Window of Time Sets the window of time a scenario must intersect with to be shown in the Target Creation Scenarios section. Selectable periods are based on those configured in the system down to the Period set for the current assortment plan.
Display Currency Sets the currency to display for all revenue based fields in the view; defaults to the Home Currency for the active Assortment Plan.

Toolbar buttons

  • New: Launches the Target Creation (new) window to create a new scenario.

  • Copy: Launches the Target Creation window, populated with the Target Data Selection parameters, Target Segment Definitions, and Segment Cluster Details from the selected scenario in the Target Creation Scenarios section.

    • All Target Data Selection and Target Segment Definitions are editable in copied scenarios.

To unlock target segment definitions and cluster details for editing, copied scenarios must first be saved. To do this without changing the scenario’s core data structure, simply change the scenario name to a unique name then click Save.

  • Edit: Launches the Target Creation window, populated with all data from the selected scenario in the Target Creation Scenarios section.

    • All parameters are editable in scenarios of status Created, Calculated, or Rationalized.
    • For scenarios of status Smart Started, Target Segment Definitions and Segment Cluster Details can be edited, but doing so will require recalculation, re-Rationalization, and re-Smart Starting for changes to take effect in the active assortment plan.
  • Delete: Deletes the selected Target Creation scenario.

  • Compare: Launches the Scenario Comparison window, in which users can compare two Target Creation scenarios.

    • Standard Pivot Grid functionality, including ability to graph selected data elements and Export or Open analysis in a variety of popular formats, including Microsoft Excel and Adobe PDF.
    • Only prerequisite for scenario comparison is that scenarios be in Calculated, Rationalized, or Smart Started status.
    • Comparisons of scenarios representing disparate parts of the product/site hierarchies and/or windows of time are possible - they are just of limited usefulness.
  • Smart Start: Initiates the Smart Start process for the selected scenario, generating the number of placeholder items required, and seeding with beginning sales plan data elements.

  • Fields: Launches the customization dialog.

  • Save: Commits changes to the database.
  • Cancel : Cancels any changes.
