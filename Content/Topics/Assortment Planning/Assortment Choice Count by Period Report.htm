Assortment Choice Count by Period Report

The Assortment Choice Count by Period Report is intended to provide users the ability to view and evaluate planned number of choices and productivity of the planned choices of an assortment for a defined time period.  The report contains key measures at an aggregate level of the product hierarchy or product filter related to the choices planned for an assortment separating the choice counts by sales type.  The report will display how many choices for a given period are regular price, on promotion, or designated as markdown. 

Additionally, choice counts can be measured against the same period last year to determine the balance of assortment as it relates to sales by type (reg. price, promotion, and markdown).  Other key measures that are contained within this report are related to Total Sales, Total Cost of Sales, and Total Margin.

Once the values have populated, users can use this report to slice and dice their assortment in reviewing choice counts for a defined time period.  This view can be utilized to pass key information to users that do not have access to the system.  For example, merchandise planners can export this report to a PDF or Excel to share with merchants, buyers or visual merchandisers. 

Additionally, this report is intended to provide insight to how an assortment is planned and how balanced an assortment looks in comparing regular price, promotional and markdown choices into the assortment over time. For example, a denim planner can illustrate the balance of fit types planned for a period and how many of each fit type are planned to be regular price, on promotion, or on markdown. 

This report makes it very easy to draw comparisons to last year in total sales, COGS, margin, or productivity. By using the Graph functionality with this report, users can easily see movement in all choice types as a season progresses to see if they remain in line with overall strategies for the business. These insights may invoke discussion in the organization as to what the trends of the business are and if the trends are covered in reviewing future period plans.

Parameters

The Assortment Choice Count by Period Report is controlled by seven parameters:

  • Product: Selects a product filter for the current view.

    •   Launches the Filter Conditions dialog allowing the user to edit the criteria for the selected filter prior to executing it. Users can also create a new filter from the adjusted criteria.

    •   Launches the Maintain Filter dialog allowing the user to create, import, and work with filters.
  • Site Hierarchy: Selects the levels of the current plan’s Site Hierarchy to include in the current view.

  • Cluster Sets:   Selects the cluster sets to include in the current view. The cluster sets listed include all default and "additionally available" cluster sets for the product level selected in the Product parameter.

While cluster sets are not explicitly part of the Site Hierarchy, they are directly related in that cluster sets in a given assortment plan must be contained within a single lowest level of that hierarchy. Therefore, the Cluster Set drop down populates with the cluster sets contained beneath the selected Site Hierarchy levels.

  • Window of Time: Selects the window of time to display in the current view.

The selected time frame must be a contiguous range of periods otherwise the system still uses the window of time defined by the first and last selected periods.

  • Attribute Family:   Each attribute family contains attributes that pertain to that family and can be imported from host systems if available. 

Attribute Family = Fabric Type, and a value belonging to this family could be cotton.  This selection defines the values within a family or group of families that will be available for selection and viewing in the sixth parameter.

  • Attribute Values:  This selection returns the values associated with the attribute families selected in the fifth parameter.

It is not necessary to select Attribute Family or Attribute Values in the parameters selections as it is possible to review choice counts and productivity through the utilization of a product filter, Site Hierarchy, Cluster Sets for a defined window of time.

  • Display Currency: Sets the currency to display for all revenue based fields in the view. This field defaults to the Home Currency for the active Assortment Plan.

Deriving Choice Counts:

  • Historical Periods - choice counts for each type (Regular, Promo, Mkdn) are derived from the presence of sales of the corresponding type for a given variant.

  • Planned (future) Periods

    • Regular Choices - sales are planned for the period but no promotion is in effect, and the Mkdn Date, if set, has not yet been reached
    • Promo Choices - sales are planned for the period a promotion is in effect and the Mkdn Date, if set, has not yet been reached
    • Mkdn Choices – sales are planned for the period and the period is the same as or past the period set as the Mkdn Date
