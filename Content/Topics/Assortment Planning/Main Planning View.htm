Calculations

All calculations listed in this section are generic calculations only provided for the purposes of checking the general accuracy. However, the actual calculations used may vary depending on what metric was changed to cause a recalculation. Details can be found in the supplemental Line Visualization Calculation Behaviors MS Excel document.

When checking values using the generic calculations, it should be understood that values are presented in a rounded state for ease of display. Small differences in the actual calculated results may be present. In the database, all calculations are done with maximum precision to avoid unnecessary rounding.

Collection Level Check boxes

Check boxes in Collection mode such as those in the Active and Carryover fields display a solid gold state to indicate that only some placeholders within the collection are checked.

Reviewing pre-markdown and post-markdown metrics in Placeholder mode

The following columns have been added to the grid in the Placeholder mode allowing users to review pre-markdown and post-markdown metrics:

Grid columns

Active: Whether the current placeholder/collection is active in the assortment plan or not. Defaults to checked. Unchecking strikes through the placeholder/collection line and zeros out its Collection % Cont.

All aspects of the placeholder/collection sales plan are retained for seamless switching between active and inactive states. Inactive placeholders/collections are not included in any reporting within Assortment Planning: Target Creation, Assortment Rollup, Reconciliation, and Dashboards.

Placeholders/collections can also be made active or inactive by right clicking > Set Active / Set Inactive. Placeholders/collections of Replen Status cannot be made Inactive.

Locked: Locks or unlocks the collection the placeholder belongs to (or the collection in Collection mode) for editing. When checked, all placeholders attached to the collection are locked and read-only in the view and highlighted in pink/red depending on the row band.

Actualization of sales in a period can still affect locked placeholders/collections. Creation of new cluster set snapshots for cluster sets to which locked collections are assorted can also change locked placeholders/collections. This is done to maintain synchronicity with ROS metrics and the site count associated with clusters to which locked placeholders/collections are assorted.

Notes:

  • Recalculation of linked Seasonality Curves does not affect locked placeholders/collections.
  • Already active promotions remain active on locked placeholders/collections.
  • New promotions cannot be applied to locked placeholders
  • Placeholders/collections can also be locked by right clicking and selecting Lock Selected or Unlock Selected.

Product Level: The level of the product hierarchy to which the placeholder/collection is assigned. Recalculation of linked Seasonality Curves does not affect locked placeholders/collections. This selection does not have to be made at the lowest level of the product hierarchy, though it is recommended users do so to avoid ambiguity in reporting.

PH Collection Code: The collection to which the placeholder is assigned (read-only in Collection mode).

Filter the list by typing in the Code or Des fields. Use % to do a “Contains” filter condition rather than “Starts With”. For example, ”%Tops” returns everything containing the word “Tops”.

  • Double-click a collection in the list to change the placeholder’s assignment.
  • Enter a new collection description and click New to create a new collection and assign the selected placeholder to it.
  • Changing a placeholder’s collection assignment recalculates both its prior collection’s % and its newly assigned collection’s % contribution for all placeholders that belong to either.

PH Coll Desc: A description for the collection. Changing this for one placeholder changes it for all other placeholders currently assigned to that collection.

PH Code: (not present in Collection mode) A unique system generated code for the placeholder.

PH Desc: (not present in Collection mode) A description for the placeholder. Placeholders generated through the Smart Start process default to Smart Started from Segment and (segment name) for easy identification.

Cluster Sets: The cluster sets the placeholder or collection assorts to as set in the Cluster Details side pane.

Status: The status for the placeholder collection (read-only in Collection mode):

  • Open: Default for all new placeholders. Denotes placeholders that can be deleted by Smart Starting an overlapping Target Creation scenario. For more information, see Target Creation.
  • Created: Denotes placeholders that cannot be deleted by Smart Starting and overlapping Target Creation scenario.

  • Committed: Auto-assigned upon confirmation of units on-order for the placeholder.

    As actual units on-order is always against a variant (real style-color), once a placeholder enters Committed status, the assigned variant cannot be removed nor can the status be changed unless the on-order units are canceled.

  • Assigned: Auto-assigned upon assignment of a variant to the placeholder. Assigned placeholders inherit the Variant ID, PLM Code, PLM Description, Color Code, Color Description, and Attributes of the assigned variant. For more information, see Attributes View.

    Placeholders whose variants are unassigned via Undo > Unassign Selected are automatically returned to Created status.

Replen: A selectable for placeholders that are either in Assigned or Committed Status. This shifts control of the placeholder/collection sales plan to replenishment forecasts in the Forecasts module making the plan read-only within Assortment Planning.

Changing placeholder/collection Status of placeholders with replenishment data present

Changing placeholder/collection Status of placeholders with replenishment data present (see Available for Replen) to Replen displays the following:

Clicking Yes irreversibly overwrites the sales plan for the changed item with available replenishment forecast data. While you can change the item Status back, the system is not able to maintain the previous sales plan data. The state of the sales plan at the time of the change, including metric values, cluster assignments, and location exceptions, will become the go-forward sales plan, though the plan elements will become editable once more.

Changing placeholder/collection Status of placeholders without replenishment data present

Changing placeholder/collection Status of placeholders without replenishment data present (see Available for Replen) to Replen displays the following:

Clicking Yes irreversibly zeros out the sales plan for the changed item. While you can change the item Status back, the system is not able to maintain the previous sales plan data. Plan metrics stay at zero though the plan elements are editable once more.

Notes on Replen items:

  • Sales plans for Replen items are updated nightly to coincide with updates to replenishment forecasts.

  • Replen item sales plans can also be updated on demand by clicking the Recalculate button in the Replen section of the toolbar. However, this utilizes the settings of the most recent replenishment run.

    It is advisable to verify the output of the last replenishment run created using the same parameters as are used in the nightly run or the data brought into AP may be inconsistent or incomplete .

  • As only future data is ever written in a replen run and data is written daily, this requires the system to specially handle inter-period sales and on-hand inventory. BOP on-hand is created from actuals and not changed throughout the period. Sales and receipts are combined as "future periods from replenishment" and "past periods from actuals" to maintain a total period number throughout the period until it is fully actualized. EOP on-hand values always come from replenishment data.
  • Certain plan elements still remain editable. Most are informational fields (PH Description and Code, Tags, Notes) but all Shrink/Inventory Adjustments metrics in Sales Plan View are editable.
  • Replen items cannot be locked, made inactive, or deleted.
  • Seasonality Curve, Size Curve, and Min Pres are cleared as these elements are controlled within the Forecasts module. If the status changes from Replen to something else, the item inherits the defaults set on the plan for the product level to which it belongs.
  • Any Sales Plan and Receipt Plan overrides are irrevocably cleared.

  • Receipt Plan constraints are cleared as these elements are controlled within the Forecasts module. If the status changes from Replen to something else the item inherits the defaults set on the plan for the product level to which it belongs.

    Currently, Forecasts uses buckets and Assortment Planning uses periods. These are two different time entity types that do not have to align in order for Assortment Planning to read in replenishment forecast data.

    The system attempts to match the calendars in each module by cross referencing Forecast bucket start dates with the Assortment Planning periods beginning and ending dates. However, if the buckets and periods do not coincide, this may lead to sub-optimal data within Assortment Planning.

    For example, if the buckets in Forecasts are set to monthly and the periods in AP are weekly, AP puts the sales for the entire monthly bucket into the first weekly period available during that month, etc.

    In Collection mode, displays the most advanced status of any placeholders within the collection (with Open being least and Replen being most advanced).

Receipt Status: The calculation status of the receipt plan for the placeholder/collection (Clean or Dirty).

  • In the main planning and Cluster Set views, changes to Seasonality Curve, Collection % Contribution, Phase In, Mkdn Date, Phase Out, AUT, Ttl Sls $, Ttl Sls U, ROS $, ROS U, Min Pres, Ttl Buy $, Ttl Buy U, Additional Buy, or S/T will set this to Dirty.
  • In Sales Plan view, changes to Selling Period, Sales, Sales Units, Shrink Units, Shrink Units % of Sales Units, Shrink Cost, Shrink Cost % of COGS, Inventory Adjustment Units and Inventory Adjustment Cost will set this to Dirty.
  • In Receipt Plan view, changes to Periods of Supply, Periods Before Phase In, Min Stock to Sales, Shrink % of Sales, Minimum Presentation, Receipt Period, and Receipt Units will set this to Dirty.

Available for Replen: read-only check box; indicates if replenishment forecast data is available for the placeholder/collection

Check this field before changing the status of a placeholder/collection to Replen. The system warns if no data is present, however, checking first saves time. This is also useful for verifying that the replenishment forecast matches up with the assigned variants.

This field is checked if ANY portion of the combination assigned variants, cluster sets, and product lifespan has a replenishment forecast against it. For example, the field is checked if a single item master related to the assigned variant has a forecast for a single site contained within the clusters that variant is assorted to during the lifespan of the variant.

Forecasts must be set for all intended item masters and sites to ensure completeness of replenishment data in Assortment Planning.

Replen Status: The Replen Status, Clean or Dirty, as an indicator of if the placeholder/collection has been updated with the most recent Replenishment forecast data (valid for placeholders/collections of Replen status only).

Last Replen: The date replenishment data was last updated for the placeholder/collection (valid for placeholders/collections of Replen status).

PH Count: (not present in Placeholder mode) The count of the placeholders assigned to a collection.

Variant ID: (not present in Collection mode) A unique system identifier for the assigned variant; blank if no variant is assigned.

PLM Code: (not present in Collection mode) If integrated with a product lifecycle management (PLM) system, then this field displays the PLM code associated with the assigned variant.

Color Code: (not present in Collection mode) The color code for the placeholder. When a variant is assigned it overwrites any user-entered value and displays the variant color code. On unassigning a variant, reverts back to the user-entered value, if any.

Color: (not present in Collection mode) The color description for the placeholder. When a variant is assigned it overwrites any user-entered value and displays the variant color description. On unassigning a variant, it reverts back to any user-entered value.

Supplier Code: The unique system supplier code of the supplier associated with the variant assigned to the placeholder, if any.

Supplier Name: The supplier name associated with the variant assigned to the placeholder, if any.

Default Lead Days: Either the Default Lead Time Days set for the Product Level the placeholder is assigned to or the Default Lead Time of the supplier associated with the variant assigned to the placeholder.

If a variant is assigned to the placeholder but that variant has no associated supplier, the system uses the Default Lead Time Days set in the plan defaults. For more information on setting default lead days, see Tracking Against Ordering Deadlines.

Next Order Date: The earliest order deadline for receipts planned for the corresponding placeholder based on assigned default lead times. Essentially, this is the earliest receipt date for any cluster level receipt planned for the placeholder minus Default Lead Days.

This should also correspond with the earliest Order By Date shown in Receipt Plan view for the placeholder.

Size Curve: The size curve to apply to the placeholder/collection. Displays all defined size curves. Filter by typing in the Size Curve Option or Description fields; use % to do a “Contains” filter condition rather than “Starts With” (ex. %Tops returns everything containing the word “Tops”).

Applying at the collection level pushes a size curve down to all placeholders in a collection The system has no way to validate the relevance of sizes contained in the applied curve to new placeholders.

Curve Mode: The mode of the selected Seasonality Curve:

Mode Description
Static The selected curve is applied exactly as it exists at the time of application and the placeholder/collection’s curve will not change if changes are made to the curve, whether manually in the Curves module or through dynamic recalculation. Think of this as making a copy of the applied curve.
Dynamic Changes made to the curve are pushed down to the placeholder/collection and the dependent sales plans recalculate when those changes are pushed.

Seasonality Curve: The seasonality curve applied to the placeholder/collection. Used to break down Ttl Sls $ and Ttl Sls U by period in Sales Plan View. For more information, see Sales Plan View.

Only curves of the Manual or Specified Conditions Using Enhanced Filtering types are available for selection within Assortment Planning.

Click the ellipsis () to launch the Curve Selection screen. For more information, see Curve Selection Screen.

Image: (not present in Collection mode) A thumbnail of the primary image for the placeholder or attached variant. If Image Tool tip is enabled in View Options, then hovering over the thumbnail image displays a larger version of the image.

Images can be assigned to placeholders prior to variants being attached, but are replaced by the actual variant images once a variant is attached. To retain placeholder images when attaching variants, go to Variant Manager > Image Maintenance for the variant you wish to attach and add the placeholder images there, or add them after the variant has been attached.

Where a PLM (Product Lifecycle Management) system integration is present, the primary image of a placeholder is not editable once a variant has been assigned. This is because the image is dictated via data imported from the PLM. The ability to edit the primary image is controlled by the system policy Variant Primary Image Classification Update (Maintenance > Policies > Module: Assortment).

To edit, click the image, right click and choose Edit Primary or Edit All:

  • Edit Primary displays the Image Maintenance dialog (see Header Area Controls > Images earlier in this section) and allows selection of another primary image
  • Edit All displays the Image Gallery dialog (see Header Area Controls > Images earlier in this section) and allows editing/deletion of existing images or addition of more images; this can also be launched by double-clicking the image thumbnail

Carryover: Whether a placeholder/collection is an already existing item from a previous window of time. This is driven solely off a placeholder status of Committed. Committed placeholders/collections have confirmed on order quantities for actual variants and cannot be edited to the same degree as placeholders/collections that have yet to be ordered.

Collection % Cont: user editable percentage (10.1%); displays/sets the contribution percentage of a placeholder to the collection to which it belongs (not present in Collection mode). Placeholders in collections by themselves will always have Collection % Cont of 100%. When shifting placeholder collection assignments, percentages for all placeholders in affected collections are recalculated, but sales hold for all. When adjusting Collection % Cont directly, collection level sales hold and placeholder sales recalculate.

As changes are made to this metric within a collection, the % Locked field will be checked, so subsequent changes won’t alter your previous entries within the collection. Sum of Collection % Cont for all placeholders in a collection must equal 100%; system will re-normalize entries that would cause the sum to exceed by dividing each by the sum of the percentages.

Calculation: Ttl Sls U / (Ttl Sls U for all placeholders in collection).

Entering values above 100% will raise an Invalid Value error.

% Locked: Indicates a change has been made to the corresponding % contribution that has not yet been Applied (not present in Collection mode).

AUT: Currency ($10.01) The average unit ticket price for the placeholder/collection. Displays the weighted average of AUT values for all periods, both actualized and future, beneath.

AUT is assumed to be VAT inclusive and is treated as such for all dependent calculations. Because of the potential for multiple currencies, VAT (value added tax) rates, and promotional participation to vary by site within a single cluster, changes to AUT require you to click Apply to properly calculate the effects on the rest of the plan for a given placeholder/collection. Details for this calculation are available in a supplemental Excel model.

Period level changes to AUT can be made in the AUT right pane view. See AUT View for more information.

  • Once a placeholder has begun to actualize (current date is greater than Phase In date), AUT is not directly editable.

  • AUR (average unit retail), being an outcome of the calculation mentioned above, is not directly editable.

Margin %: read-only percentage (10.01%); gross margin percentage for the placeholder/collection

Calculation: ((AUR ex VAT / (1 + VAT %)) – AUC) / (AUR ex VAT/ 1 + VAT %)

Tags: user editable alpha-numeric; allows the ad-hoc creation and assignment of informational descriptors for a given item

Multiple tags can be entered for a single line with inputs being confirmed by the use of the comma ( , ) key. As users begin to type, any tags already present in the current view that begin with the entered characters appear in an auto-complete selector below the field. These entries can be selected via click or by using the arrow keys to navigate to the desired item and the comma key to select.

Phase In: The phase in date for the placeholder/collection. If set, selections are limited to periods prior to Phase Out.

Defaults to the earliest non-actualized period within the Window of Time in the data selection parameters

Mkdn Date: Selects the markdown date for the placeholder/collection. Selections are limited to periods from Phase In through Phase Out. Mkdn Date sets which period to stop respecting the various constraints utilized in the Receipt Plan supporting view including Min Pres. When viewing the by-period plan for the placeholder in Sales Plan or Receipt Plan view, Min Pres will drop to zero in the period specified by Mkdn Date. An existing markdown date splits the plan into pre- and post-markdown data. Promotions can be applied to markdown periods.

  • If no markdown date has been specified, then the plan exists as standard.
  • If a markdown date is added at time of the Placeholder creation, then indexes for both halves of the plan are normalized together, and total sales is spread across.
  • If a markdown date is added after the total sales has been entered, then sales plan pre-markdown is locked in as it already exists, and the markdown plan is re-angled to head to zero.

Mkdn date can be pushed by changing Phase In or Phase Out. If Phase Out changes, the zero-trend point moves with the phase out and the slope of the mkdn curve changes accordingly.

Phase Out: The phase out date for the placeholder/collection. If set, selections are limited to periods after Phase In. Defaults to the last period within the Window of Time in the data selection parameters.

For placeholders/collections of Replen Status, Phase Out dates are derived from the maximum Phase Out date set in the Forecasting module for all relevant inventories.

SPV Overrides: Indicates if overrides to Sales $ or Sales U have been made to the placeholder/collection in the Sales Plan supporting view

Changes made in Sales Plan view do not directly affect the metrics in Line Visualization with the exception of removing Selling Periods as these metrics are considered the System Plan when working in Sales Plan view. This allows the user to see differences in the higher level placeholder and cluster plans created in Line Visualization and the results of more granular period level adjustments made in Sales Plan view.

SPV Sls $:The sum of all Sales for the placeholder/collection in Sales Plan view. Provided for easy comparison to the Line Visualization plan at the placeholder/collection level.

SPV Sls U: The sum of all Sales Units for the placeholder/collection in Sales Plan view. Provided for easy comparison to the Line Visualization plan at the placeholder/collection level.

ROS $: user editable currency ($10.01); sets the rate of sale (average sales per site per period) in revenue for the placeholder/collection

Calculation: (Ttl Sls $ / No. Periods / Site Count)

ROS U: The rate of sale (average sales per site per period) in units for the placeholder/collection

Calculation: (Ttl Sls U / No. Periods / Site Count)

Promo: Indicates whether there are any active promotions applied to the placeholder/collection. Checked means at least one promotion is applied to the placeholder/collection. Unchecked means no promotions are applied. Inactive promotions will not cause the box to be checked.

Specific promotions applied can be viewed in the Promotions bottom pane view. For more information, see Promotions View (not available in Collection mode).

Ttl Sls $: user editable currency ($10.01); sets the total sales revenue for the placeholder/collection from Phase In through Phase Out

Calculation: (ROS $ * No. Periods * Site Count

Ttl Sls U: user editable integer; sets the total sales units for the placeholder/collection from Phase In through Phase Out

Calculation: (ROS U * No. Periods * Site Count)

AUR: read-only currency ($10.01); displays the average unit retail price for the placeholder/collection. Because of the potential for multiple currencies, VAT (value added tax) rates, and promotional participation to vary by site within a single cluster, AUR is not directly editable and is only changed as a result of changes to AUT, Phase In, Phase Out, and promotion applications. Details for this calculation are available in a supplemental Excel model.

Calculation: (Ttl Sls $ / Ttl Sls U)

AUR ex VAT: Non-editable currency ($1.01); average retail (selling) price excluding VAT per unit for the placeholder item

Calculation: AUR / (1 + VAT %)

AUC: user editable currency ($10.01); sets the average unit cost price for the placeholder/collection. Displays the weighted average of AUC values for all periods, both actualized and future, beneath.

Once a placeholder/collection has begun to actualize (current date is greater than Phase In date), AUC will not directly editable.

Min Pres: user editable integer; sets the desired minimum presentation stock per site for the placeholder/collection. Min Pres represents additional non-sales units to be purchased for the placeholder/collection to maintain adequate stock for presentation within each store.

Calculation: SUM (cluster site count * cluster min pres – for all clusters) / Site Count (yields a weighted average)

Ttl Buy $: user editable currency ($10.01); sets the total buy in revenue for the placeholder/collection. Changes to this metric directly affect the placeholder/collection sales plan.

Calculation: (Ttl Sls U + (Min Pres * Site Count) + Additional Buy) * AUC

Ttl Buy U: The total buy in revenue for the placeholder/collection. Changes to this metric directly affect the placeholder/collection sales plan.

Calculation: (Ttl Sls U + (Min Pres * Site Count) + Additional Buy)

Additional Buy: The quantity of additional non-sales and non-min pres units to buy for the placeholder/collection. The primary function of this field is to show discrepancies between the sum of Ttl Sls U and (Min Pres * Site Count) and Ttl Buy U that can occur because of the ability to directly edit S/T %. To remove unwanted excess units and rebalance a placeholder/collection’s sales plan, simply zero this field out and Apply.

Calculation: (Ttl Buy U – Ttl Sls U – (Min Pres * Site Count))

Target S/T %: user editable, has no calculation effect or relationship, defaults to S/T %.

S/T %: read-only. percentage (10.1%); sets the target sell-through percentage for the placeholder/collection. Changes to this metric alter Ttl Buy U to achieve the entered S/T %.

Calculation: (Ttl Sls U / Ttl Orders U)

Reg S/T %: read-only. The calculation is Reg Sales U / Total Orders U.

S/T % Delta: The calculation is ( Target S/T % - S/T % ) / ( S/T % ).

Total Orders ($): The sum of Total Orders Cost from Receipt Plan view for the placeholder/collection. This is included as a comparison of the actual receipt plan to the estimated Ttl Buy $ in the Main Planning View.

Total Orders (U): The sum of Total Orders from Receipt Plan view for the placeholder/collection. This is included as a comparison of the actual receipt plan to the estimated Ttl Buy U in the Main Planning View.

No. Periods: The MAX period count for the placeholder/collection. Because Phase In and Phase Out at the placeholder/collection level are set to the earliest and latest periods, respectively, of all clusters to which it has been assorted, No. Periods must calculate this way to balance the other calculations in Line Visualization.

A placeholder is assorted to two clusters: cluster A has Phase In/Phase Out of Wk 1 and Wk 4 and cluster B has a Phase In/Phase Out of Wk 2 and Wk 5.

The placeholder Phase In/Phase Out is Wk 1 and Wk 5, yielding a No. Periods of 5.

Site Count (Active): The average total site count and the average active site count (in parenthesis) of all clusters a placeholder/collection is assorted to between Phase In and Phase Out.

Typically, this metric is always a static whole number that equates to the sum of the cluster level site counts to which the placeholder/collection is assorted. However, when multiple cluster set snapshots are present during a placeholder/collection’s life, it is possible for site count by cluster to vary, thus necessitating the need for an average count.

Location Exceptions: Indicates the presence of location level exceptions for the placeholder/collection. This is checked if any sites contained in clusters the placeholder/collection is assorted to have been excluded via the Manage Locations function.

Location exceptions do not alter Site Count. This is because when location exceptions are made the system recalculates the placeholder/collection rates of sale according to each site’s % contribution to the cluster. Also decrementing Site Count would cause the resulting sales plan to drop too low as doing so essentially double-counts the removal of the excluded sites.

Lock Type: Displays the lock type if any part of a placeholder/collection line is locked.

Health Error: An error message indicating if a cluster has zero sites in a particular time frame. This column and the Health Error Details column together allow users to better plan their assortments.

Health Error Details: Click the drop-down arrow to display additional details regarding messages displayed in the Health Error field.

Example of Cluster Sets with zero site counts

PLM Desc: If integrated with a product lifecycle management (PLM) system, then this displays the PLM description associated with the assigned variant (not present in Collection mode).

Actual Sales U: The sum of all actualized sales units (from historical periods) for the placeholder/collection.

Actualization occurs as part of overnight processing (exact frequency can vary by implementation) in which the sales units that were realized for a given period are fed to the Demand Management database and read into Assortment Planning. Planned sales values are overwritten by these actual sales values.

Actualized Periods: The number of actualized periods present in the life of the placeholder/collection.

For a placeholder whose Phase In was Wk 1 and it is now Wk 5 and assuming regular overnight processing has occurred, then the actualized periods is 4.

Actualized: Indicates the presence of any actualized periods during the placeholder/collection’s life.
